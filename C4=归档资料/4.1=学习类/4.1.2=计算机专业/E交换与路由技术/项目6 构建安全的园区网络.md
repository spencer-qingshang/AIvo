# 项目 6：构建安全的园区网络 (专家级深度笔记)

## 一、 接入层安全：端口安全 (Port Security) 详解

### 1.1 MAC 地址安全分类与生命周期
交换机通过端口安全功能将接口学习到的 MAC 地址转换为特定类型，从而控制接入主机的合法性。

| MAC 类型 | 学习方式 | 存储位置 | 特点与应用场景 |
| :--- | :--- | :--- | :--- |
| **Dynamic (动态)** | 自动学习 | RAM (MAC 表) | 接口重启或老化时间到期即丢失。默认安全级别。 |
| **Static (静态)** | 手动配置 | 配置文件 | 永久保存。适用于服务器等固定设备，维护成本高。 |
| **Sticky (粘性)** | **自动学习并转化** | 配置文件 | **核心考点**：保存配置后重启不丢失。兼顾动态学习的便捷与静态存储的可靠性。 |

### 1.2 保护动作 (Violation Mode) 深度对比
当学习到的 MAC 数量超过 `max-mac-num` 或出现未授权 MAC 访问时触发。

*   **Protect (保护)**：丢弃违规数据帧。**静默处理**，管理员无法通过日志感知攻击。
*   **Restrict (限制)**：丢弃违规数据帧。**触发告警**（SNMP Trap & Log），并增加 `Violation` 计数。生产环境最常用，便于审计。
*   **Shutdown (关闭)**：接口直接进入 **Error-down** 状态。彻底切断物理连接。
    *   *恢复手段*：手动 `undo shutdown` 或配置 `error-down auto-recovery`（自动恢复）。

### 1.3 专家级验证指令
```shell
display port-security interface G0/0/1  # 查看指定接口的安全配置及状态
display mac-address sticky              # 查看已学习到的粘性MAC地址
```

## 二、 管理平面安全：AAA 架构与远程管理

### 2.1 AAA 逻辑架构
*   **Authentication (认证)**：你是谁？（用户名/密码）
*   **Authorization (授权)**：你能干什么？（0-15 级权限）
*   **Accounting (计费/审计)**：你干了什么？（操作日志）

### 2.2 权限等级划分 (Privilege Level)
*   **0 (访问级)**：网络诊断工具（Ping/Tracert）。
*   **1 (监控级)**：查看配置（Display）。
*   **2 (配置级)**：业务配置（ACL/VLAN）。
*   **3-15 (管理级)**：所有权限，包括删除文件、重启设备。

### 2.3 SSH (STelnet) vs Telnet (安全性分析)
*   **Telnet (TCP 23)**：明文传输。中间人通过抓包可直接获取管理员密码。**自考常考：为什么不推荐用 Telnet？**
*   **SSH (TCP 22)**：加密传输。通过 RSA/ECC 非对称加密建立安全隧道，支持身份认证。

### 2.4 核心配置闭环
```shell
# RSA密钥对生成（SSH必备）
rsa local-key-pair create

# AAA 局部用户定义
local-user huawei password pbkdf2 Huawei@123  # PBKDF2为更安全的加密算法
local-user huawei service-type ssh telnet
local-user huawei privilege level 15

# VTY 线路安全增强
user-interface vty 0 4
 authentication-mode aaa
 protocol inbound ssh    # 仅允许SSH，彻底禁止Telnet
```

## 三、 ACL 访问控制：精准流量清洗

### 3.1 匹配机制：命中即止 (First Match)
ACL 规则按照 `Rule-ID`（默认步长为 5）从小到大排列。一旦报文匹配到某条规则，立即执行 `Permit` 或 `Deny`，不再继续下查。
*   **隐含规则**：华为 ACL 在匹配结束若无任何规则命中，**默认动作为 Permit (允许)**（注意：防火墙默认为 Deny）。

### 3.2 高级 ACL 的五元组过滤
高级 ACL（3000-3999）能够基于 **源IP、目的IP、协议号、源端口、目的端口** 进行精细化过滤。
*   **考试高频端口号**：
    *   TCP: FTP(21/20), SSH(22), Telnet(23), HTTP(80), HTTPS(443)
    *   UDP: DNS(53), DHCP(67/68), SNMP(161/162)

### 3.3 通配符掩码 (Wildcard Mask) 逻辑
*   **0**：检查该位，必须匹配。
*   **1**：不检查该位（忽略）。
*   *例*：`192.168.1.0 0.0.0.255` 匹配整个 192.168.1.0 网段。

## 四、 边界安全：NAT 与 PPP 认证

### 4.1 PPP 认证双雄对比
PPP 是广域网（WAN）最常用的二层协议，其安全性由认证协议保障。

| 特性 | PAP (Password Authentication Protocol) | CHAP (Challenge Handshake Authentication Protocol) |
| :--- | :--- | :--- |
| **握手方式** | **两次握手**（认证请求 -> 确认） | **三次握手**（挑战 -> 响应 -> 确认） |
| **密码安全性** | **明文**传输。 | **密文**摘要传输。密码不在链路上跑。 |
| **主导权** | 被认证方主动发起请求。 | 认证方主动发起挑战（更安全）。 |

### 4.2 NAT 应用场景与 NAPT 原理
*   **NAT Server (静态NAT)**：公网 IP:80 <-> 私网服务器 IP:80。实现外部访问内部。
*   **Easy-IP (NAPT)**：利用端口号（Port）区分不同内部主机。
    *   *原理*：修改 IP 包头的源 IP + 源 Port。通过 **NAT 转换表** 记录映射关系。

## 五、 防火墙核心：状态检测与安全区域

### 5.1 安全区域 (Zone) 的本质
防火墙通过安全区域来划分网络的可信程度，区域间（Interzone）必须配置安全策略。
*   **优先级 (Priority)**：Local(100) > Trust(85) > DMZ(50) > Untrust(5)。
*   **默认逻辑**：同一区域内流量默认放行；不同区域间流量默认拒绝。

### 5.2 状态检测技术 (Stateful Inspection)
防火墙不仅看首包的五元组，还会记录连接状态。
*   *Outbound 流程*：内网 Trust 主机发起请求，防火墙记录状态 -> 回包时直接匹配状态表通过，无需再查安全策略。

## 六、 考试避坑与实战 Tips (CheckList)

1.  **ACL 编号记混**：2000 开头是基本（只看源 IP），3000 开头是高级。
2.  **配置 ACL 没生效**：检查是否在接口下配置了 `traffic-filter`，以及方向（Inbound/Outbound）是否正确。
3.  **NAT 配置没流量**：Easy-IP 的 ACL 规则中必须是 `permit` 要转换的流量。
4.  **远程登录进不去**：检查 `user-interface vty` 下是否配置了 `authentication-mode aaa`。
5.  **防火墙策略失效**：检查源/目的区域是否选反，以及是否有更高优先级的规则被命中。

## 七、 实验命令汇总 (速查表)
```shell
# 开启接口安全并设置动作为告警
interface G0/0/1
 port-security enable
 port-security protect-action restrict

# 配置NAT Server发布内网Web
interface G0/0/1 (外网口)
 nat server protocol tcp global current-interface 80 inside 192.168.1.100 80

# 配置ACL禁止特定访问
acl 3000
 rule deny tcp source 192.168.1.1 0 destination 192.168.2.1 0 destination-port eq 80
```
