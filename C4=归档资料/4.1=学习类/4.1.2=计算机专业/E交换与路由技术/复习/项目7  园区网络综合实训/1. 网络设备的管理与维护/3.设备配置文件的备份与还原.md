这可是网络工程师保住饭碗的“免死金牌”，也是实操考试中非常经典的压轴小题！在《交换与路由技术》中，**配置文件的备份与还原**不仅是理论，更是必须掌握的硬核实操代码。

想象一下这个灾难场景：你花了一整天时间敲了几千行复杂的 OSPF、ACL 和 NAT 配置，结果设备突然断电，或者主板烧了。如果没有备份，你只能绝望地从头再敲一遍。

在华为 VRP 体系中，备份和还原主要依靠两种协议来充当“搬运工”：**FTP（文件传输协议，可靠但配置复杂）** 或 **TFTP（简单文件传输协议，极简且最常用）**。

考试中最爱考、现网中最爱用的就是 **TFTP**。咱们直接上考场满分流程！

### 🎯 考场实战场景

**背景：** 你的路由器（IP: `192.168.1.1`）配置已经全部完成。现在网管电脑（IP: `192.168.1.100`）上已经开启了 TFTP 服务端软件。

**任务：** 将路由器的配置文件备份到电脑上；假设设备损坏，再演示如何将配置还原。

---

### 第一阶段：配置文件的备份（导出）

在把文件传走之前，有一个**无数考生都会踩的“致命陷阱”**！

#### ⚠️ 第 1 步：绝对不能忘的 `save`

你在命令行里敲的所有代码，都暂时存在设备的**内存（RAM）**里，断电就丢。你必须先把它们固化到设备的**硬盘（Flash）**里，变成一个实实在在的配置文件（通常叫 `vrpcfg.zip`）。

Plaintext

```
<Huawei> save
  The current configuration will be written to the device.
  Are you sure to continue? (y/n)[n]: y    ! 输入 y 确认保存
```

_(考场避坑：如果没有先敲 `save` 就直接导出，你导出的将是设备上一次保存的旧配置，今天敲的代码全白费了！)_

#### 第 2 步：将文件推送到 TFTP 服务器 (Put)

在用户视图下（注意是 `<Huawei>` 视图，不是 `system-view`），直接调用 tftp 命令，把 Flash 里的 `vrpcfg.zip` **推（put）** 到网管电脑上。

Plaintext

```
! 语法：tftp <服务器IP> put <本地文件名>
<Huawei> tftp 192.168.1.100 put vrpcfg.zip
```

看到屏幕提示 `File transfer successfully`，备份大功告成！这台设备现在的配置已经安全地躺在你的电脑硬盘里了。

---

### 第二阶段：配置文件的还原（导入与激活）

假设有一天这台路由器主板烧了，公司买了一台全新的、空空如也的同型号路由器换上，并配好了基础的 `192.168.1.1` IP 地址能 Ping 通网管电脑。你需要让它瞬间“复活”。

#### 第 1 步：从 TFTP 服务器把文件拉回来 (Get)

让新路由器去网管电脑上，把当初备份的配置文件**拉（get）** 下来，存到自己的 Flash 硬盘里。为了区分，我们给拉下来的文件起个新名字叫 `newcfg.zip`。

Plaintext

```
! 语法：tftp <服务器IP> get <服务器上的文件名> <保存到本地的新文件名>
<Huawei> tftp 192.168.1.100 get vrpcfg.zip newcfg.zip
```

#### 第 2 步：指定下次启动的配置文件（灵魂指令）

文件拉下来了，但新路由器并不知道要用它。你必须明确指定：**“下次重启时，请吃透这个新配置！”**

Plaintext

```
<Huawei> startup saved-configuration newcfg.zip
```

#### 第 3 步：重启设备，涅槃重生

Plaintext

```
<Huawei> reboot
  Warning: The configuration has been modified, and it will be saved to the next startup saved-configuration file. Continue? (y/n)[n]: n  ! ⚠️ 这里一定要选 n！千万别保存当前空配置去覆盖新文件
  System will reboot! Continue? (y/n)[n]: y  ! 确认重启
```

设备重启完毕后，所有的 OSPF、ACL、NAT 全部瞬间恢复，完美交工！

### 💡 考前一分钟极速记忆卡

- **保存配置的命令：** `<Huawei> save`
    
- **查看当前启动文件的命令：** `display startup`
    
- **备份（往外发）：** `tftp 服务器IP put 文件名`
    
- **还原（往回拿）：** `tftp 服务器IP get 文件名`
    
- **指定启动文件：** `startup saved-configuration 文件名`
