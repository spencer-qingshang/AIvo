在华为 VRP 系统的实操大题中，配置动态 NAT（排队借用模式）的核心灵魂就在于最后的 **`no-pat`** 关键字。少了这个词，它就会变成 NAPT。

### 🎯 考场终极实战演练

**【考题背景】**

公司内部研发部网段为 `192.168.1.0/24`。公司向运营商购买了 5 个连续的公网 IP，范围是从 `200.1.1.10` 到 `200.1.1.14`。

**要求：** 在出口路由器（外网接口为 `GigabitEthernet 0/0/1`）上配置动态 NAT，允许研发部员工上网，但**明确要求不进行端口转换**。

---

### 满分配置“三步曲”

#### 第一步：建池子（定义公网 IP 范围）

你得先告诉路由器，你手里到底有哪几个合法的公网 IP 可以往外借。

Plaintext

```
<Huawei> system-view
! 1. 创建编号为 1 的 NAT 地址池，并圈定首尾 IP
[Huawei] nat address-group 1 200.1.1.10 200.1.1.14
```

#### 第二步：发通行证（用基本 ACL 圈定内网人员）

这和 Easy IP 一模一样，必须用 ACL 告诉路由器，内网里到底谁有资格去池子里借 IP。

Plaintext

```
! 2. 创建基本 ACL 2000，抓取研发部网段
[Huawei] acl 2000
[Huawei-acl-basic-2000] rule 5 permit source 192.168.1.0 0.0.0.255
[Huawei-acl-basic-2000] quit
```

#### 第三步：守大门（接口下调用并附带灵魂参数）

拿着通行证（ACL）和公网 IP 池，站到离互联网最近的那个接口上去执行转换动作。

Plaintext

```
! 3. 进入连接外网的物理接口
[Huawei] interface GigabitEthernet 0/0/1

! 4. 终极绝杀指令：绑定 ACL 和地址池，并强制附加 no-pat
[Huawei-GigabitEthernet0/0/1] nat outbound 2000 address-group 1 no-pat
[Huawei-GigabitEthernet0/0/1] quit
```

---

### 💡 考场排错与验证必背

全部敲完后，用这条终极验证命令来确认你的成绩：

Plaintext

```
[Huawei] display nat outbound
```

- **得分眼：** 只要在屏幕打印出的信息中，清晰地看到 `ACL/Traffic-filter: 2000` 对应着 `Address-group: 1`，并且后面赫然写着 **`No-pat: Y`**，这道综合配置大题的分数就彻彻底底被你收入囊中了。
