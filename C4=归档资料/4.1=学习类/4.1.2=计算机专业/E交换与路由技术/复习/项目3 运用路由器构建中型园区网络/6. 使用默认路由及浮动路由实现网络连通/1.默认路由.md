# 核心考点笔记：默认路由 (Default Route)

## 一、什么是默认路由？

**定义：** 默认路由是一种**极其特殊的静态路由**。它的目标网段和子网掩码全部是 `0`。

- **表现形式：** `0.0.0.0/0` （在华为设备配置中写为 `0.0.0.0 0`）。
    
- **通俗理解：** 它代表**“所有未知的目的地”**。就像商场里的“安全出口”指示牌——当你不知道具体的专卖店在哪时，只要顺着“安全出口”走，总能走到大马路上。
    

## 二、默认路由的工作原理 (最长掩码匹配原则的终极体现)

在上一节查路由表的铁律中，我们讲过“最长掩码匹配原则”。

- 默认路由的掩码长度是 **`/0`**，是所有路由中**最短的**。
    
- **这意味着什么？** 意味着它能匹配全宇宙所有的 IP 地址，但它的**匹配优先级是最低的**。
    
- **工作机制：** 只有当路由器查遍了路由表里所有的直连路由、精确静态路由、OSPF 动态路由，**发现全都不认识目标 IP 时**，才会无可奈何地说：“行吧，既然都不认识，那就统统扔给默认路由指的那个接口吧。”
    

## 三、绝对必考场景：企业网出口 (为什么需要默认路由？)

**场景设定：** 你们公司的出口路由器（边界路由器）连接着电信运营商（ISP），从而接入互联网。

- **痛点：** 互联网上有几十亿个公网 IP（比如百度的 IP、腾讯的 IP、各大网站的 IP）。难道你要在出口路由器上手工敲几十亿条静态路由吗？路由器内存早就撑爆了。
    
- **完美解法：** 在出口路由器上，只需要敲**唯一的一条**默认路由，下一跳指向电信运营商的路由器接口。
    
- **效果：** 公司员工访问内网服务器，路由器懂怎么走；只要是访问外网（比如聊微信、看网页），路由器查不到具体内网地址，全部通过默认路由打包扔给电信运营商：“我不知道怎么走，大哥你帮我发吧！”
    

## 四、满分配置代码 (纯简写版)

假设 R 1 是企业出口路由器，电信运营商给你的网关 IP 是 `200.1.1.2`。

Plaintext

```
<Huawei> sys
[Huawei] sysn R1

// 核心简写：配置默认路由指向外网网关
[R1] ip route-s 0.0.0.0 0 200.1.1.2   // 目的网段0.0.0.0 掩码0 下一跳IP
```

## 五、考场排错避坑指南

- **⚠️ 永远不要在内网的核心路由器之间乱指默认路由！**
    
    默认路由通常**只配置在网络的边缘（出口）**。如果在内网两台相连的路由器上互相指对方为默认路由，一旦出现一个两人都不认识的神秘 IP，数据包就会在这两台路由器之间疯狂踢皮球（形成**路由环路**），直到把网络卡死。
