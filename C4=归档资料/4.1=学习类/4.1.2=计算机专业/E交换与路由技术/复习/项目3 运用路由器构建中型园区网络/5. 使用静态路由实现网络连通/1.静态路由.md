# 核心考点笔记：静态路由 (Static Route)

## 一、什么是静态路由？

**定义：** 由网络管理员**纯手工**逐条在路由器上配置的路由条目。

**原理机制：** 就像你给路由器硬塞了一张“指路牌”——“凡是去往 `192.168.2.0` 网段的数据包，统统扔给隔壁 `10.1.1.2` 那个路口”。路由器绝对服从，不加思考。

## 二、优缺点对比 (⚠️ 简答题必考)

|**维度**|**静态路由的特点**|
|---|---|
|**优点 1：零额外开销**|不需要像动态路由（OSPF 等）那样互相发送协议报文交换地图，**完全不占用网络带宽和路由器 CPU 算力**。|
|**优点 2：绝对安全**|路线是管理员锁死的，黑客无法通过伪造路由协议来篡改路由表。|
|**缺点 1：不知变通**|**没有动态适应性**。如果链路断了或者拓扑改变，静态路由不会自动切换，会导致断网，必须靠管理员手工删改。|
|**缺点 2：维护量极大**|在中大型网络中，手工写几百条路由极易出错，仅适用于**小型或极度稳定的网络边缘**（如企业出口连向宽带运营商）。|

## 三、特殊的静态路由：默认路由 (缺省路由)

- **核心概念：** 目的地址和掩码全为 `0.0.0.0` 的特殊静态路由。它代表**“所有未知的目的地”**。
    
- **作用（最后兜底）：** 当路由器查遍了路由表里的所有精细路线都找不到匹配项时，就会把数据包统统甩给默认路由指定的方向。（通常用在企业网的出口路由器上，指向互联网）。
    

## 四、满分配置代码 (纯简写版)

**指令解析：** `ip route-s <目的网段IP> <目的掩码> <下一跳IP>`

Plaintext

```
<Huawei> sys
[Huawei] sysn R1

// 场景 1：配置普通静态路由 (去往 192.168.2.0/24 网段，下一跳交给 10.1.1.2)
[R1] ip route-s 192.168.2.0 24 10.1.1.2    // 核心简写：ip route-s 是 ip route-static 的极限简写

// 场景 2：配置默认路由 (不知道去哪的包，全扔给外网网关 200.1.1.2)
[R1] ip route-s 0.0.0.0 0 200.1.1.2

// 撤销/删除写错的静态路由 (前面加 undo 即可)
[R1] un ip route-s 192.168.2.0 24 10.1.1.2
```

## 五、验证与黄金考点

写完代码后，必须敲查看路由表的命令验证：

Plaintext

```
[R1] dis ip rou       // 全称：display ip routing-table
```

- **⚠️ 考试挖坑点：** 在这块弹出的路由表中，静态路由的 `Proto` (协议) 字段会显示为 **`Static`**，在华为体系中，静态路由的**默认优先级 (Preference) 固定为 `60`**（数字越小优先级越高）。
