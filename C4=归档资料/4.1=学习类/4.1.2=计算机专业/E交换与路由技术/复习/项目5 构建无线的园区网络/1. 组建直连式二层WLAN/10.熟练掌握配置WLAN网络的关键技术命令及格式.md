掌握 WLAN 网络的关键技术命令，最忌讳的就是死记硬背枯燥的代码。在华为 VRP 体系中，AC 的配置逻辑是高度**“模块化（模板化）”**的。

就像拼乐高积木一样，你需要先分别捏好“安全积木”（密码）、“名字积木”（SSID）、“转发积木”（VLAN 和转发模式），最后把它们拼成一个“大积木”（VAP），再安插到 AP 的天线上。

下面为你整理了实操大题和填空题中最核心的**“满分命令速查表”**（以最常考的旁挂式+直接转发为例）：

### 第一块基石：底层网络与 DHCP（在交换机/网关上配置）

这一步的目的是让 AP 拿到 IP 地址，并知道去哪里找 AC。

Plaintext

```
! 1. 全局开启 DHCP 功能（极易漏敲）
[Huawei] dhcp enable

! 2. 创建 AP 的地址池，并配置 Option 43（必考填空题）
[Huawei] ip pool AP_Pool
[Huawei-ip-pool-AP_Pool] network 192.168.10.0 mask 24
[Huawei-ip-pool-AP_Pool] gateway-list 192.168.10.254
! 👇格式极其严谨，sub-option 2 代表 IP 地址类型，后面跟 AC 的源接口 IP
[Huawei-ip-pool-AP_Pool] option 43 sub-option 2 ip-address 10.1.1.1 
[Huawei-ip-pool-AP_Pool] quit

! 3. 配置连接 AP 的交换机接口（直接转发模式下的经典陷阱）
[Huawei] interface GigabitEthernet 0/0/1
[Huawei-GigabitEthernet0/0/1] port link-type trunk
! 👇必须同时放行管理 VLAN（如10）和业务 VLAN（如20）
[Huawei-GigabitEthernet0/0/1] port trunk allow-pass vlan 10 20
! 👇将 PVID 设为管理 VLAN，这样 AP 发出的不带标签报文就会被打上 VLAN 10
[Huawei-GigabitEthernet0/0/1] port trunk pvid vlan 10
```

---

### 第二块基石：AC 基础与 AP 注册（在 AC 上配置）

这一步是让 AC 准备好迎接 AP，并做好白名单准入。

Plaintext

```
! 1. 制定建立 CAPWAP 隧道的源接口（没有这句，AP 永远上不了线）
[AC] capwap source interface vlanif 100  ! 假设 VLAN 100 是 AC 与 AP 通信的接口

! 2. 进入 WLAN 视图，配置 AP 认证
[AC] wlan
! 👇常用的认证模式有 mac-auth（MAC认证）或 no-auth（不认证）
[AC-wlan-view] ap auth-mode mac-auth

! 3. 离线录入 AP（绑定 MAC 地址并命名）
[AC-wlan-view] ap-id 1 ap-mac 00e0-fc12-3456
[AC-wlan-ap-1] ap-name Office-AP1
[AC-wlan-ap-1] quit
```

---

### 第三块基石：创建三大模板（WLAN 视图下的核心操作）

这是配置量最大、最容易乱的地方。严格按照 **安全 ➔ SSID ➔ VAP** 的顺序创建。

Plaintext

```
! 1. 创建安全模板（Security Profile）—— 负责密码
[AC-wlan-view] security-profile name Sec_Office
! 👇配置 WPA2-PSK 认证，密码为 Huawei@123，加密方式为 aes
[AC-wlan-sec-prof-Sec_Office] security wpa2 psk pass-phrase Huawei@123 aes
[AC-wlan-sec-prof-Sec_Office] quit

! 2. 创建 SSID 模板（SSID Profile）—— 负责 Wi-Fi 名字
[AC-wlan-view] ssid-profile name SSID_Office
[AC-wlan-ssid-prof-SSID_Office] ssid Company_WiFi
[AC-wlan-ssid-prof-SSID_Office] quit

! 3. 创建 VAP 模板（VAP Profile）—— 负责终极融合
[AC-wlan-view] vap-profile name VAP_Office
! 👇指定转发模式为直接转发（本地转发），若是隧道转发则改为 tunnel
[AC-wlan-vap-prof-VAP_Office] forward-mode direct-forward
! 👇指定无线用户的业务 VLAN
[AC-wlan-vap-prof-VAP_Office] service-vlan vlan-id 20
! 👇将前面的两个模板绑定进来
[AC-wlan-vap-prof-VAP_Office] security-profile Sec_Office
[AC-wlan-vap-prof-VAP_Office] ssid-profile SSID_Office
[AC-wlan-vap-prof-VAP_Office] quit
```

---

### 第四块基石：将 VAP 绑定到 AP 射频（发射信号）

最后一步，进入 AP 组（或具体的 AP），把组装好的 VAP 模板挂载到天线上。

Plaintext

```
! 进入默认的 AP 组（新建的 AP 默认都在 default 组里）
[AC-wlan-view] ap-group name default
! 👇核心长命令：将 VAP 模板绑定到射频。wlan 1 是虚拟局域网编号，radio all 代表 2.4G 和 5G 都发这个信号
[AC-wlan-ap-group-default] vap-profile VAP_Office wlan 1 radio all
[AC-wlan-ap-group-default] quit
```

### 考场必背的排错与验证命令

如果配置题要求写出验证命令，或者在模拟器上排错，下面这两条是最高频的：

- **`display ap all`**
    
    - **作用：** 在 AC 上查看所有 AP 的上线状态。
        
    - **得分眼：** 重点看 `State` 字段，必须是 **`nor`**（Normal）。如果是 `flt`（Fault），说明 AP 没拿到 IP 或路由不通；如果是 `idle`，说明 AP 还在重启或认证中。
        
- **`display vap ssid Company_WiFi`**
    
    - **作用：** 检查某个具体的 Wi-Fi 信号是否成功发射，以及它绑定了哪个业务 VLAN 和转发模式。
