在《交换与路由技术》这门课中，将 RIPv 2 和 RIPng 的配置命令进行“左右互搏”式的对比记忆，是应对实操大题和排错题最稳妥的策略。

在华为 VRP 系统中，这两者的配置逻辑有着本质的分野：**RIPv 2 是“找网段”，而 RIPng 是“绑接口”。**

以下是为您整理的配置命令级对比清单和实战代码块：

### 一、核心命令单行对比表

|**配置步骤 / 目标**|**RIPv 2 (IPv 4) 命令语法**|**RIPng (IPv 6) 命令语法**|**⚠️ 必记差异点**|
|---|---|---|---|
|**1. 全局基础开关**|无需配置，IPv 4 默认开启| `[Huawei] ipv6` |**致命易错点：** RIPng 必须先全局开启 IPv 6，否则后续命令全报错。|
|**2. 启动协议进程**| `[Huawei] rip 1` | `[Huawei] ripng 1` |进程号均为本地有效。|
|**3. 指定协议版本**| `[Huawei-rip-1] version 2` |无此命令|RIPng 天生就是新版本，无需额外指定版本号。|
|**4. 关闭自动汇总**| `[Huawei-rip-1] undo summary` |无此命令|RIPng 原生支持无类域间路由（CIDR），不需要手工关汇总。|
|**5. 宣告路由方式**| `[Huawei-rip-1] network 10.0.0.0` |进接口：`[Huawei-GigabitEthernet0/0/0] ripng 1 enable` |**核心大考点：** RIPv 2 在进程视图里敲主类网络号；RIPng 直接进物理接口激活。|
|**6. 查路由表命令**| `display ip routing-table protocol rip` | `display ipv6 routing-table protocol ripng` |关键字由 `ip` 变为了 `ipv6`。|

### 二、完整实战代码块对比

为了更直观，我们假设路由器 R 1 的 `GigabitEthernet 0/0/0` 接口用于互联，`GigabitEthernet 0/0/1` 接口连接局域网。我们分别看看在同一台设备上，两者的完整配置流有何不同。

#### ⬅️ 左手：RIPv 2 完整配置 (面向网段)

RIPv 2 的逻辑是：进入 RIP 进程后，告诉它你要宣告哪些主类网段，路由器会自动去寻找属于这些网段的物理接口。

Plaintext

```
<R1> system-view
[R1] interface GigabitEthernet 0/0/0
[R1-GigabitEthernet0/0/0] ip address 10.1.1.1 255.255.255.252
[R1-GigabitEthernet0/0/0] quit
[R1] interface GigabitEthernet 0/0/1
[R1-GigabitEthernet0/0/1] ip address 192.168.1.254 255.255.255.0
[R1-GigabitEthernet0/0/1] quit

! 开始配置 RIPv2
[R1] rip 1
[R1-rip-1] version 2                             ! 必须指定版本 2
[R1-rip-1] undo summary                          ! 强烈建议关闭自动汇总
[R1-rip-1] network 10.0.0.0                      ! 宣告互联接口所在的主类网段（A类）
[R1-rip-1] network 192.168.1.0                   ! 宣告局域网接口所在的主类网段（C类）
[R1-rip-1] quit
```

#### ➡️ 右手：RIPng 完整配置 (面向接口)

RIPng 的逻辑是：建好进程放着不管，直接一头扎进物理接口里，不仅在接口配 IPv 6 地址，还在接口直接把 RIPng 给打开。

Plaintext

```
<R1> system-view
[R1] ipv6                                        ! 第一步：全局开启 IPv6

! 启动 RIPng 进程
[R1] ripng 1                                     ! 创建进程后直接退出，里面不用配东西
[R1-ripng-1] quit

! 进入接口，边配 IP 边激活路由
[R1] interface GigabitEthernet 0/0/0
[R1-GigabitEthernet0/0/0] ipv6 enable            ! 开启接口 IPv6 能力
[R1-GigabitEthernet0/0/0] ipv6 address 2001:db8:12::1 64
[R1-GigabitEthernet0/0/0] ripng 1 enable         ! 直接把该接口拉入 RIPng 进程 1 中
[R1-GigabitEthernet0/0/0] quit

[R1] interface GigabitEthernet 0/0/1
[R1-GigabitEthernet0/0/1] ipv6 enable
[R1-GigabitEthernet0/0/1] ipv6 address 2001:db8:1::254 64
[R1-GigabitEthernet0/0/1] ripng 1 enable         ! 同理，激活局域网接口
[R1-GigabitEthernet0/0/1] quit
```

### 三、考试避坑总结

1. **写错 Network 参数：** 在写 RIPv 2 配置题时，`network` 后面绝对不能带子网掩码，且必须是**主类网络号**。比如你的接口 IP 是 `10.1.1.1/30`，宣告时只能写 `network 10.0.0.0`。
    
2. **RIPng 里乱敲 Network：** 很多考生习惯了 IPv 4，在配置 RIPng 时进入 `[R1-ripng-1]` 视图后，死活想敲 `network` 宣告 `2001::`，这会导致命令直接报错，因为华为设备的 RIPng 进程视图里根本没有这个参数。
    
3. **忘记 `ipv6 enable`：** 在物理接口敲 `ripng 1 enable` 之前，如果该接口没敲 `ipv6 enable`，协议也是跑不起来的。
