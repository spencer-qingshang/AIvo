# 项目 6：构建安全的园区网络 (自考 13833 大纲深度版)

## 考点 1：实现计算机的安全接入 (Port Security)

### 1.1 识记：MAC 地址类型与生命周期
在开启端口安全（Port Security）后，交换机处理 MAC 地址的方式分为以下三类：
*   **安全动态 MAC 地址 (Secure Dynamic MAC)**：
    *   **学习方式**：使能端口安全后，接口根据流量自动学习。存储在 **MAC 地址表 (CAM 表)** 中，**不进入配置文件**。
    *   **生存周期**：遵循老化时间；**设备重启或接口物理 Down 后立即消失**，必须重新学习。
*   **安全静态 MAC 地址 (Secure Static MAC)**：
    *   **学习方式**：管理员手动通过命令绑定指定端口、VLAN 和 MAC。保存在配置文件中。
    *   **生存周期**：**永不老化**，除非手动删除，设备重启不丢失。
*   **Sticky MAC 地址 (粘性 MAC - 核心考点)**：
    *   **核心逻辑**：动态学习与静态保存的“结合体”。自动将学习到的动态 MAC 转换为 Sticky MAC。
    *   **存储位置**：**保存配置（save）后进入配置文件**。
    *   **生存周期**：接口 Down 不丢失；**保存配置后重启不丢失**。手动删除或关闭功能时消失。

### 1.2 领会：端口安全作用与保护动作
*   **核心作用**：
    *   **抑制 MAC 地址泛洪攻击 (MAC Flooding)**：通过限制 `max-mac-num`（最大 MAC 数量），防止攻击者填满 CAM 表导致合法业务地址学习失败。
    *   **防止非授权接入**：锁定特定 MAC 到物理端口，防止私接设备。
*   **保护动作 (Violation Actions) 对比**：
    | 动作名称 | 数据转发状态 | 是否生成日志 | 计数器是否增加 | 接口物理状态 |
    | :--- | :--- | :--- | :--- | :--- |
    | **Protect (保护)** | **丢弃** | 否 | 否 | Up (正常) |
    | **Restrict (限制/必背)** | **丢弃** | **是 (SNMP Trap)** | **是** | Up (正常) |
    | **Shutdown (关闭)** | **丢弃** | **是** | **是** | **Down (Error-down)** |

### 1.3 应用：熟练配置与验证
*   **标准配置流程**：
    ```shell
    [SW1-G0/0/1] port-security enable             # 使能端口安全 (必备前提)
    [SW1-G0/0/1] port-security max-mac-num 1      # 限制最大学习数量为1
    [SW1-G0/0/1] port-security mac-address sticky # 开启粘性MAC自动转换
    [SW1-G0/0/1] port-security protect-action restrict # 设置违规动作为限制
    [SW1-G0/0/1] port-security aging-time 10      # (可选) 设置老化时间10分钟
    ```
*   **验证与排障命令**：
    *   `display port-security interface G0/0/1`：查看端口安全总体配置（Port Status, Violation Counter）。
    *   `display mac-address security`：查看安全 MAC 地址表（查看 Type 为 Sticky 的表项）。
    *   **恢复 Error-down 接口**（针对 Shutdown 模式）：在接口下执行 `restart` 或 `shutdown` / `undo shutdown`。

## 考点 2：实现网络设备的远程管理 (VTY & Management Security)

### 2.1 识记：协议基础与验证方式
*   **Telnet 与 STelnet (SSH)**：
    *   **Telnet**：基于 TCP 协议，默认端口 **23**。数据以 **明文** 形式传输，安全性极低，易被嗅探密码。
    *   **STelnet (SSH)**：基于 TCP 协议，默认端口 **22**。采用 **加密和认证** 机制，即使报文被抓取也无法解密，是现网管理的首选。
*   **用户验证模式**：
    *   **Password 验证**：只需输入全局密码即可进入设备，操作简便但无法追踪具体操作人员（不支持审计）。
    *   **AAA 验证**：需要输入 **用户名 + 密码**。支持审计（记录谁做了什么）和精细的权限授权，安全性最高。

### 2.2 领会：管理的优缺点与安全性
*   **远程管理的优缺点对比**：
    *   **优点**：高效便捷（无需跑机房）、支持集中化管控全网设备。
    *   **缺点/风险**：强依赖业务网络的通畅性；面临来自全网任何位置的非法暴力破解尝试。
*   **安全性增强手段**：
    *   **加密协议**：强制禁用 Telnet，仅允许 STelnet。
    *   **权限分级 (0-15 级)**：
        *   0-1 级：监控级，仅能执行简单查询。
        *   3 级及以上：管理级，拥有配置权限。
        *   **15 级**：最高权限级，可执行所有命令（包括删除配置）。

### 2.3 应用：实战配置与验证 (交换机/路由器通用)
*   **场景 A：配置 Telnet + Password 验证 (基础场景)**
    ```shell
    [SW1] user-interface vty 0 4             # 进入虚拟终端 (允许5人同时在线)
    [SW1-ui-vty0-4] authentication-mode password
    [SW1-ui-vty0-4] set authentication password cipher Huawei@123
    [SW1-ui-vty0-4] user privilege level 15   # 授予最高登录权限
    ```
*   **场景 B：配置 Telnet + AAA 验证 (考纲重点)**
    ```shell
    [SW1] aaa
    [SW1-aaa] local-user admin password cipher Admin@123
    [SW1-aaa] local-user admin service-type telnet    # 授权Telnet
    [SW1-aaa] local-user admin privilege level 15     # 授权最高等级
    [SW1] user-interface vty 0 4
    [SW1-ui-vty0-4] authentication-mode aaa           # 切换验证模式为AAA
    ```
*   **场景 C：配置 SSH (STelnet) 加密管理 (企业规范)**
    ```shell
    [SW1] rsa local-key-pair create           # 生成RSA密钥对 (SSH必须前提)
    [SW1] stelnet server enable               # 开启STelnet服务
    [SW1-aaa] local-user suser password cipher SSH@123
    [SW1-aaa] local-user suser service-type ssh       # 授权SSH服务
    [SW1-aaa] local-user suser privilege level 15
    [SW1] user-interface vty 0 4
    [SW1-ui-vty0-4] authentication-mode aaa
    [SW1-ui-vty0-4] protocol inbound ssh              # 仅允许SSH，禁止Telnet
    ```

## 考点 3：使用基本 ACL 限制网络访问 (Basic ACL)

### 3.1 识记：ACL 规则原理与分类
*   **ACL 的分类与编号**：
    *   **基本 ACL**：编号范围 **2000-2999**。仅能根据报文的 **源 IP 地址** 信息制定规则。
    *   **高级 ACL**：编号范围 **3000-3999**。可根据源/目 IP、协议、端口等精细信息制定规则。
*   **ACL 规则匹配顺序**：
    *   遵循“**从小到大，命中即止**”原则。
    *   报文从 ID 最小的 rule 开始匹配，一旦匹配成功（命中），则执行该 rule 定义的动作（Permit/Deny），不再检查后续规则。
*   **步长 (Step)**：默认规则编号间隔为 5（如 rule 5, 10, 15），方便在现有规则间插入新规则。

### 3.2 领会：ACL 作用与通配符逻辑
*   **ACL 的主要作用**：
    *   **流量过滤**：拒绝（Deny）或允许（Permit）特定数据包通过。
    *   **流量标识**：配合 NAT、QoS 等技术“抓取”感兴趣流。
*   **通配符掩码 (Wildcard Mask) 逻辑**：
    *   **0**：代表“必须匹配”（严格检查该位）。
    *   **1**：代表“忽略”（不检查该位）。
    *   *示例*：`192.168.1.0 0.0.0.255` 表示匹配 1.0 网段的所有主机。
*   **应用位置原则**：
    *   **基本 ACL 应尽量应用在靠近目的地 (Destination) 的接口上**。
    *   原因：基本 ACL 仅看源 IP，若在源端过滤，会误伤该源访问其他正常目的地的流量。

### 3.3 应用：实战配置基本 ACL
*   **配置需求**：禁止 192.168.1.0 网段访问外网，但允许其他流量。
    ```shell
    [R1] acl 2000                                # 创建基本ACL
    [R1-acl-basic-2000] rule deny source 192.168.1.0 0.0.0.255 # 拒绝特定源
    [R1-acl-basic-2000] rule permit source any   # 显性放行其他所有
    [R1] interface GigabitEthernet 0/0/1         # 进入靠近目的地的接口
    [R1-GigabitEthernet0/0/1] traffic-filter outbound acl 2000 # 在出方向应用
    ```
*   **验证命令**：
    *   `display acl 2000`：查看规则内容及匹配次数（matches）。
    *   `display traffic-filter applied-record`：查看 ACL 应用的接口和方向。

## 考点 4：使用高级 ACL 限制服务器端口防攻击 (Advanced ACL)

### 4.1 识记：高级 ACL 与端口概念
*   **高级 ACL 编号范围**：**3000-3999**。
*   **IP 协议端口 (Logical Port) vs 物理端口 (Physical Port)**：
    *   **逻辑端口**：存在于传输层协议头中，用于区分服务。如 **HTTP(80)、HTTPS(443)、FTP(21)**。
    *   **物理端口**：设备上的硬件接口（如 G0/0/1）。ACL 规则应用在物理端口上，用以过滤逻辑端口的数据流。

### 4.2 领会：高级 ACL 规则原理与对比
*   **高级 ACL 的主要作用**：实现精细化流量控制，精确到具体的协议和服务端口。
*   **五元组匹配原则**：源 IP、目的 IP、协议号、源端口、目的端口。
*   **高级 ACL 与基本 ACL 的深度对比**：
    | 特性 | 基本 ACL (2000-2999) | 高级 ACL (3000-3999) |
    | :--- | :--- | :--- |
    | **检查字段** | 仅源 IP 地址 | **五元组** (源/目IP, 协议, 源/目端口) |
    | **精细度** | 粗略（针对网段/主机） | 极精细（针对 Web/FTP 等服务） |
    | **应用位置** | **靠近目的地** | **靠近源端** (尽早拦截以节省带宽) |

### 4.3 应用：熟练配置高级 ACL
*   **配置需求**：禁止 PC (192.168.1.10) 访问服务器 (172.16.1.100) 的 Web 服务 (端口 80)，但允许其他访问。
    ```shell
    [R1] acl 3000                                # 创建高级ACL
    [R1-acl-adv-3000] rule deny tcp source 192.168.1.10 0 destination 172.16.1.100 0 destination-port eq 80
    [R1-acl-adv-3000] rule permit ip source any destination any # 显性放行
    [R1] interface GigabitEthernet 0/0/0         # 进入靠近源端的接口
    [R1-GigabitEthernet0/0/0] traffic-filter inbound acl 3000 # 在入方向应用
    ```
*   **两种 ACL 的区分应用**：
    *   若仅需阻断两个网段间的通信：使用 **基本 ACL**。
    *   若需针对特定应用权限（如只给看网页不给下文件）：必须使用 **高级 ACL**。

## 考点 5：实现园区网络安全接入互联网 (PPP 认证安全)

### 5.1 识记：PPP 基础与两种认证协议
*   **PPP (Point-to-Point Protocol)**：点对点协议。广域网主流链路层协议，由 LCP (链路控制) 和 NCP (网络控制) 组成，支持身份认证。
*   **PAP (Password Authentication Protocol)**：
    *   **两次握手**。被验证方主动发送用户名和密码。
    *   **明文传输**。安全性低。
*   **CHAP (Challenge Handshake Authentication Protocol)**：
    *   **三次握手**。验证方发起挑战（Challenge），被验证方返回哈希值。
    *   **密文校验**。密码原文不经过链路，安全性高。

### 5.2 领会：PAP 与 CHAP 的作用与区别
*   **核心区别对比**：
    | 特性 | PAP | CHAP |
    | :--- | :--- | :--- |
    | **认证逻辑** | 两次握手，简单高效 | 三次握手，复杂且安全 |
    | **密码安全性** | **明文传输**，易被抓包 | **哈希校验**，密码不传链路 |
    | **控制权** | 被验证方掌握发起时机 | **主验证方**控制认证频率 |
*   **应用场合判断**：
    *   **PAP 场合**：对设备性能要求极低、或在绝对安全的物理私网环境下使用。
    *   **CHAP 场合**：**互联网接入 (PPPoE)**、对安全性有明确要求的企业分支互联。

### 5.3 应用：熟练配置 PPP 认证
*   **CHAP 配置 (主验方)**：
    ```shell
    [R1] aaa
    [R1-aaa] local-user huawei password cipher Huawei@123
    [R1-aaa] local-user huawei service-type ppp
    [R1] interface Serial 1/0/0
    [R1-Serial1/0/0] ppp authentication-mode chap
    ```
*   **CHAP 配置 (被验方)**：
    ```shell
    [R2] interface Serial 1/0/0
    [R2-Serial1/0/0] ppp chap user huawei
    [R2-Serial1/0/0] ppp chap password cipher Huawei@123
    ```
*   **PAP 关键命令 (被验方)**：
    `[R2-S1/0/0] ppp pap local-user huawei password cipher Huawei@123`

## 考点 6：利用静态 NAT 实现内网服务器向互联网发布信息 (Static NAT)

### 6.1 NAT 的分类与作用
*   **NAT (Network Address Translation)**：网络地址转换。通过修改报文头部的 IP 地址信息，实现私网与公有网络互通。
*   **主要作用与优点**：
    *   **节省 IP 资源**：缓解 IPv4 公网地址枯竭。
    *   **增强安全性**：**隐藏内部网络拓扑**，外网无法直接探测内网 IP。
*   **NAT 的核心类型**：
    *   **静态 NAT**：1对1映射。常用语发布内网服务器。
    *   **动态 NAT**：多对多映射。利用地址池随机分配公网 IP（1对1，不涉及端口）。
    *   **NAPT (端口复用)**：多对1映射。利用“IP+端口号”实现多个内网用户共用一个公网 IP。

### 6.2 静态 NAT (NAT Server) 工作原理与配置
*   **静态 NAT 特点**：1对1固定映射，地址转换双向对称，永不老化。
*   **实战配置 (NAT Server)**：
    ```shell
    # 将内网Web服务器(192.168.1.100)的80端口发布到外网出接口IP
    [R1-G0/0/1] nat server protocol tcp global current-interface 80 inside 192.168.1.100 80
    ```

## 考点 7：利用动态 NAPT 实现局域网主机访问互联网 (Dynamic NAT/NAPT)

### 7.1 识记：动态 NAT 与 NAPT 的区别
*   **动态 NAT**：从公网地址池中随机抽取 IP 分配给内网主机。**转换比例为 1:1**，若地址池 IP 耗尽，新用户无法上网。
*   **NAPT (Network Address Port Translation)**：端口复用技术。
    *   **转换比例为 N:1**。
    *   **核心逻辑**：利用“IP+端口号”唯一标识会话，允许多个内网私有地址映射到同一个公有 IP 的不同端口。

### 7.2 领会：动态 NAT 的原理与优点
*   **作用**：主要用于大规模内网用户同时访问互联网。
*   **优点**：极高的地址利用率；自动化的地址转换表项管理；极佳的用户隐私保护。
*   **工作过程**：
    1.  流量进入路由器，匹配 **ACL**（感兴趣流）。
    2.  从预设的 **Address Group** (地址池) 中抓取可用 IP。
    3.  建立转换表项（私网IP:端口 <-> 公网IP:端口）。
    4.  回程数据根据映射表逆向转换。

### 7.3 应用：实战配置动态 NAT (带地址池)
*   **配置流程**：
    ```shell
    [R1] acl 2000                                # 1. 定义允许上网的源网段
    [R1-acl-basic-2000] rule permit source 192.168.1.0 0.0.0.255
    [R1] nat address-group 0 202.1.1.10 202.1.1.11 # 2. 创建公网地址池
    [R1] interface GigabitEthernet 0/0/1         # 3. 在外网接口关联
    [R1-GigabitEthernet0/0/1] nat outbound 2000 address-group 0
    ```
*   **验证命令**：`display nat session all` (查看当前 NAT 转换流)。

## 考点 8：使用防火墙隐藏内部网络地址保护内网安全 (Firewall Security)

### 8.1 识记：防火墙关键技术与区域模式
*   **关键技术**：
    *   **包过滤**：基础过滤，不检查状态。
    *   **状态检测 (核心)**：通过 **会话表 (Session Table)** 追踪通信流，只有匹配会话的后续报文才允许通过。
*   **安全区域 (Zone) 与优先级 (必背)**：
    *   **Local (100)**：防火墙自身。
    *   **Trust (85)**：受信区，通常指内网。
    *   **DMZ (50)**：服务器存放区（非军事化区）。
    *   **Untrust (5)**：非受信区，通常指互联网。
*   **工作模式**：**路由模式**（三层转发）与 **透明模式**（二层接入）。

### 8.2 领会：防火墙原理与安全策略逻辑
*   **工作原理**：状态检测防火墙仅对流的“首包”进行安全策略检查，后续报文根据首包建立的会话表直接转发，实现了高效的双向安全控制。
*   **安全策略 (Security Policy)**：
    *   **默认拒绝**：防火墙默认拒绝所有跨区域流量。
    *   **匹配逻辑**：基于源/目区域、源/目IP、服务、用户等多维度匹配，匹配即执行 Permit/Deny。
*   **核心功能**：区域隔离、访问控制、NAT 地址隐藏、攻击防护。

### 8.3 应用：正确配置防火墙安全
*   **配置规划流程**：
    1.  **接口入区**：将物理接口划分到相应的安全区域。
    2.  **配置策略**：根据业务流向编写安全策略。
    3.  **配置 NAT**：在出口方向配置 NAT 策略以隐藏内网 IP。
*   **配置示例 (允许内网访问外网并开启 NAT)**：
    ```shell
    # 1. 划分区域
    [FW] firewall zone trust
    [FW-zone-trust] add interface G1/0/1
    [FW] firewall zone untrust
    [FW-zone-untrust] add interface G1/0/2

    # 2. 安全策略
    [FW] security-policy
    [FW-policy-security] rule name internet_access
    [FW-policy-security-rule-internet_access] source-zone trust
    [FW-policy-security-rule-internet_access] destination-zone untrust
    [FW-policy-security-rule-internet_access] action permit

    # 3. NAT策略 (Easy-IP)
    [FW] nat-policy
    [FW-policy-nat] rule name internal_hide
    [FW-policy-nat-rule-internal_hide] source-zone trust
    [FW-policy-nat-rule-internal_hide] destination-zone untrust
    [FW-policy-nat-rule-internal_hide] action source-nat easy-ip
    ```

## 考前必背核心口诀 (项目 6 全方位总结)
1.  **Sticky MAC 重启不丢**：限制终端接入首选。
2.  **Restrict 告警又丢包**：最常用惩罚模式。
3.  **SSH 加密 22 端口**：远程管理必选。
4.  **基本 ACL 2000 靠目端**：只看源 IP。
5.  **高级 ACL 3000 靠源端**：五元组精细过滤。
6.  **CHAP 三次密文握手**：广域网认证最安全。
7.  **NAT Server 发布服务器**：静态端口映射。
8.  **防火墙 Trust 优先级 85**：默认拒绝跨区流。
