## 一、 单项选择题 (每小题 2 分，共 40 分)

1. **答案：B**
   **解析：** 端口安全的三种动作为：Protect（丢弃包）、Restrict（丢弃包并产生告警/计数）、Shutdown（直接关闭接口并显示 Error-down）。题目要求产生告警并计数，因此选 B。

2. **答案：C**
   **解析：** 粘性 MAC 地址（Sticky MAC）会将动态学习到的 MAC 转换为非易失性条目，save 后重启不丢失。它不属于静态手动配置，也不遵循自动老化时间。

3. **答案：C**
   **解析：** Shutdown 动作会导致接口变为 Error-down 状态，需要管理员手动执行 undo shutdown 或重启接口（符合逻辑自动恢复）才能恢复。

4. **答案：D**
   **解析：** AAA（认证、授权、计费）模式可以为每个用户设置独立账号密码，并利用“计费”功能实现对操作行为的审计追踪。

5. **答案：D**
   **解析：** 华为设备权限级别中，0 级为参观级，3-14 级为管理级，15 级为最高等级（管理级），拥有所有命令的控制权。

6. **答案：A**
   **解析：** 基本 ACL（2000-2999）仅支持根据源 IP 地址进行报文匹配，无法识别目的 IP、协议或端口。

7. **答案：C**
   **解析：** 基本 ACL 是 2000-2999，高级 ACL 是 3000-3999，二层 ACL 是 4000-4999。

8. **答案：D**
   **解析：** 通配符掩码中 0 表示必须匹配，1 表示忽略。`0.0.0.0` 表示 32 位全部必须匹配，即精确匹配一个特定的 IP 地址。

9. **答案：A**
   **解析：** 高级 ACL 功能强大（可匹配五元组），建议应用在靠近源端的地方，目的是在流量进入核心网前尽早拦截，避免浪费链路带宽。

10. **答案：C**
    **解析：** 华为防火墙区域优先级：Local(100)、Trust(85)、DMZ(50)、Untrust(5)。

11. **答案：A**
    **解析：** Local 区域代表防火墙设备本身，凡是防火墙主动发起的流量或发往防火墙自身的流量（如 Ping 防火墙、SSH 登录防火墙）都属于该区域。

12. **答案：D**
    **解析：** 会话表记录的是三层和四层信息（源/目 IP、源/目端口、协议），而不是二层 MAC 信息。

13. **答案：A**
    **解析：** NAT 的初衷是为了让多个内网私有地址共享少量公网地址，从而解决了 IPv4 公网地址严重枯竭的问题。

14. **答案：C**
    **解析：** Easy-IP 是一种特殊的 NAPT，它直接利用出口公网接口当前的 IP 地址进行转换，不需要配置额外的地址池，非常适合拨号或动态 IP 环境。

15. **答案：B**
    **解析：** NAT Server 用于“外访内”场景，通过端口映射将内网服务器的服务发布到公网。

16. **答案：B**
    **解析：** PAP 是两次握手、明文传输；CHAP 是三次握手、密文（哈希值）交互，安全性更高。

17. **答案：C**
    **解析：** PAP 直接在链路上传送密码明文，容易被截获；CHAP 传送的是基于随机值计算出的 MD5 哈希结果，密码不在链路上传输。

18. **答案：A**
    **解析：** TCP 端口 21 是 FTP 服务的控制端口（20 是数据端口）。

19. **答案：A**
    **解析：** SSH 依赖非对称加密算法，因此在开启 SSH 服务前，必须先生成 RSA 或 DSA 密钥对。

20. **答案：B**
    **解析：** ACL 匹配遵循“一经命中即停止”原则。规则 5 明确拒绝了 10.1.1.1，匹配成功，报文被丢弃，不再执行后续规则 10。

## 二、 填空题 (每小题 2 分，共 20 分)

1. **答案：Restrict**
   **解析：** 该动作与 Protect 的核心区别在于是否产生 Log 记录和违规计数。
2. **答案：Sticky**
   **解析：** “粘性”二字体现了其固化 MAC 地址的功能。
3. **答案：目的端口**
   **解析：** 五元组包括：源/目 IP、源/目端口、协议号。
4. **答案：15**
   **解析：** 华为设备的最高权限级别。
5. **答案：隐藏**
   **解析：** 外部用户只能看到 NAT 转换后的公网 IP，无法获知内网的真实私有拓扑，起到了屏障作用。
6. **答案：5**
   **解析：** Untrust（非信任区）是防火墙中默认优先级最低的区域。
7. **答案：PAP**
   **解析：** PAP (Password Authentication Protocol) 认证过程不加密。
8. **答案：eq**
   **解析：** `eq` 代表 equal，即端口号等于某个值。
9. **答案：命中即止**

10. **答案：NAT Server**
    **解析：** 也就是俗称的“端口映射”。

## 三、 简答题 (每小题 4 分，共 20 分)

1. **原则：基本 ACL 靠近目的端，高级 ACL 靠近源端。**
   **解析：** 基本 ACL 过滤颗粒度粗（仅看源 IP），若放在源端会导致“误伤”，即用户无法访问所有目的地。高级 ACL 颗粒度细，放在源端可以尽早丢弃非法报文，减少骨干网络压力。

2. **区别：Restrict 与 Protect 共同点是均丢弃包且端口不 Down。不同点在于 Restrict 会生成告警日志（Log）并触发违规计数增加，而 Protect 是静默丢弃。**

3. **原理：首包机制。防火墙只对流量的第一个报文（首包）进行复杂的安全策略检查。一旦通过，则生成会话表项。后续所有属于该流的报文只需匹配会话表即可直接快速转发，大幅提升效率。**

4. **握手流程：PAP 是 2 次握手，CHAP 是 3 次握手。安全性：PAP 明文传密，易被抓包破解；CHAP 通过 Challange 挑战值和哈希值计算验证，密码不在线路上流动，更安全。**

5. **区别：NAT Server 用于将内网服务器发布给外网访问（外对内）；Easy-IP 用于内网多主机共享出口 IP 访问互联网（内对外）。**

## 四、 综合应用题 (每小题 10 分，共 20 分)

1. **(1) tcp | (2) current-interface | (3) 80**
   **解析：** 
   - Web 服务通常基于 TCP 协议。
   - `global current-interface` 表示使用当前接口的公网 IP 映射。
   - 内部和外部对应的 Web 端口均为 80。

2. **(4) deny | (5) 0.0.0.255 | (6) eq | (7) inbound | (8) 3001**
   **解析：** 
   - 需求是“禁止”，故动作选 `deny`。
   - 掩码 24 位对应通配符 `0.0.0.255`。
   - 匹配特定端口 23 使用 `eq`。
   - ACL 建议应用在靠近源端（研发部）接口的进入（`inbound`）方向。
