在 PPP 的身份认证中，**PAP（密码认证协议，Password Authentication Protocol）**是最简单、最基础的一种。

### 一、考场核心理论：PAP 的“两次握手”

PAP 的工作逻辑极其简单粗暴，就是纯粹的“你问我答”：

1. **被认证方（客户端）：** 主动把自己的**用户名和密码（明文形式）**打包，发给主认证方。
    
2. **主认证方（服务端）：** 收到后，拿着在自己本地 AAA 数据库里查对。对上了就回发 `Authenticate-Ack`（允许接入），对不上就发 `Authenticate-Nak`（拒绝接入）。
    

**致命考点：** PAP 的密码在广域网链路上是**完全明文（Plaintext）传输**的。黑客只要在中间抓个包，你的密码就底裤全掉了，安全性极差。

---

### 二、实战满分配置流（华为 VRP）

在广域网大题中，配置 PPP 认证必定涉及两台路由器：一个是**主认证方（当保安查别人）**，一个是**被认证方（当访客被查）**。

假设 R 1 是主认证方（总部），R 2 是被认证方（分部）。两边通过 `Serial 4/0/0` 串口相连。

#### 第一步：配置主认证方（R 1 - 设关卡）

Plaintext

```
<R1> system-view
! 1. 进入 AAA 建立合法名单
[R1] aaa
! 创建专门用于 PPP 接入的账号，必须指定服务类型为 ppp
[R1-aaa] local-user admin password cipher Huawei@123
[R1-aaa] local-user admin service-type ppp
[R1-aaa] quit

! 2. 进入串口，开启 PAP 认证大门
[R1] interface Serial 4/0/0
[R1-Serial4/0/0] link-protocol ppp                   ! 确保链路协议是 PPP（华为串口默认就是）
[R1-Serial4/0/0] ip address 10.1.1.1 24              ! 配置接口 IP
[R1-Serial4/0/0] ppp authentication-mode pap         ! 核心指令：要求对方使用 PAP 认证
[R1-Serial4/0/0] quit
```

#### 第二步：配置被认证方（R 2 - 交通行证）

被认证方不需要进 AAA 建账号，它只需要把账号密码“递过去”就行。

Plaintext

```
<R2> system-view
[R2] interface Serial 4/0/0
[R2-Serial4/0/0] link-protocol ppp
[R2-Serial4/0/0] ip address 10.1.1.2 24
! 核心指令：主动向对方发送 PAP 的用户名和密码
[R2-Serial4/0/0] ppp pap local-user admin password cipher Huawei@123
[R2-Serial4/0/0] quit
```

_(💡 考场避坑：R 2 这里的 `cipher` 只是让密码在 R 2 自己的本地配置文件里加密显示，防止别人偷看屏幕，**它在网线上往 R 1 传的时候，依然是毫无防备的明文！**)_

### 三、考场排错终极绝招

配置完之后，如果发现两边 Ping 不通，立刻在两台路由器上分别敲下这条命令：

Plaintext

```
[R1] display interface Serial 4/0/0
```

- **找得分眼 1：** 看物理状态 `Line protocol current state:`。如果是 `UP`，说明认证成功了；如果是 `DOWN`，说明密码敲错了或者方向搞反了。
    
- **找得分眼 2：** 看 LCP 和 NCP 状态。如果显示 `LCP opened, IPCP opened`，说明不仅底层的 PPP 隧道通了，上层的 IP 业务也彻底打通了，满分到手！
