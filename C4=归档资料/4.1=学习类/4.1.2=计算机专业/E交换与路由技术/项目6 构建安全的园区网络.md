# 项目 6：构建安全的园区网络 (深度备考版)

## 一、 接入安全：端口安全 (必考识记)

### 1.1 核心概念
*   **端口安全的作用**：通过限制接口学习到的 **MAC 地址数量**，防止黑客进行 MAC 地址溢出攻击（如交换机 CAM 表攻击）。
*   **动态 MAC (Dynamic)**：设备接入后自动学习，接口重启或老化后会丢失。
*   **Sticky MAC (粘性 MAC - 考纲重点)**：
    *   **特点**：将动态学习到的 MAC 地址转化为“粘性”地址。
    *   **优势**：保存配置后 **重启不丢失**。无需手动输入海量 MAC，又能固定设备接入，安全性与维护性的平衡。

### 1.2 保护动作 (Violation Mode - 必背)
当接入设备超过 `max-mac-num` 限制或 MAC 不匹配时：
1.  **Protect**：丢弃违规数据包，不发送告警，不记录次数（静默丢弃）。
2.  **Restrict (考纲应用重点)**：丢弃违规数据包，**发送 SNMP Trap 告警日志**，并记录违规次数。
3.  **Shutdown**：直接将物理接口 **Error-down** 关闭。需管理员手动 `undo shutdown` 或配置自动恢复。

### 1.3 核心配置命令
```shell
[SW-G0/0/1] port-security enable             # 开启端口安全功能
[SW-G0/0/1] port-security max-mac-num 2      # 限制接口下最多学习2个MAC
[SW-G0/0/1] port-security mac-address sticky # 开启粘性MAC
[SW-G0/0/1] port-security protect-action restrict # 设置惩罚动作为restrict
```

## 二、 远程管理安全 (Telnet 与 AAA)

### 2.1 验证方式对比
*   **Password 验证**：只需输入一个全局密码。简单但不安全，无法追溯具体是哪个管理员的操作。
*   **AAA 验证 (推荐)**：需要输入用户名 + 密码。支持权限分级（0-15级），审计清晰，安全性高。

### 2.2 核心配置命令 (基于 AAA)
```shell
# 1. 配置 AAA 局部用户
[R1] aaa
[R1-aaa] local-user admin password cipher Huawei@123
[R1-aaa] local-user admin service-type telnet    # 授权Telnet登录
[R1-aaa] local-user admin privilege level 15     # 授权最高管理级别

# 2. 在 VTY 线路应用
[R1] user-interface vty 0 4                     # 进入虚拟终端线路
[R1-ui-vty0-4] authentication-mode aaa           # 切换验证模式为AAA
```

## 三、 ACL 访问控制列表 (重难点 - 必须精通)

### 3.1 ACL 匹配规则与通配符
*   **匹配规则**：**从小到大，命中即止**。一旦匹配到某条 rule，后续规则不再检查。如果所有规则都不匹配，则根据协议不同可能有“默认允许”或“默认拒绝”。
*   **通配符 (Wildcard Mask)**：
    *   `0` 表示必须匹配（严格一致）；`1` 表示忽略。
    *   `0.0.0.255` 表示匹配前 24 位，即整个网段。

### 3.2 基本 ACL vs 高级 ACL
| 特性 | 基本 ACL (2000-2999) | 高级 ACL (3000-3999) |
| :--- | :--- | :--- |
| **匹配依据** | **仅源 IP 地址** | 源/目的 IP、**协议号、TCP/UDP 端口号** |
| **颗粒度** | 粗糙，只能控制“谁”能去哪 | 精细，可控制“谁”在“什么时候”通过“什么协议”干“什么事” |
| **应用位置** | **靠近目的地** | **靠近源头** (减少带宽浪费) |

### 3.3 高级 ACL 典型配置命令
**任务**：禁止 192.168.1.0 网段的 PC 访问 192.168.2.100 服务器的 HTTP (80) 服务。
```shell
[R1] acl 3005
[R1-acl-adv-3005] rule deny tcp source 192.168.1.0 0.0.0.255 destination 192.168.2.100 0 destination-port eq 80
[R1] interface GigabitEthernet 0/0/0
[R1-G0/0/0] traffic-filter inbound acl 3005      # 入方向应用过滤
```

## 四、 广域网安全与 NAT 技术 (核心应用)

### 4.1 PPP 认证：PAP vs CHAP
*   **PAP (二次握手)**：**明文**发送密码。
*   **CHAP (三次握手 - 必背)**：**密文**挑战。密码不直接在链路上跑，防止截获。

### 4.2 NAT (网络地址转换) 核心类型
1.  **静态 NAT (NAT Server)**：1对1映射。常用于**内网服务器发布**。
    *   `nat server protocol tcp global [外网IP] 80 inside [内网IP] 80`
2.  **Easy-IP (NAPT)**：多对1映射。内网用户共用路由器出接口 IP 上网。
    *   配置：`acl 2000` 定义允许转换的网段 -> 出口接口敲 `nat outbound 2000`。

## 五、 防火墙基础 (安全区域)

### 5.1 安全区域 (Zone) 优先级
*   **Local (100)**：防火墙自己。
*   **Trust (85)**：内部信任网络。
*   **DMZ (50)**：服务器非军事区（隔离区）。
*   **Untrust (5)**：外部非信任网络（互联网）。
*   **策略原则**：**流量默认拒绝**。必须手动编写安全策略放行特定源到特定目的的流量。

---

## 六、 考纲重点提炼：哪些必须背？

1.  **ACL 分类编号**：2000-2999 (基本)；3000-3999 (高级)。
2.  **ACL 通配符**：0 表示匹配，1 表示不看。
3.  **NAT 优点**：节省公网 IP，隐藏内部拓扑。
4.  **端口安全模式**：Restrict 既丢包又告警，是最均衡的选择。
5.  **远程管理**：SSH (STelnet) 优于 Telnet，因为 SSH 是加密传输。
6.  **防火墙区域**：掌握不同区域的默认优先级，理解“高到低”与“低到高”的不同策略需求。
