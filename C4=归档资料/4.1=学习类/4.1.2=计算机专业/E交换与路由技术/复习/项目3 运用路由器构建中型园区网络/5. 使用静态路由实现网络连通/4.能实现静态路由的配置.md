# 核心考点笔记：静态路由满分配置模板 (纯简写版)

**实验经典拓扑设定：**

- **R 1：** 左连 PC 1 (`192.168.1.0/24` 网段)，右连 R 2 (`10.1.1.0/24` 网段，R 1 接口 IP 为 `.1`)。
    
- **R 2：** 左连 R 1 (`10.1.1.0/24` 网段，R 2 接口 IP 为 `.2`)，右连 PC 2 (`192.168.2.0/24` 网段)。
    
- **目标：** 让 PC 1 能成功 ping 通 PC 2。
    

---

## 第一步：R 1 配置 (接口 IP + 去程路由)

Plaintext

```
<Huawei> sys
[Huawei] sysn R1

// 1. 配置物理接口 IP
[R1] int g0/0/0
[R1-GigabitEthernet0/0/0] ip add 192.168.1.254 24   // 连接 PC1 的局域网网关
[R1-GigabitEthernet0/0/0] int g0/0/1
[R1-GigabitEthernet0/0/1] ip add 10.1.1.1 24        // 连接 R2 的广域网接口
[R1-GigabitEthernet0/0/1] q

// 2. 核心简写：配置静态路由 (告诉 R1 去 2.0 网段怎么走)
[R1] ip route-s 192.168.2.0 24 10.1.1.2   // 目的网段 掩码 下一跳(R2的接口IP)
```

---

## 第二步：R 2 配置 (接口 IP + 回程路由)

**⚠️ 避坑：** 静态路由是单向的，R 1 知道怎么去，R 2 必须知道怎么回！

Plaintext

```
<Huawei> sys
[Huawei] sysn R2

// 1. 配置物理接口 IP
[R2] int g0/0/0
[R2-GigabitEthernet0/0/0] ip add 10.1.1.2 24        // 连接 R1 的广域网接口
[R2-GigabitEthernet0/0/0] int g0/0/1
[R2-GigabitEthernet0/0/1] ip add 192.168.2.254 24   // 连接 PC2 的局域网网关
[R2-GigabitEthernet0/0/1] q

// 2. 核心简写：配置回程静态路由 (告诉 R2 去 1.0 网段怎么走)
[R2] ip route-s 192.168.1.0 24 10.1.1.1   // 目的网段 掩码 下一跳(R1的接口IP)
```

---

## 第三步：必杀验证与排错命令

1. **查路由表是否生效：**
    
    Plaintext
    
    ```
    [R1] dis ip rou
    ```
    
    _👉 检查：看路由表最下方，是否有一条 `Proto` 为 `Static`，目标为 `192.168.2.0/24` 的记录。_
    
2. **模拟数据包连通性测试：**
    
    如果不方便切到 PC 界面，直接在 R 1 上带源地址 Ping R 2 后的网络：
    
    Plaintext
    
    ```
    [R1] ping -a 192.168.1.254 192.168.2.254
    ```
    
    _👉 含义：从我的 1.254 接口发包，去 ping 你的 2.254 接口。如果通了（显示 Reply），说明静态路由配置完美！_
