VRP 的命令行界面（CLI）是层级化的，逻辑非常严密。熟练掌握这些基础命令的逻辑，对于应对 4 月份《交换与路由技术》考试中的配置大题和排错题至关重要。

在 VRP 系统中，你所处的“视图”（View）决定了你能执行哪些命令。这就好比你在公司里的权限：在员工大厅只能看公告，进了机房才能拔插网线。

以下是搭建任何网络拓扑都必须掌握的四大核心操作模块：

### 1. 核心逻辑：视图的切换

这是 VRP 最基础的规则。注意观察命令行提示符的括号形状：

- **用户视图 `<Huawei>`**：
    
    - 设备刚启动或刚登录时所在的视图。
        
    - **权限**：只能执行基本的查询和测试命令（如 `ping`、`telnet`），不能修改系统配置。
        
- **系统视图 `[Huawei]`**：
    
    - 进入方法：在用户视图下输入 `system-view`。
        
    - **权限**：可以配置系统全局参数（如修改设备名称），并且是进入其他所有高级视图的“中转站”。
        
- **接口视图 `[Huawei-GigabitEthernet0/0/1]`**：
    
    - 进入方法：在系统视图下输入具体的接口编号，例如 `interface GigabitEthernet 0/0/1`（或者简写为 `int g0/0/1`）。
        
    - **权限**：专门用来配置该物理接口的参数，比如分配 IP 地址。
        
- **退回上一级与直接回退**：
    
    - `quit`（简写为 `q`）：退回上一级视图。
        
    - `return`（快捷键 `Ctrl+Z`）：无论在哪个深层视图，直接退回到用户视图 `<Huawei>`。
        

### 2. 基础配置与“反悔”机制

在模拟器做实验时，最常用的几条配置命令：

- **修改设备名称**：
    
    - 命令：`sysname R1` （必须在 `[ ]` 系统视图下操作）
        
    - 效果：提示符会变成 `[R1]`。在多台设备的拓扑中，第一步改名可以防止配置串台。
        
- **配置 IP 地址**：
    
    - 命令：`ip address 192.168.1.1 24` （必须在接口视图下操作，`24` 是子网掩码 `255.255.255.0` 的简写）。
        
- **撤销配置（undo 命令）**：
    
    - VRP 系统非常人性化，如果你配置错了，不需要到处找删除按钮。只需在原命令前加 `undo` 即可。
        
    - 例如，配错了 IP 地址，直接在当前接口下输入：`undo ip address`。
        

### 3. 查看与保存（排错必备）

配置完之后，必须通过 `display` 命令验证是否生效，这是排错的核心手段：

- **查看接口 IP 简表**：`display ip interface brief`（简写为 `dis ip int b`）。这能一眼看出哪个接口配了什么 IP，以及接口物理状态（Physical）和协议状态（Protocol）是否都为 UP。
    
- **查看当前运行配置**：`display current-configuration`（简写为 `dis cu`）。可以查看你对这台设备敲过的所有生效命令。
    
- **保存配置**：`save`。
    
    - **极易错点**：`save` 命令**必须且只能**在用户视图 `< >` 下执行。如果在真实考场或机房配置完不保存就重启，所有配置都会清空。
        

### 4. 提升效率的“神仙”快捷键

在 eNSP 中，你完全不需要死记硬背完整的单词：

- **Tab 键补全**：输入 `sys` 然后按 `Tab` 键，系统会自动补全为 `system-view`。这不仅快，还能保证拼写绝对正确。
    
- **问号 `?` 帮助**：当你忘记某条命令怎么拼，或者不知道后面该接什么参数时，直接打个 `?`。比如输入 `ip ?`，系统会列出 `ip` 后面所有合法的可选命令。
    
- **方向键 `↑` 和 `↓`**：可以快速调出你刚刚敲过的历史命令，避免重复打字。
