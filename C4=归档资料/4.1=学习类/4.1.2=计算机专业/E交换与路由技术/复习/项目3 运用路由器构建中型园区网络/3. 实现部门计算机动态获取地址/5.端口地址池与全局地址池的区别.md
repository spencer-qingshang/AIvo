# 核心考点笔记：接口地址池与全局地址池的区别 (DHCP)

在华为 VRP 系统中，DHCP 提供了两种分配 IP 地址的机制，它们在**配置方式、作用范围和适用场景**上有着本质的区别。

## 一、核心区别提炼 (简答题答题点)

### 1. 作用范围与灵活度 (核心差异)

- **接口地址池 (Interface Pool)：** **死绑硬件**。地址池是基于具体的物理或逻辑接口动态生成的。它只能为**直连到该接口**的设备分配 IP，且分配的网段被死死限制在该接口的 IP 网段内。
    
- **全局地址池 (Global Pool)：** **彻底解耦**。地址池建立在系统的全局视图下，像一个独立存在的“大水库”。它可以被**任意一个或多个接口调用**，非常灵活。
    

### 2. 网关与网段的生成方式

- **接口地址池：** **全自动**。不需要手动划定网段，系统会自动把“接口自身的 IP 地址”作为分配出去的默认网关；把“接口所在的网段”作为 IP 分配范围。
    
- **全局地址池：** **纯手动**。必须由网络管理员一笔一画地敲命令，手动规定 `network` (分配网段) 和 `gateway-list` (默认网关)。
    

### 3. 应用场景 (设计大题选型依据)

- **接口地址池：** 适用于**小型网络、扁平化拓扑**。例如一台路由器直接连着几个部门的交换机，每个接口管一个部门。配置极简，敲两行命令就能用。
    
- **全局地址池：** 适用于**中大型网络、复杂拓扑**。特别是在以下两种考试必考场景中**必须**用全局池：
    
    1. **DHCP 中继 (Relay)：** DHCP 服务器和员工电脑不在同一个网段，必须用全局池跨网段发 IP。
        
    2. **单臂路由/三层交换：** 在规划大量 VLAN 时，通常会建一排全局地址池集中管理，然后分配给对应的 VLAN 虚拟接口 (Vlanif) 或子接口。

## 二、终极背诵对比表 (考前必看)

|**比较维度**|**接口地址池 (Interface Pool)**|**全局地址池 (Global Pool)**|
|---|---|---|
|**创建位置**|接口视图下自动生成|全局视图下手动创建|
|**分配网段**|自动继承该接口的 IP 网段|手动使用 `network` 命令指定|
|**默认网关**|自动使用该接口的 IP 作为网关|手动使用 `gateway-list` 命令指定|
|**配置复杂度**|极简 (即插即用)|较复杂 (需精细化配置)|
|**跨网段分配 (中继)**|❌ **不支持**|✅ **完全支持**|
|**接口启用命令**| `dhcp select interface` | `dhcp select global` |

**💡 考场避坑指南：**

在 eNSP 做实验时，一个接口下**只能选择其中一种模式**。如果你先敲了 `dhcp select interface`，后来又想换成全局池，必须先敲 `undo dhcp select interface`，否则系统会报错。
