# Track Project 4: IPv6 Practical Series Spec

## Overview
本任务旨在创建一套详细的 IPv6 实战演练系列手册，包含4个独立文件。每个文件都将提供从零开始（或基于基础环境）的“保姆级”操作指南。同时，更新原有的 `项目4 IPv6的园区网络.md` 作为索引和理论基础。

## Functional Requirements

### 1. 创建实战手册系列 (4个新文件)

#### 文件 1: `项目4_1_IPv6地址基础配置实战.md`
*   **核心目标**：从零搭建 IPv6 实验环境。
*   **步骤详解**：
    1.  **新建拓扑**：详细指导拖拽3台路由器、2台PC，并手动连线。
    2.  **全局配置**：开启 IPv6 功能。
    3.  **接口配置**：配置 Global Unicast Address (2001::) 和 Link-Local Address (FE80::)。
    4.  **PC 配置**：设置 IPv6 地址和网关。
    5.  **验证**：使用 `ping ipv6` 测试直连连通性。

#### 文件 2: `项目4_2_IPv6静态与默认路由实战.md`
*   **核心目标**：使用静态路由实现全网互通。
*   **步骤详解**：
    1.  **环境准备**：基于 4_1 的拓扑，或指导快速搭建。
    2.  **配置静态路由**：R1 和 R3 配置去往对端的静态路由。
    3.  **配置默认路由**：R1 指向出口的默认路由配置。
    4.  **验证**：查看 IPv6 路由表，Ping 测试跨网段连通性。

#### 文件 3: `项目4_3_RIPng动态路由实战.md`
*   **核心目标**：配置 RIPng 协议。
*   **步骤详解**：
    1.  **清理环境**：删除静态路由的命令。
    2.  **启动进程**：`ripng 1`.
    3.  **接口使能**：进入每个接口执行 `ripng 1 enable`。
    4.  **验证**：观察路由表的自动学习过程。

#### 文件 4: `项目4_4_OSPFv3动态路由实战.md` (重点)
*   **核心目标**：配置 OSPFv3 协议（企业主流）。
*   **步骤详解**：
    1.  **清理环境**：删除 RIPng 配置。
    2.  **Router ID**：解释为什么 IPv6 还需要配 IPv4 格式的 ID。
    3.  **区域配置**：在接口下绑定 `ospfv3 1 area 0`。
    4.  **验证**：查看 OSPFv3 邻居状态 (Full) 和路由表。

## Non-Functional Requirements
*   **零基础友好**：每一个步骤都必须假设用户是第一次操作，不省略“进入视图”、“保存配置”等基础动作。
*   **解释到位**：对于 IPv6 特有的概念（如地址太长怎么简写、为什么下一跳是 Link-Local 地址）进行“大白话”解释。
