面对即将到来的 4 月份自考，**VRRP（Virtual Router Redundancy Protocol，虚拟路由冗余协议）**绝对是网络可靠性设计中**最容易出综合大题**的核心考点。

在企业网络中，如果所有电脑都把网关指向同一台物理路由器，一旦这台路由器死机，整个部门都会断网（单点故障）。VRRP 的作用就是“找替身”：把两台或多台物理路由器在逻辑上捆绑成一台**“虚拟路由器”**，给它们分配一个共同的**虚拟 IP**。电脑只要把网关配成这个虚拟 IP，即使主路由器坏了，备用路由器也会在一秒内无缝接管，底下的人上网完全感觉不到断线。

在 eNSP 中配置 VRRP 的逻辑非常清晰。请务必记住一个核心大前提：**VRRP 的所有命令，都是在具体的接口视图下配置的。**

假设我们有两台路由器（R 1 和 R 2）连接着同一个局域网，我们准备把 R 1 设置为**主网关（Master）**，R 2 设置为**备用网关（Backup）**。内网电脑的虚拟网关 IP 规划为 `192.168.1.254`。

### 第一步：配置主路由器（R 1 - Master）

默认情况下，VRRP 的优先级是 100。为了让 R 1 成为主路由器，我们需要把它的优先级调高（数字越大越优先）。

Plaintext

```
[R1] interface GigabitEthernet 0/0/0               // 进入连接内网的接口
[R1-GigabitEthernet0/0/0] ip address 192.168.1.1 24  // 配置真实的物理IP地址
[R1-GigabitEthernet0/0/0] vrrp vrid 1 virtual-ip 192.168.1.254  // 开启VRRP组1，并设置虚拟网关IP
[R1-GigabitEthernet0/0/0] vrrp vrid 1 priority 120   // 将优先级调高到120（使其成为Master）
[R1-GigabitEthernet0/0/0] quit
```

_(可选高阶考点：考卷上有时会要求配置**抢占延时**。比如 R 1 故障恢复后，为了防止网络不稳定，不立刻抢回主控权，而是等 20 秒。命令是：`vrrp vrid 1 preempt-mode timer delay 20`)_

### 第二步：配置备用路由器（R 2 - Backup）

备用路由器的配置极其简单，只需要保持默认优先级（100）即可。只要它的优先级比 R 1 低，它就会乖乖做备胎。

Plaintext

```
[R2] interface GigabitEthernet 0/0/0               // 进入连接内网的接口
[R2-GigabitEthernet0/0/0] ip address 192.168.1.2 24  // 配置真实的物理IP地址
[R2-GigabitEthernet0/0/0] vrrp vrid 1 virtual-ip 192.168.1.254  // 加入VRRP组1，设置相同的虚拟IP
[R2-GigabitEthernet0/0/0] quit
```

### 第三步：电脑终端测试与排错查表

配置完成后，把底下 PC 的网关全部设置为 `192.168.1.254` 即可。

在考场排错或者 eNSP 验证时，这条查看命令是你必须要记住的：

Plaintext

```
[R1] display vrrp brief
```

这条命令会显示一张表格，重点看两个关键信息：

1. **State（状态）**：R 1 应该显示为 `Master`，R 2 应该显示为 `Backup`。如果两台都显示 Master，说明它们之间的心跳线断了（脑裂故障）。
    
2. **Pri（优先级）**：确认 R 1 是 120，R 2 是 100。
    

---

**🔥 经典考场陷阱**

考题经常会这样挖坑：VRRP 只能解决**网关设备本身**死机的问题。如果 R 1 连着外网（比如连着电信的光猫）的那根线上联端口断了，R 1 本身没死，它依然会霸占着 Master 的位置，但底下的电脑其实已经上不了网了。

为了解决这个致命缺陷，华为 VRP 引入了 **VRRP 联动接口监视（Track 机制）**：让 VRRP 盯着上联接口，一旦发现上联接口断了，立刻“自降优先级”，主动把主控权让给 R 2。

这部分的配置命令非常具有实战价值，你想接着看看如何敲出这行**“接口联动减分”**的神仙代码，还是想换个口味，进入动态路由协议（比如 OSPF）的基础学习？
