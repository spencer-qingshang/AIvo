### 一、考前默写速记表：基础配置（两派对决）

无论是哪种配置，敲击键盘的第一下永远是：

Plaintext

```
[Huawei] dhcp enable       # 全局开启 DHCP（没它全盘皆输）
```

**1. 全局地址池模式 (大题最爱：集中管理)**

Plaintext

```
[Huawei] ip pool {name}
[Huawei-ip-pool-{name}] network {ip} mask {mask}      # 宣告网段
[Huawei-ip-pool-{name}] gateway-list {ip}             # 设定网关
[Huawei-ip-pool-{name}] dns-list {ip}                 # 设定 DNS
[Huawei] interface Vlanif {id}                        # 必须进入用户所在的网关接口
[Huawei-Vlanif{id}] dhcp select global                # 挂载全局地址池
```

**2. 接口地址池模式 (选择/排错题常客：简单粗暴)**

Plaintext

```
[Huawei] interface GigabitEthernet 0/0/0              # 或进入 Vlanif 接口
[Huawei-GigabitEthernet0/0/0] ip address {ip} {mask}  # 接口 IP 即为用户网关
[Huawei-GigabitEthernet0/0/0] dhcp select interface   # 开启接口 DHCP
```

### 二、进阶防坑大招：DHCP 中继 (DHCP Relay)

联系一下我们刚才复习的**“三层组网”**：如果 AP/电脑在 VLAN 10，而真正的 DHCP 服务器在总机房的 VLAN 100，广播报文是过不去路由器的。这时候，VLAN 10 的网关设备就必须充当**“中继代理人（Relay）”**。

这在《交换与路由技术》的高阶组网大题中是必刷考点！

**DHCP 中继配置标准动作（在用户的网关设备上敲）：**

Plaintext

```
[Gateway] dhcp enable
[Gateway] interface Vlanif 10                            ! 进入客户端所在的网关接口
[Gateway-Vlanif10] ip address 192.168.10.254 24
[Gateway-Vlanif10] dhcp select relay                     ! 核心指令：告诉接口，你现在是个中继代理
[Gateway-Vlanif10] dhcp relay server-ip 10.1.1.100       ! 指路明灯：真实的 DHCP 服务器在这个 IP，把请求单播转给它！
```

### 三、考场排错“火眼金睛”

如果在模拟器或卷面的排错题中发现“PC 拿不到 IP”，迅速扫描卷面上的这几个“暗雷”：

1. **缺斤少两：** 全局地址池配得很完美，但忘了进 `Vlanif` 接口敲 `dhcp select global`。
    
2. **掩码格式错乱：** 在 `ip pool` 视图下宣告网段时，错写成反掩码（OSPF 的习惯）或者忘记写 `mask` 关键字。
    
3. **中继死胡同：** 配置了 DHCP 中继，但是**底层的路由根本不通**（网关 Ping 不通真实的 DHCP 服务器），导致代转发的报文石沉大海。
