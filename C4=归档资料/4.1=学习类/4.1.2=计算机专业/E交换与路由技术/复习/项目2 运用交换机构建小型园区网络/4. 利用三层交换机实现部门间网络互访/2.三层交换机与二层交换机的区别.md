在学习 VLAN 间通信时，最容易让初学者产生困惑的就是：既然大家都是交换机，为什么有的能实现网段互通，有的却只能在网段内转发？理解两者的本质区别，是掌握园区网层级架构（接入、汇聚、核心）的基础。

以下是深度对比拆解：

### 1. 工作层次的本质区别（核心考点）

- **二层交换机（Layer 2 Switch）**：
    - **工作层次**：OSI 模型的 **数据链路层**。
    - **转发依据**：**MAC 地址表**。
    - **处理对象**：数据帧（Frame）。它只看 MAC 地址，不关心 IP 地址。
- **三层交换机（Layer 3 Switch）**：
    - **工作层次**：OSI 模型的 **网络层**（同时也包含二层功能）。
    - **转发依据**：**路由表**（以及由此生成的硬件转发表）。
    - **处理对象**：IP 数据报文（Packet）。它能够解开数据帧，查看 IP 包头。

### 2. 核心功能的侧重点

- **二层交换机**：
    - **主要任务**：在同一个 VLAN（同一个网段）内高速交换数据。
    - **局限性**：无法实现不同 VLAN 之间的通信。
- **三层交换机**：
    - **主要任务**：**VLAN 间路由**。它既能在本网段高速交换，也能在不同网段之间进行路由转发。
    - **优势**：消除了传统路由器转发不同 VLAN 流量时的带宽瓶颈。

### 3. 详细对比表（建议背诵）

| 特性 | 二层交换机 | 三层交换机 |
| :--- | :--- | :--- |
| **转发引擎** | MAC 地址表（硬件芯片） | 硬件转发表（ASIC 芯片） |
| **寻址方式** | 根据目标 MAC 地址转发 | 根据目标 IP 地址转发 |
| **VLAN 通信** | 只能在同一个 VLAN 内 | 能够实现跨 VLAN 通信 |
| **网络层协议** | 不支持（如 OSPF, Static Route） | 支持各种路由协议 |
| **物理位置** | 通常位于网络**接入层** | 通常位于网络**汇聚层或核心层** |

### 4. 为什么不能用二层交换机直接实现部门互访？

**场景**：财务部（VLAN 10）想访问人事部（VLAN 20）。
- 如果中间只有二层交换机，当它收到 VLAN 10 的广播包或单播包时，即便找到了目标 MAC，如果发现目标属于不同 VLAN，由于二层交换机没有“三层网关”的概念，数据包会直接被丢弃。
- 必须有一个能识别 IP、并能给每个 VLAN 分配 IP 地址作为“出口（网关）”的设备，这就是三层交换机的用武之地。

### 5. 考点挑战：实战分析

**题目**：某公司网络中，PC1 (192.168.1.1/24) 与 PC2 (192.168.2.1/24) 连接在同一台交换机上。
1. 若该交换机为二层交换机，PC1 能否 Ping 通 PC2？为什么？
2. 若该交换机为三层交换机，如何配置才能让它们互通？

**标准答案：**
1.  **不能**。二层交换机只根据 MAC 地址在同一网段（VLAN）转发。由于两台 PC 在不同网段，PC1 会将包发给默认网关，而二层交换机无法提供网关 IP 接口，导致数据包无法路由。
2.  **配置步骤**：
    - 创建 VLAN 10 和 VLAN 20。
    - 将连接 PC1 的端口划入 VLAN 10，连接 PC2 的端口划入 VLAN 20。
    - 配置 **VLANIF 10** 的 IP 为 192.168.1.254（作为 PC1 网关）。
    - 配置 **VLANIF 20** 的 IP 为 192.168.2.254（作为 PC2 网关）。
    - 三层交换机会自动生成这两条直连路由，从而实现互通。

**实战逻辑总结**

在 eNSP 中，你可以通过 `display device` 命令查看设备型号。如果是 S3700、S5700 等型号，它们通常具备三层功能；如果是低端接入设备，则只支持二层。
