# 核心考点笔记：路由器基础命令大全 (华为 VRP)

## 一、系统管理与基础设置

这类命令主要用于设备的初始化和全局管理。

|**功能描述**|**完整命令**|**常用简写**|**视图层级**|
|---|---|---|---|
|**进入系统视图**| `system-view` |**`sys`**| `<Huawei>` 用户视图|
|**退回上一级视图**| `quit` |**`q`**|任意子视图|
|**直接退回用户视图**| `return` |**`ret`** 或 `Ctrl+Z` |任意子视图|
|**更改设备名称**| `sysname <名称>` |**`sysn R1`**| `[Huawei]` 系统视图|
|**保存当前配置 (⚠️必考)**| `save` |**`sa`**| `<Huawei>` 用户视图|
|**恢复出厂设置**| `reset saved-configuration` |无| `<Huawei>` 用户视图|
|**重启设备**| `reboot` |无| `<Huawei>` 用户视图|

- **考试避坑点：** `save` 命令只能在 `< >` 尖括号的用户视图下敲击，如果在 `[ ]` 系统视图下敲会报错！考场上做完实验，千万别忘了退回用户视图执行一次 `save`，否则一旦模拟器崩溃，配置全部清零。
    

## 二、接口与 IP 地址配置

这是让路由器接入网络的第一步。

|**功能描述**|**完整命令**|**常用简写**|**视图层级**|
|---|---|---|---|
|**进入千兆接口视图**| `interface GigabitEthernet 0/0/0` |**`int g0/0/0`**|系统视图|
|**进入环回口视图**| `interface LoopBack 0` |**`int l0`**|系统视图|
|**配置 IP 地址和掩码**| `ip address <IP> <掩码>` |**`ip add 192.168.1.1 24`**|接口视图|
|**撤销/删除当前 IP**| `undo ip address` |**`un ip add`**|接口视图|
|**开启物理接口**| `undo shutdown` |**`un sh`**|接口视图|
|**关闭物理接口**| `shutdown` |**`sh`**|接口视图|

- **💡 技巧：** 华为设备支持直接输入掩码位（如 `24` 代替 `255.255.255.0`），这在机考中能为你省下大量敲键盘的时间。
    

## 三、黄金状态查看命令 (Show 命令)

当网络不通时，不要盲目乱改，先用以下命令“诊断”病情。

|**功能描述**|**完整命令**|**常用简写**|**作用解析**|
|---|---|---|---|
|**查看接口 IP 简表**| `display ip interface brief` |**`dis ip int br`**|最核心命令！一眼看出哪个接口配了什么 IP，以及接口是不是处于 Up 状态。|
|**查看路由表**| `display ip routing-table` |**`dis ip rou`**|检查路由器有没有学习到去往目的地的路线。|
|**查看当前全部配置**| `display current-configuration` |**`dis cu`**|相当于查看路由器的“运行内存”，所有生效的命令全在这里。|
|**查看当前视图下的配置**| `display this` |**`dis th`**|极其好用！在哪个接口下敲，就只显示这个接口当前的配置，非常适合检查有没有敲错代码。|

## 四、网络连通性测试命令

配置完 IP 和路由后，必须验证网络是否通畅。

1. **Ping 命令 (测试端到端连通性)**
    
    - **基础用法：** `ping <目标IP>` （例如：`ping 192.168.2.10`）
        
    - **高级用法 (带源 Ping)：** `ping -a <源IP> <目标IP>`
        
        - _考点解析：_ 路由器通常有多个接口。默认情况下，路由器会用离目标最近的接口 IP 去 ping。如果你想测试“从我的 Loopback 0 口出发，能不能通目标网段”，就需要用 `-a` 参数指定源地址。
            
2. **Tracert 命令 (路由追踪)**
    
    - **用法：** `tracert <目标IP>`
        
    - _作用：_ 当 ping 不通时，tracert 可以帮你一跳一跳地追踪数据包走过的路径。停在哪一个 IP 上，就说明是从那台路由器开始断的，非常适合排查多台路由器串联的故障节点。

### 📝 综合配置小模板 (考前建议手敲几遍形成肌肉记忆)

**场景：** 给 R 1 改名，为其 G 0/0/0 接口配置 IP，并写一条去往 `10.0.0.0/8` 的静态路由。

Plaintext

```
<Huawei> system-view                 // 进入系统视图
[Huawei] sysname R1                  // 将设备改名为 R1
[R1] interface GigabitEthernet 0/0/0 // 进入 G0/0/0 接口
[R1-GigabitEthernet0/0/0] ip address 192.168.1.1 24  // 配置IP和24位掩码
[R1-GigabitEthernet0/0/0] quit       // 退回系统视图
[R1] ip route-static 10.0.0.0 8 192.168.1.2  // 配置静态路由（下一跳为1.2）
[R1] quit                            // 退回用户视图
<R1> save                            // 保存配置 (按 Y 确认)
```
