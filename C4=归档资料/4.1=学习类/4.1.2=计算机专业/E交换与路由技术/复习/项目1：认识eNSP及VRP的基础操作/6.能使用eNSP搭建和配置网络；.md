太棒了！软件安装并跑通后，我们就真正进入了实操阶段。为了在下个月的《交换与路由技术》考试中稳稳拿下配置与排错大题，最好的方法就是亲自动手搭建一个完整的网络并让它通信。

我们今天来做一个最经典、也是所有网络互通基础的综合小实验：**“跨网段通信”**。

我们将使用一台路由器连接两个不同网段的 PC，让它们能够互相 `ping` 通。这会把你前面学到的连线、设备启动和 VRP 基础命令全部串联起来。

请跟着以下四个步骤在 eNSP 中操作：

### 第一步：搭建基础拓扑

1. **拖拽设备**：从左侧设备区拖出一台 **AR 2220 路由器**放到中间，再从终端区拖出**两台 PC（PC 1 和 PC 2）**分别放到路由器左右两侧。
    
2. **物理连线**：
    
    - 选择连线区的 **Copper（双绞线）**。
        
    - 点击 PC 1，选择 `Ethernet 0/0/1`，连接到路由器的 `GigabitEthernet 0/0/0` 接口。
        
    - 点击 PC 2，选择 `Ethernet 0/0/1`，连接到路由器的 `GigabitEthernet 0/0/1` 接口。
        
3. **启动设备**：点击顶部工具栏的**绿色播放按钮**，开启所有设备。等待接口旁的红点全部变成绿点。
    

### 第二步：配置 PC 终端（图形化操作）

真实网络中，PC 需要 IP 地址和网关才能上网。在 eNSP 中，双击 PC 图标即可进入配置页面：

- **配置 PC 1（左侧，属于 1.0 网段）**：
    
    - IP 地址：`192.168.1.10`
        
    - 子网掩码：`255.255.255.0`
        
    - 网关：`192.168.1.254` （这是稍后我们要配在路由器接口上的地址）
        
    - 配置完后点击右下角的“应用”。
        
- **配置 PC 2（右侧，属于 2.0 网段）**：
    
    - IP 地址：`192.168.2.10`
        
    - 子网掩码：`255.255.255.0`
        
    - 网关：`192.168.2.254`
        
    - 点击“应用”。
        

### 第三步：配置路由器（VRP 命令行操作）

双击路由器 AR 1，进入命令行界面。当出现 `<Huawei>` 提示符时，开始敲击以下命令（可以利用 `Tab` 键补全）：

Plaintext

```
<Huawei> system-view                  // 进入系统视图
[Huawei] sysname R1                   // 给路由器改名为R1
[R1] interface GigabitEthernet 0/0/0  // 进入连接 PC1 的 g0/0/0 接口视图
[R1-GigabitEthernet0/0/0] ip address 192.168.1.254 24  // 配置IP和掩码长度（充当PC1的网关）
[R1-GigabitEthernet0/0/0] quit        // 退回系统视图
[R1] interface GigabitEthernet 0/0/1  // 进入连接 PC2 的 g0/0/1 接口视图
[R1-GigabitEthernet0/0/1] ip address 192.168.2.254 24  // 配置IP和掩码长度（充当PC2的网关）
[R1-GigabitEthernet0/0/1] return      // 直接退回用户视图
<R1> save                             // 输入y保存配置
```

**小贴士**：配置完成后，你可以输入 `display ip interface brief`，检查一下两个接口是否都有了 IP 地址，并且状态都是 `UP`。

### 第四步：见证奇迹的连通性测试

现在，PC 1 和 PC 2 应该可以跨越网段互相通信了。

1. 双击 **PC 1**，在弹出的窗口中切换到 **“命令行”** 标签页。
    
2. 输入命令：`ping 192.168.2.10` （也就是去 ping PC 2 的地址），然后按回车。
    
3. 如果你看到类似 `Reply from 192.168.2.10: bytes=32 time=15ms TTL=127` 这样连续的回复信息，并且没有丢包（Lost = 0），说明你的第一个网络搭建与配置完美成功！
    

这个实验虽然简单，但它是所有大型网络的核心缩影。路由器因为两端接口配置了 IP，自动生成了“直连路由”，从而实现了两个不同子网的互通。
