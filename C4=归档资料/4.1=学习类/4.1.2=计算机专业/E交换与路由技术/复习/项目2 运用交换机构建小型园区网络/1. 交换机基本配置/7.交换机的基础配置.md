当我们拿到一台崭新的交换机，或者在 eNSP 中刚刚拖出一台设备时，第一步要做的就是“基础配置”（Device Initialization）。

在四月份的《交换与路由技术》考试中，不管是排错题还是大题，这些基础配置代码往往是默认你已经熟练掌握的“垫脚石”。如果基础没配好，后面的高级协议（如 OSPF 或 STP）根本跑不起来。

与路由器不同，**二层交换机的物理接口是不能直接配置 IP 地址的**。这是一个极其经典的易错点！要远程管理交换机，我们必须给它配置一个“逻辑”接口的 IP。

下面是一套标准的、可以在 eNSP 中直接敲击验证的“交换机开荒 SOP（标准作业程序）”：

### 第一步：修改设备名称与系统时钟

在拥有多台设备的网络拓扑中，如果所有交换机都叫 `Huawei`，你很快就会不知道自己在敲哪台设备。

Plaintext

```
<Huawei> clock datetime 12:00:00 2026-03-29  // 在用户视图下校准时间
<Huawei> system-view                         // 进入系统视图
[Huawei] sysname SW1                         // 将设备重命名为 SW1
[SW1] 
```

### 第二步：配置管理 IP 地址（核心重点：VLANIF 接口）

既然物理口不能配 IP，我们就需要用到 **VLANIF（VLAN 接口）**。你可以把它理解为交换机内部一张虚拟的网卡。默认情况下，交换机的所有物理接口都属于 VLAN 1，因此我们直接给 VLANIF 1 配置 IP，作为这台交换机的管理地址。

Plaintext

```
[SW1] interface Vlanif 1                     // 进入 VLAN 1 的逻辑接口视图
[SW1-Vlanif1] ip address 192.168.100.254 24  // 为交换机配置管理 IP 和子网掩码
[SW1-Vlanif1] quit                           // 退回系统视图
```

### 第三步：开启 Telnet 远程登录服务

有了 IP 地址后，我们还需要在交换机上打开一扇“远程大门”（VTY 虚拟终端接口），并设置密码，这样同网段的 PC 才能通过网线远程登录控制它。

Plaintext

```
[SW1] telnet server enable                   // 开启 Telnet 服务功能（较新的VRP版本默认关闭，需手动开启）
[SW1] user-interface vty 0 4                 // 进入 VTY 0 到 4 号虚拟终端线路（允许同时5个用户远程登录）
[SW1-ui-vty0-4] authentication-mode password // 设置认证模式为密码认证
[SW1-ui-vty0-4] set authentication password cipher Huawei@123 // 设置远程登录的密文密码
[SW1-ui-vty0-4] user privilege level 15      // 【关键】将登录用户的权限提升至最高级（15级），否则登录后只能看不能改
[SW1-ui-vty0-4] return                       // 直接退回用户视图
```

### 第四步：保存与验证

在真实机房中，拔掉电源前如果不执行这一步，刚才敲的所有代码都会灰飞烟灭。

Plaintext

```
<SW1> save                                   // 保存配置（系统会提示 Y/N，输入 y 确认）
<SW1> display ip interface brief             // 验证 VLANIF 1 的 IP 是否配置成功且双 UP
```

---

**实战检验**

这四步完成后，这台交换机就已经具备了被远程接管的能力。在 eNSP 中，你可以用一根双绞线（Copper）把一台 PC 连到这台交换机的任意接口上，给 PC 配置 `192.168.100.1` 的 IP 地址，然后就可以在 PC 的命令行里输入 `telnet 192.168.100.254` 尝试远程登录了。
