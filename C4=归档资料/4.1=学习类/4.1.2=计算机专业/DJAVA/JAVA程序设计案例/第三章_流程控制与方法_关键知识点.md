# 第三章：流程控制与方法 - 必备关键知识点

本章涵盖了程序的基本运行逻辑，包括如何组织代码（顺序、分支、循环）以及如何封装可重用代码（方法）。

---

## 1. 三种基本程序结构

*   **顺序结构 (Sequential Structure)**：
    *   **概念**：程序从上到下，一行一行地执行代码。
    *   **特点**：没有任何跳转或判断，是程序执行的默认流程。
*   **分支结构 (Branching / Selection Structure)**：
    *   **概念**：根据条件判断，选择执行不同的代码块。
    *   **关键词**：`if`, `else`, `switch`。
*   **循环结构 (Loop Structure)**：
    *   **概念**：在满足特定条件下，重复执行某段代码。
    *   **关键词**：`for`, `while`, `do-while`。

---

## 2. 分支结构

*   **`if` 语句**：
    *   **单分支**：`if (条件) { 代码 }` (满足条件就做)
    *   **双分支**：`if (条件) { 代码A } else { 代码B }` (二选一)
    *   **多分支**：`if (条件1) { ... } else if (条件2) { ... } else { ... }` (多选一)
    *   **特点**：擅长处理**区间范围**和**布尔值**判断，是最通用的选择结构。
*   **`switch` 语句**：
    *   **语法**：
        ```java
        switch (变量) {
            case 值1: ... break;
            case 值2: ... break;
            default: ... break; // default 可选，且位置灵活
        }
        ```
    *   **特点**：擅长处理**固定值/离散值**的匹配，结构更清晰。
    *   **注意点 (必记)**：
        *   **`break` 关键字**：用于跳出 `switch` 结构，防止“穿透现象”。
        *   **支持类型**：`byte`, `short`, `int`, `char`, `String` (JDK7+), `enum`。
        *   **不支持类型**：`long`, `float`, `double`, `boolean`。

---

## 3. 循环结构

*   **`for` 循环**：
    *   **格式**：`for (初始化; 条件判断; 步进迭代) { 循环体 }`
    *   **适用场景**：**循环次数固定**的情况。
*   **`while` 循环**：
    *   **格式**：`while (条件) { 循环体 }`
    *   **特点**：**先判断，后执行**。如果条件一开始就不满足，一次都不执行（0次）。
    *   **适用场景**：**循环次数不确定**，只知道停止条件的情况。
*   **`do-while` 循环**：
    *   **格式**：`do { 循环体 } while (条件);` (注意末尾分号)
    *   **特点**：**先执行，后判断**。无论条件如何，循环体**至少执行一次**（1次）。
*   **跳转语句**：
    *   **`break`**：**终止/跳出**。彻底结束当前所在的整个循环结构。
    *   **`continue`**：**跳过/继续**。跳过本次循环中尚未执行的剩余代码，直接进入下一次循环。

---

## 4. 方法 (Method)

*   **概念**：将具有独立功能的**代码块**组织起来，封装成一个整体，以便**复用**。
*   **作用**：**代码复用**，简化程序结构，提高可读性。
*   **定义三要素**：
    1.  **返回值类型**：方法执行完后返回给调用者的结果类型 (`void` 表示无返回值)。
    2.  **方法名**：遵循小驼峰命名法，见名知意。
    3.  **参数列表**：方法执行所需的数据（形式参数），调用时传入具体值（实际参数）。
*   **定义格式**：`修饰符 返回值类型 方法名(参数列表) { 方法体; [return 返回值;] }`
*   **调用方法**：`方法名(实际参数);`
*   **递归 (Recursion)**：
    *   **概念**：**方法自己调用自己**。
    *   **核心要点 (必记)**：
        1.  **递归出口**：必须有明确的结束条件，否则会导致栈内存溢出 (`StackOverflowError`)。
        2.  **递归规律**：每次调用都必须比上一次更接近出口。
