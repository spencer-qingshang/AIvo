# Track Specification: IPv6 Campus Network

#### **1. 概述 (Overview)**
本项目旨在通过“理论回答 + 独立实战”的模式，带领用户（网络小白）掌握 IPv6 的核心知识。用户将首先在原大纲文件中完成理论填空，随后通过四个独立的实验文件，掌握从 IPv6 基础配置到动态路由协议的操作。

#### **2. 功能需求 (Functional Requirements)**
1.  **理论部分**：在 `项目4 IPv6的园区网络.md` 中补充所有大纲要求的理论答案。
    *   解释 IPv6 的优势、报头结构、地址记录格式。
    *   对比 IPv6 与 IPv4 的区别。
    *   解释各种 IPv6 地址表达方式。
2.  **实战部分**：创建 4 个独立的 Markdown 文件，分别对应以下实战：
    *   `项目4_1_IPv6地址基础配置实战.md`
    *   `项目4_2_IPv6静态与默认路由实战.md`
    *   `项目4_3_RIPng动态路由实战.md`
    *   `项目4_4_OSPFv3动态路由实战.md`
3.  **实战内容标准**：每个文件必须包含以下四个元素：
    *   **A. 生活化比喻**：用小白能听懂的故事解释枯燥概念。
    *   **B. 逐词命令拆解**：对关键配置命令进行拆分讲解。
    *   **C. 避坑必杀技**：总结新手最容易犯的错误。
    *   **D. 实验自测**：提供验证实验成功的简单方法。

#### **3. 交付标准 (Acceptance Criteria)**
1.  `项目4 IPv6的园区网络.md` 包含完整的理论知识点解答。
2.  创建 4 个内容详实、符合“小白风格”的实战手册文件。
3.  用户能够通过阅读实战手册，独立在 eNSP 中完成相应的网络搭建。

#### **4. 非功能性需求 (Non-Functional Requirements)**
*   **语言**：必须使用中文，语调亲切、浅显易懂。
*   **格式**：符合项目现有的 Markdown 规范，代码块清晰。

#### **5. 超出范围 (Out of Scope)**
*   不涉及 IPv6 的高级特性（如 BGP4+ 或 IPsec 细节）。
*   不涉及 IPv4/IPv6 转换技术（如双栈、隧道、NAT64）。
