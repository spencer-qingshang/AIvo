了解了 VLAN 的底层标签（802.1 Q 帧格式）后，下一个核心考点就是：交换机到底是怎么判断一个普通数据帧进来时，应该给它套上哪个 VLAN 标签的？这就是**VLAN 的划分方式**。

在华为 VRP 系统和你的《交换与路由技术》备考中，主要有 5 种划分方式。按照实际工程应用频率和考试考查的侧重点，我们为你做一个清晰的优先级梳理：

### 1. 基于端口划分（绝对核心、必考大题）

这是目前企业网络中最常用、最简单，也是你在 eNSP 综合实验中用得最多的一种方式。

- **工作原理**：简单粗暴，直接把交换机的物理接口和 VLAN 编号绑定。比如规定 1 到 10 号口属于 VLAN 10，只要你的电脑插在这几个口上，你发出的数据就会被强制打上 VLAN 10 的标签。
    
- **优点**：配置极其简单，逻辑清晰，排错容易。
    
- **缺点**：如果员工带着笔记本电脑去了别的楼层（换了交换机接口），网管就必须登录设备重新配置端口，缺乏移动灵活性。
    
- **VRP 实战代码**（你必须烂熟于心）：
    
    Plaintext
    
    ```
    [Huawei] vlan 10                               // 创建 VLAN 10
    [Huawei-vlan10] quit
    [Huawei] interface GigabitEthernet 0/0/1       // 进入接口
    [Huawei-GigabitEthernet0/0/1] port link-type access  // 设为 Access 模式
    [Huawei-GigabitEthernet0/0/1] port default vlan 10   // 将接口划入 VLAN 10
    ```
    

### 2. 基于 MAC 地址划分（选择题高频考点）

- **工作原理**：交换机内部维护了一张“MAC 地址与 VLAN 的映射表”。当数据帧进入交换机时，交换机会去查这个源 MAC 地址属于哪个 VLAN。
    
- **优点**：**极大地提高了用户的物理移动性**。员工抱着电脑在公司里随便换座位，插在任何交换机的任何接口上，所在的 VLAN 都不变（因为网卡的 MAC 地址是唯一的）。
    
- **缺点**：前期配置简直是网管的噩梦。如果公司有 500 号人，网管需要收集 500 个 MAC 地址并在交换机里一条条敲命令录入；而且如果有人换了新电脑，还得重新配置。
    

### 3. 基于 IP 子网划分

- **工作原理**：根据报文中的源 IP 地址或所在子网来划分 VLAN。比如规定只要源 IP 是 `192.168.2.0/24` 网段的，统统打上 VLAN 20 的标签。
    
- **应用场景**：适合对安全性和移动性要求都比较高的场景，或者公司内部本来就是按 IP 网段来区分部门的。
    

### 4. 基于协议划分

- **工作原理**：根据网络层跑的具体协议类型（如 IPv 4、IPv 6、IPX 等）来划分。
    
- **应用场景**：在现代企业网中已经很少见，通常用于一些非常老旧、混合了多种异构网络协议的特殊环境。
    

### 5. 基于策略划分（最复杂）

- **工作原理**：大杂烩。你可以把 MAC 地址、IP 地址、端口号等多种条件组合成一条“策略”。只有当终端完全符合这些苛刻的条件时，才会被分到特定的 VLAN。
    
- **特点**：安全性最高，但配置和排错极其复杂。
    

---

**⚠️ 考场避坑指南：匹配优先级**

如果一道选择题问你：“一台电脑的 MAC 地址被划在 VLAN 10，但它插在了一个被划在 VLAN 20 的端口上，这台电脑发出的数据最终属于哪个 VLAN？”

这就涉及到了 VLAN 划分方式的**生效优先级**。在华为设备默认情况下，优先级顺序是：

**基于 MAC 地址 > 基于 IP 子网 > 基于协议 > 基于端口**

（也就是说，只要配置了基于 MAC 地址，交换机会优先听 MAC 的，忽略端口配置）。
