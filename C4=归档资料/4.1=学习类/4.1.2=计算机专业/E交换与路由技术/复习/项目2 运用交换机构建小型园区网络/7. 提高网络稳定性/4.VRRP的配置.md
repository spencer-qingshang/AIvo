在华为 VRP 平台上配置 VRRP，核心在于“接口下配置”。你需要进入作为网关的逻辑接口（通常是 VLANIF），定义虚拟路由器的 ID 和 IP，并手动干预优先级来确定谁是 Master。

以下是标准配置流程与实战命令：

```markdown
在 eNSP 综合组网大题中，VRRP 通常与 **MSTP（多生成树）** 结合使用，实现流量的负载均衡。掌握 VRRP 的基础配置，是确保网关高可用的最后一步。

以下是实现“主备备份”模式的标准配置手册：

### 1. 配置逻辑：先有接口，后有 VRRP

VRRP 是跑在三层接口上的。在配置前，请确保两台交换机的 **VLANIF** 接口已经配好了各自的真实物理 IP，且二层链路（Trunk）已经打通。

### 2. 核心命令拆解

假设：虚拟网关 IP 为 `192.168.10.254`，VRID 为 `10`。

#### **SW1（设为主网关 Master）**
```bash
[SW1] interface Vlanif 10
[SW1-Vlanif10] vrrp vrid 10 virtual-ip 192.168.10.254  # 开启虚拟组并设 VIP
[SW1-Vlanif10] vrrp vrid 10 priority 120               # 提高优先级（默认100），确保成主
```

#### **SW2（设为备份网关 Backup）**
```bash
[SW2] interface Vlanif 10
[SW2-Vlanif10] vrrp vrid 10 virtual-ip 192.168.10.254  # VIP 必须与主一致
# 优先级保持默认 100，自动成为 Backup
```

### 3. 验证与检查（排错必敲）

配置完成后，必须检查状态是否符合预期：

- **查看 VRRP 详细状态**：
  `display vrrp`
  - **Master 设备**：`State` 应显示为 `Master`。
  - **Backup 设备**：`State` 应显示为 `Backup`。
  - **检查项**：确认 `Virtual IP` 是否正确，`Priority` 是否生效。

- **查看摘要信息**：
  `display vrrp brief`
  - 快速确认当前设备在各个虚拟组中的角色。

### 4. 考点挑战：联动配置实战

**场景描述**：
SW1 是 Master，SW2 是 Backup。SW1 通过 GE0/0/24 接口连接互联网路由器。
**问题**：如果 SW1 的 GE0/0/24 线路断了，但 SW1 没关机，流量依然会发给 SW1 导致断网。请写出配置“接口联动”的命令，让 SW1 在发现线路故障时自动“让位”给 SW2。

**标准答案：**
在 SW1 的接口下配置 **VRRP 监控（Track）** 功能：
```bash
[SW1] interface Vlanif 10
[SW1-Vlanif10] vrrp vrid 10 track interface GigabitEthernet 0/0/24 reduced 30
```
**逻辑分析**：
这条命令的意思是：实时盯着 GE0/0/24 接口。一旦它变成 Down，SW1 自动将自己的 VRRP 优先级**减少 30**（120 - 30 = 90）。由于 90 小于 SW2 的默认优先级 100，SW2 会立即抢占成为新的 Master，从而引导流量改走备用路径。

**实战逻辑总结**

在综合实训中，VRRP 的配置顺序通常是：
**配物理网关 IP** -> **设虚拟网关 IP** -> **主设备调高优先级** -> **主设备配置接口联动**。
