# 核心考点笔记：RIP 版本深度对比

## 一、 RIPv 1 与 RIPv 2 的本质区别 (⚠️ 必考大题)

| **特性**   | **RIPv 1 (老旧/淘汰)**       | **RIPv 2 (主流/常用)**        |
| -------- | ------------------------ | ------------------------- |
| **路由类型** | **有类 (Classful)**        | **无类 (Classless)**        |
| **掩码支持** | 不携带子网掩码 (不支持 VLSM/CIDR)  | **携带子网掩码 (支持 VLSM/CIDR)** |
| **更新方式** | **广播** (255.255.255.255) | **组播** (224.0.0.9)        |
| **认证功能** | 不支持 (不安全)                | **支持 (明文或 MD 5 认证)**      |
| **发送方式** | 泛洪整个网段，所有主机都得处理          | 只发给监听组播地址的邻居，效率高          |

- **⚠️ 考试避坑：** 如果你在 eNSP 实验中使用了子网划分（如 `192.168.1.128/25`），但忘了切换到 `version 2`，路由器会默认按 `192.168.1.0/24` 来传播，导致网络直接瘫痪。
    

---

## 二、 RIPng (Next Generation)

- **适用对象：** 专门为 **IPv 6** 设计的版本。
    
- **更新方式：** 使用组播地址 **`FF02::9`**。
    
- **跳数限制：** 依然保持 **15 跳** 的限制（16 跳不可达）。
    
- **注意点：** 它是基于 IPv 6 运行的独立协议，与 IPv 4 的 RIPv 1/v 2 不兼容。
    

---

## 三、正确实现 RIPv 2 配置 (纯简写版)

在华为设备上，默认启动的是 RIPv 1，所以**“进进程，先切版本”**是满分操作的第一步。

Plaintext

```
<Huawei> sys
[Huawei] rip 1                 // 1. 启动进程
[Huawei-rip-1] ver 2           // 2. 关键：手动切换到 Version 2
[Huawei-rip-1] network 192.168.1.0   // 3. 宣告主类网段 (不要带掩码)
[Huawei-rip-1] network 10.0.0.0      // 宣告另一个主类网段
[Huawei-rip-1] q
```

## 四、进阶考点：水平分割 (Split Horizon)

不管哪个版本的 RIP，为了防止**路由环路**，默认都会开启“水平分割”。

- **原理：** 路由器从一个接口学习到的路由信息，**绝对不会**再从这个接口发回去。
    
- **作用：** 避免数据包在两台路由器之间“踢皮球”。
