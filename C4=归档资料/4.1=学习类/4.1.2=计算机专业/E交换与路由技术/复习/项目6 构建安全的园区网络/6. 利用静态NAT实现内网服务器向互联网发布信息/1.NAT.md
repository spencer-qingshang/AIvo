如果说 ACL 是网络里的“保安”，那么 NAT 就是站在公司大门（网关）处的**“神级翻译官”**。

在《交换与路由技术》中，NAT 几乎是所有综合配置大题的必考项，因为它解决了现网中最现实的一个问题：**私网 IP 是绝对上不了公网（互联网）的，必须换成公网 IP。**

### 一、为什么必须有 NAT？（大白话原理）

全世界的 IPv 4 地址早就枯竭了，不可能给每台电脑都发一个公网 IP。所以规定了一批**私网 IP（比如你最熟悉的 `192.168.x.x` 或者 `10.x.x.x`）**。

私网 IP 可以在公司、家里随便用，但一旦跑到公网上，运营商的路由器会直接把它丢弃。

- **NAT 的作用：** 当你（私网 IP `192.168.1.10`）要访问百度时，数据包来到公司的出口路由器。路由器会把数据包里的“源私网 IP”擦掉，临时替换成路由器自己的“公网 IP”，然后再发到互联网上。等百度回包时，路由器再根据之前的记录，把“公网 IP”翻译回“私网 IP”，交给你。
    

### 二、考场必背：NAT 的四大门派（分类）

这是选择题和配置题最爱考的地方，务必分清它们的应用场景：

|**NAT 类型**|**转换逻辑**|**核心特点**|**适用场景与考点**|
|---|---|---|---|
|**1. 静态 NAT (Static NAT)**|**一对一**（1 个私网配 1 个公网）|地址写死，绝不改变。|极其浪费公网 IP。通常用于给内部**特定服务器**分配一个固定的公网脸面。|
|**2. 动态 NAT (Dynamic NAT)**|**多对多**（不带端口号）|有一个公网 IP 池。谁先上网谁先用，用完别人就上不了了。|依然浪费地址，**现网几乎被淘汰，考试仅作理论考察**。|
|**3. NAPT (网络地址端口转换)🌟**|**多对一 / 多对多**（带端口号）|**绝杀技术！** 利用不同的**端口号**来区分不同的内网电脑。|**现网绝对主流！** 让成百上千台内网电脑，共享 1 个或几个公网 IP 上网。|
|**4. NAT Server (端口映射)**|**外网主动访问内网**|将公网 IP 的特定端口（如 80），映射给内网服务器的特定端口。|公司内网有一台 Web 服务器，想让全世界的人都能访问它。|

### 三、华为最经典的配置大招：Easy IP

在 NAPT 中，有一种最极端、在考卷中最常见的方式叫 **Easy IP**。

它不需要你购买额外的公网 IP 地址池，而是直接**利用路由器出接口（拨号口）那个仅有的公网 IP**，让全公司的人上网。

这就完美呼应了你刚才复习的 **基本 ACL**！NAT 必须和 ACL 联动：**ACL 负责抓人，NAT 负责翻译。**

**【考试满分代码流：允许 192.168.1.0/24 网段上网】**

Plaintext

```
<Huawei> system-view
! 1. 用基本 ACL 圈定哪些人有资格被翻译（抓人）
[Huawei] acl 2000
[Huawei-acl-basic-2000] rule 5 permit source 192.168.1.0 0.0.0.255
[Huawei-acl-basic-2000] quit

! 2. 进入连接外网（互联网）的出接口
[Huawei] interface GigabitEthernet 0/0/1
[Huawei-GigabitEthernet0/0/1] ip address 200.1.1.1 24  ! 假设这是公网IP

! 3. 核心大招：在这个出接口上调用 ACL，执行 Easy IP 转换
[Huawei-GigabitEthernet0/0/1] nat outbound 2000
[Huawei-GigabitEthernet0/0/1] quit
```

_💡 考场秒杀眼：看到 `nat outbound <ACL编号>` 这行代码，直接答 **Easy IP (或 NAPT)**，意思是“凡是符合 ACL 2000 规则的私网数据包，从这个接口出去时，全部换成这个接口的公网 IP”。_

### 四、另一个大题核心：NAT Server（端口映射）

如果公司内网有一台服务器（`192.168.1.100`），提供网页服务（端口 80）。你想让外网用户通过公网 IP `200.1.1.1` 访问它。

**【只需在公网接口下一条命令搞定】**

Plaintext

```
[Huawei-GigabitEthernet0/0/1] nat server protocol tcp global 200.1.1.1 80 inside 192.168.1.100 80
```

_这句长代码的翻译：把外部（global）访问 `200.1.1.1` 的 TCP 80 端口的流量，统统扔给内部（inside）的 `192.168.1.100` 的 80 端口。_

**最后总结：**

- 员工要上外网 ➡️ **nat outbound + ACL** (多对一出差)
    
- 外网要进来看服务器 ➡️ **nat server** (开专线迎客)
