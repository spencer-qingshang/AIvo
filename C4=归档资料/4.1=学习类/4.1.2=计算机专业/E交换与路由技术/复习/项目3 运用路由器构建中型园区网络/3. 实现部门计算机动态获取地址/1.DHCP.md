# 核心考点笔记：DHCP 原理与基础配置 (华为 VRP)

## 一、 DHCP 的核心作用

**DHCP (Dynamic Host Configuration Protocol，动态主机配置协议)** 的主要作用是为局域网内的终端设备（如 PC、手机）**自动分配**网络参数，从而减轻管理员工作量并避免 IP 地址冲突。

一个完整的 DHCP 分配会包含四大核心参数：

1. **IP 地址 (IP Address)**
    
2. **子网掩码 (Subnet Mask)**
    
3. **默认网关 (Default Gateway)** - _让设备能跨网段上网_
    
4. **DNS 服务器 (DNS Server)** - _让设备能解析域名_
    

---

## 二、 DHCP 工作原理：经典 DORA 过程 (⚠️ 简答/流程题必考)

当一台没有任何网络配置的新电脑接入网络时，它获取 IP 地址的过程分为标准的四个步骤（简称 **DORA**）：

1. **D - Discover (发现阶段 - 客户端发送广播)：**
    
    - **动作：** 电脑向全网发送 DHCP Discover 广播包。
        
    - **含义：** “网络里有 DHCP 服务器吗？我需要一个 IP 地址！”
        
    - _(源 IP: 0.0.0.0，目的 IP: 255.255.255.255)_
        
2. **O - Offer (提供阶段 - 服务器回复单播/广播)：**
    
    - **动作：** DHCP 服务器收到请求后，从地址池中挑出一个未分配的 IP（例如 192.168.1.10），发送 DHCP Offer 包给电脑。
        
    - **含义：** “我有这个 IP，你要不要用？”
        
3. **R - Request (请求阶段 - 客户端发送广播)：**
    
    - **动作：** 电脑正式决定使用该 IP，并发送 DHCP Request **广播包**。
        
    - **⚠️ 经典考点：为什么这里还要用广播？** 因为网络中可能存在多台 DHCP 服务器，电脑通过广播的方式不仅是告诉选中的服务器“我要了”，同时也是通知其他未被选中的服务器：“我已经有 IP 了，你们把之前预留给我的 IP 收回吧。”
        
4. **A - Acknowledgment (确认阶段 - 服务器回复单播/广播)：**
    
    - **动作：** 服务器收到 Request 后，发送 DHCP ACK 包确认，并下发租期（Lease Time）等详细参数。
        
    - **含义：** “成交！这个 IP 租给你了，配置生效。”

## 三、华为路由器 DHCP 必考配置命令

### 前置条件：全局开启 DHCP (极易遗漏的丢分点)

无论使用哪种方式，都**必须**先在系统视图下开启 DHCP 功能：

Plaintext

```
<Huawei> system-view
[Huawei] dhcp enable           // 全局开启 DHCP 功能 (简写: dhcp en)
```

### 方式一：基于接口的 DHCP 分配 (最简单、最常用)

**适用场景：** 路由器某接口直连某个部门，直接把该接口所在的网段分配给下面的人。

_假设 G 0/0/0 接口 IP 为 192.168.1.254，要为该网段分配 IP：_

Plaintext

```
[Huawei] interface GigabitEthernet 0/0/0
[Huawei-GigabitEthernet0/0/0] ip address 192.168.1.254 24
[Huawei-GigabitEthernet0/0/0] dhcp select interface         // 开启接口 DHCP (核心命令)
[Huawei-GigabitEthernet0/0/0] dhcp server dns-list 8.8.8.8  // (可选) 配置下发的 DNS
[Huawei-GigabitEthernet0/0/0] quit
```

### 方式二：基于全局地址池的 DHCP 分配 (常结合单臂路由/三层交换考)

**适用场景：** 需要精细化管理多个不同的网段，先在路由器上建好“IP 水池”，再分配给指定接口调用。

_假设我们要建一个名为 `vlan10_pool` 的地址池，网段为 `192.168.10.0/24`：_

**第一步：创建并配置全局地址池**

Plaintext

```
[Huawei] ip pool vlan10_pool                     // 创建名为 vlan10_pool 的地址池
[Huawei-ip-pool-vlan10_pool] network 192.168.10.0 mask 24    // 指定宣告的网段和掩码
[Huawei-ip-pool-vlan10_pool] gateway-list 192.168.10.254     // 指定下发的默认网关
[Huawei-ip-pool-vlan10_pool] dns-list 114.114.114.114        // 指定下发的 DNS
[Huawei-ip-pool-vlan10_pool] quit
```

**第二步：在接口上调用该地址池**

Plaintext

```
[Huawei] interface GigabitEthernet 0/0/1
[Huawei-GigabitEthernet0/0/1] ip address 192.168.10.254 24
[Huawei-GigabitEthernet0/0/1] dhcp select global             // 告诉该接口使用全局地址池
[Huawei-GigabitEthernet0/0/1] quit
```
