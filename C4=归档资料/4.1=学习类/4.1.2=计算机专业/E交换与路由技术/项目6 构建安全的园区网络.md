# 项目 6：构建安全的园区网络 (自考 13833 大纲深度版)

## 考点 1：实现计算机的安全接入 (Port Security)

### 1.1 识记：MAC 地址类型与生命周期
在开启端口安全（Port Security）后，交换机处理 MAC 地址的方式分为以下三类：
*   **安全动态 MAC 地址 (Secure Dynamic MAC)**：
    *   **学习方式**：使能端口安全后，接口根据流量自动学习。存储在 **MAC 地址表 (CAM 表)** 中，**不进入配置文件**。
    *   **生存周期**：遵循老化时间；**设备重启或接口物理 Down 后立即消失**，必须重新学习。
*   **安全静态 MAC 地址 (Secure Static MAC)**：
    *   **学习方式**：管理员手动通过命令绑定指定端口、VLAN 和 MAC。保存在配置文件中。
    *   **生存周期**：**永不老化**，除非手动删除，设备重启不丢失。
*   **Sticky MAC 地址 (粘性 MAC - 核心考点)**：
    *   **核心逻辑**：动态学习与静态保存的“结合体”。自动将学习到的动态 MAC 转换为 Sticky MAC。
    *   **存储位置**：**保存配置（save）后进入配置文件**。
    *   **生存周期**：接口 Down 不丢失；**保存配置后重启不丢失**。手动删除或关闭功能时消失。

### 1.2 领会：端口安全作用与保护动作
*   **核心作用**：
    *   **抑制 MAC 地址泛洪攻击 (MAC Flooding)**：通过限制 `max-mac-num`（最大 MAC 数量），防止攻击者填满 CAM 表导致合法业务地址学习失败。
    *   **防止非授权接入**：锁定特定 MAC 到物理端口，防止私接设备。
*   **保护动作 (Violation Actions) 对比**：
    | 动作名称 | 数据转发状态 | 是否生成日志 | 计数器是否增加 | 接口物理状态 |
    | :--- | :--- | :--- | :--- | :--- |
    | **Protect (保护)** | **丢弃** | 否 | 否 | Up (正常) |
    | **Restrict (限制/必背)** | **丢弃** | **是 (SNMP Trap)** | **是** | Up (正常) |
    | **Shutdown (关闭)** | **丢弃** | **是** | **是** | **Down (Error-down)** |

### 1.3 应用：熟练配置与验证
*   **标准配置流程**：
    ```shell
    [SW1-G0/0/1] port-security enable             # 使能端口安全 (必备前提)
    [SW1-G0/0/1] port-security max-mac-num 1      # 限制最大学习数量为1
    [SW1-G0/0/1] port-security mac-address sticky # 开启粘性MAC自动转换
    [SW1-G0/0/1] port-security protect-action restrict # 设置违规动作为限制
    [SW1-G0/0/1] port-security aging-time 10      # (可选) 设置老化时间10分钟
    ```
*   **验证与排障命令**：
    *   `display port-security interface G0/0/1`：查看端口安全总体配置（Port Status, Violation Counter）。
    *   `display mac-address security`：查看安全 MAC 地址表（查看 Type 为 Sticky 的表项）。
    *   **恢复 Error-down 接口**（针对 Shutdown 模式）：在接口下执行 `restart` 或 `shutdown` / `undo shutdown`。

## 考点 2：网络设备的远程管理 (管理平面安全)

### 2.1 远程管理协议与验证
*   **【识记】协议类型**：
    *   **Telnet**：TCP 23 端口，明文传输（不安全，易被抓包盗取密码）。
    *   **STelnet (SSH)**：TCP 22 端口，加密传输（安全性高，推荐使用）。
*   **【领会】验证方式对比**：
    *   **Password 验证**：仅需验证密码即可进入。
    *   **AAA 验证 (重点)**：需要验证用户名 + 密码。支持审计和权限分级（0-15 级，**15 为最高管理级**）。
*   **【应用】核心配置命令 (以 AAA 为例)**：
    ```shell
    [R1] aaa
    [R1-aaa] local-user admin password cipher Huawei@123
    [R1-aaa] local-user admin service-type telnet    # 授权Telnet
    [R1-aaa] local-user admin privilege level 15     # 授权最高等级15
    [R1] user-interface vty 0 4                     # 进入虚拟终端线路
    [R1-ui-vty0-4] authentication-mode aaa           # 切换验证模式为AAA
    ```

## 考点 3：ACL 访问控制列表 (流量安全)

### 3.1 ACL 规则与匹配
*   **【识记/领会】匹配顺序**：遵循“**从小到大，命中即止**”原则。即先匹配 rule 5，只要匹配上一条，后面就不再检查。
*   **【识记】分类编号**：
    *   **基本 ACL**：编号 2000-2999。仅能依据 **源 IP 地址** 过滤流量。
    *   **高级 ACL (重点)**：编号 3000-3999。可依据 **源/目 IP、协议号、源/目端口号** 进行精细过滤。
*   **【应用】通配符掩码**：0 代表“严格匹配”，1 代表“忽略”。
    *   例如：`192.168.1.0 0.0.0.255` 表示匹配 192.168.1.0 整个网段。

### 3.2 ACL 应用位置原则 (领会)
*   **基本 ACL**：应尽量应用在 **靠近目的地** 的接口上。
*   **高级 ACL**：应尽量应用在 **靠近源端** 的接口上（为了提前过滤掉无效流量，节省网络带宽）。

## 考点 4：广域网认证安全与 NAT

### 4.1 PPP 认证协议 (识记/领会)
*   **PAP**：两次握手，**明文**发送密码。安全性低。
*   **CHAP (重点)**：**三次握手**，采用 **密文挑战** 方式。安全性高，因为密码原文不直接在链路上跑。

### 4.2 NAT 网络地址转换
*   **【识记】NAT 类型**：
    *   **静态 NAT (NAT Server)**：1对1映射。常用于将内网服务器发布到外网（如发布 Web 服务器）。
    *   **动态 NAPT (Easy-IP)**：多对1映射。利用路由器出接口 IP 让全公司人上网。
*   **【领会】NAT 的优点**：1. **节省公网 IP 地址**资源；2. **隐藏内部拓扑**，增强内网安全。
*   **【应用】Easy-IP 配置逻辑**：
    ```shell
    [R1] acl 2000
    [R1-acl-basic-2000] rule permit source 192.168.1.0 0.0.0.255 # 定义内网
    [R1] interface G0/0/1 (连接互联网的口)
    [R1-G0/0/1] nat outbound 2000                                # 应用NAT
    ```

## 考点 5：防火墙基础 (安全隔离)

### 5.1 安全区域 (Zone) 与优先级
*   **【识记】常用区域与固定优先级**：
    *   **Local (100)**：防火墙自己，优先级最高。
    *   **Trust (85)**：受信区域，通常指内网。
    *   **DMZ (50)**：服务器存放区，安全性中等。
    *   **Untrust (5)**：非受信区，通常指外网（互联网）。
*   **【领会】核心逻辑**：防火墙 **默认拒绝所有跨区域流量**。必须手动编写“安全策略（Security Policy）”才能允许数据通行。
