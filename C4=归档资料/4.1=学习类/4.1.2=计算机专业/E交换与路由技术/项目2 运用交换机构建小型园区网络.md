# 项目2 运用交换机构建小型园区网络 (深度备考版)

## 一、 交换机基础与 VLAN 技术 (必考识记)

### 1.1 交换机基础
*   **二层交换机**：工作在数据链路层，基于 **MAC 地址表** 进行数据帧的转发。
*   **管理方式**：初次配置通常通过 **Console 端口** 物理连接；后续可通过 Telnet/SSH 远程管理。

### 1.2 VLAN (虚拟局域网) 技术
*   **作用**：在物理网络上逻辑地划分出多个广播域。**主要目的是隔离广播风暴，提高网络安全性**。
*   **VLAN 帧格式**：在以太网帧中插入 **802.1Q** 标签（包含 VLAN ID，范围 1-4094）。
*   **划分方式**：最常用的是**基于端口划分**（将物理接口固定分配给某个 VLAN）。

---

## 二、 端口类型与跨交换机互访 (核心领会)

交换机接口处理带有 VLAN 标签的数据帧时，有严格的规则，这是考试和实操的重中之重！

### 2.1 Access 端口 (接入端口)
*   **连接对象**：通常连接**终端设备**（如 PC、打印机）。终端不认识带标签的帧。
*   **收发规则**：
    *   **接收**：收到 PC 发来的无标签帧，打上接口 PVID (默认VLAN) 标签。
    *   **发送**：将带有 PVID 标签的帧**剥离标签**后，发给 PC。
*   **配置命令**：
```shell
[SW1] vlan 10
[SW1] interface GigabitEthernet 0/0/1
[SW1-G0/0/1] port link-type access
[SW1-G0/0/1] port default vlan 10
```

### 2.2 Trunk 端口 (干道端口)
*   **连接对象**：通常连接**交换机与交换机**、或**交换机与路由器**。
*   **收发规则**：一条 Trunk 链路可以允许多个 VLAN 的数据帧通过（保持标签不脱落）。
*   **配置命令**：
```shell
[SW1] interface GigabitEthernet 0/0/24
[SW1-G0/0/24] port link-type trunk
[SW1-G0/0/24] port trunk allow-pass vlan 10 20  # 允许VLAN 10和20通过 (或写 all)
```

## 三、 三层交换机与 VLAN 间路由 (重点应用)

### 3.1 二层与三层交换机的区别
*   **二层交换机**：只看 MAC 地址，只能解决同一 VLAN 内的通信。
*   **三层交换机**：具备路由功能（**“一次路由，多次交换”**），通过创建 **VLANIF 虚拟接口**，实现不同 VLAN 之间的互通。

### 3.2 核心配置命令
要让 VLAN 10 和 VLAN 20 互通，必须在三层交换机上配置它们的网关：
```shell
[Core-SW] vlan batch 10 20                # 批量创建VLAN
[Core-SW] interface Vlanif 10             # 进入VLAN 10的虚拟接口
[Core-SW-Vlanif10] ip address 192.168.10.254 24  # 配置VLAN 10的网关IP
[Core-SW] interface Vlanif 20
[Core-SW-Vlanif20] ip address 192.168.20.254 24
```

## 四、 骨干网络优化：链路聚合与生成树 (难点应用)

### 4.1 链路聚合 (Eth-Trunk)
*   **作用**：将多个物理接口捆绑成一个逻辑接口。**增加带宽、提供冗余备份**。
*   **配置命令**：
```shell
[SW1] interface Eth-Trunk 1               # 创建聚合组1
[SW1-Eth-Trunk1] port link-type trunk     # 配置逻辑接口类型
[SW1-Eth-Trunk1] port trunk allow-pass vlan all
[SW1] interface GigabitEthernet 0/0/1
[SW1-G0/0/1] eth-trunk 1                  # 将物理接口加入聚合组
[SW1] interface GigabitEthernet 0/0/2
[SW1-G0/0/2] eth-trunk 1
```

### 4.2 生成树协议 (STP/RSTP)
*   **环路危害**：交换机之间连线形成环路会导致**广播风暴**和 MAC 地址表震荡，瞬间瘫痪网络。
*   **STP/RSTP 作用**：通过阻塞冗余端口，打破物理环路，构建无环的逻辑树。当主链路故障时，被阻塞的端口会自动恢复。RSTP 是 STP 的快速版本。
*   **配置命令**：
```shell
[SW1] stp mode rstp                       # 开启RSTP模式 (华为默认是MSTP)
[SW1] stp priority 4096                   # 设置优先级，数值越小越优先成为根网桥 (必须是4096的倍数)
```

## 五、 高可靠性：VRRP (虚拟路由器冗余协议)

*   **作用**：解决单点故障。将两台三层交换机虚拟成一台“虚拟路由器”。PC 的网关指向虚拟 IP，主交换机挂了，备交换机瞬间接管，用户无感知。
*   **关键术语**：Master（主用设备）、Backup（备用设备）、Virtual IP（虚拟网关 IP）。
*   **配置命令 (在 Vlanif 接口下配置)**：
```shell
# 配置主网关 (Master)
[Core1-Vlanif10] vrrp vrid 1 virtual-ip 192.168.10.254 # 设虚拟网关IP
[Core1-Vlanif10] vrrp vrid 1 priority 120              # 设优先级(默认100，越大越优先)

# 配置备网关 (Backup)
[Core2-Vlanif10] vrrp vrid 1 virtual-ip 192.168.10.254 # 虚拟网关IP必须一致
[Core2-Vlanif10] vrrp vrid 1 priority 100              # 保持默认100，作为备用
```

## 六、 DHCP 技术 (在交换机上的应用)

*   在三层交换机上配置 DHCP，自动给 VLAN 内的 PC 分配 IP。
*   **配置步骤**：
```shell
[Core-SW] dhcp enable                     # 必须先全局开启DHCP功能
[Core-SW] ip pool vlan10_pool             # 创建全局地址池
[Core-SW-ip-pool-vlan10_pool] network 192.168.10.0 mask 24
[Core-SW-ip-pool-vlan10_pool] gateway-list 192.168.10.254
[Core-SW-ip-pool-vlan10_pool] dns-list 8.8.8.8
[Core-SW] interface Vlanif 10
[Core-SW-Vlanif10] dhcp select global     # 接口下调用全局地址池
```

## 七、 考纲重点提炼：哪些必须背？

### 1. 核心理论必背
*   **VLAN 的核心价值**：隔离广播域。
*   **Access 与 Trunk 的收发剥洋葱规则**：Access 出口必须脱标签，Trunk 出口保留标签。
*   **二层与三层交换机的区别**：二层看 MAC，三层看 IP；三层交换机利用 Vlanif 接口实现跨 VLAN 通信。

### 2. 必背命令关键字
*   `port link-type access/trunk` (端口类型)。
*   `port trunk allow-pass vlan` (Trunk 放行)。
*   `interface Vlanif` (创建三层虚拟网关)。
*   `interface Eth-Trunk` (链路聚合绑定)。
*   `vrrp vrid 1 virtual-ip` (VRRP 虚拟网关配置)。
*   `dhcp select global` (调用 DHCP 全局池)。

### 3. 常见排错 Checklist
*   **PC 无法跨交换机通信**：检查中间相连的线是否配成了 Trunk，是否放行了对应的 VLAN。
*   **不同 VLAN 无法通信**：检查三层交换机上是否创建了对应的 `Vlanif` 接口，PC 的网关是否正确指向该 `Vlanif` 的 IP。
*   **DHCP 获取不到 IP**：检查交换机是否敲了 `dhcp enable`，接口下是否敲了 `dhcp select`。
