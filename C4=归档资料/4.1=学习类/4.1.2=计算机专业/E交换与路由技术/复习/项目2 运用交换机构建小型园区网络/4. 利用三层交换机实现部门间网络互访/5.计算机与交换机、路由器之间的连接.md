在 eNSP 拓扑搭建或真实机房布线中，**“连对线、配对接口”** 是所有实验成功的基石。我们需要从物理介质和逻辑配置两个维度来掌握计算机、交换机与路由器之间的“交互礼仪”。

### 1. 物理连接：直通线还是交叉线？

虽然现代华为设备具备接口自动识别功能，但在理论考试中，这依然是高频选择题：

- **直通线（Straight-through）**：连接**不同种**设备。
    - PC ↔ 交换机
    - 交换机 ↔ 路由器
- **交叉线（Cross-over）**：连接**同种**设备。
    - PC ↔ PC
    - 交换机 ↔ 交换机
    - 路由器 ↔ 路由器
    - **特例**：PC 与路由器连接也建议使用交叉线（因为它们的接口在逻辑上都属于“三层口”）。

### 2. 逻辑连接的三要素：IP、子网、网关

无论物理上怎么连，逻辑上必须遵循以下准则：

- **PC 连交换机**：
    - 交换机侧：端口必须设为 **Access** 模式并划入对应的 VLAN。
    - PC 侧：IP 地址必须与交换机的 **VLANIF 接口（网关）** 处于同一网段。
- **交换机连交换机**：
    - 级联端口必须设为 **Trunk** 模式，并放行需要透传的 VLAN。
- **交换机连路由器**：
    - 若交换机是二层的：连接路由器的口通常设为 Trunk，配合路由器做“单臂路由”。
    - 若交换机是三层的：可以设为 Trunk 连路由器，或者在交换机上专门划出一个 VLAN 互连接口与路由器对接。

### 3. 核心应用场景：网关到底在哪？

这是初学者最容易犯错的地方。**网关（Gateway）** 永远是数据包跨出本网段的“第一站”。

- **二层组网**：网关在**路由器**的物理子接口上（单臂路由）。
- **三层组网**：网关在**三层交换机**的 **VLANIF** 接口上。
- **配置要点**：PC 的“默认网关”地址必须与该 VLANIF 的 IP 地址完全一致。

### 4. 考点挑战：实战分析

**场景描述**：
公司有 2 个部门，VLAN 10 (192.168.10.0/24) 和 VLAN 20 (192.168.20.0/24)。现在使用一台 **S5700 三层交换机** 作为核心，连接两台 PC。

**任务**：
1. PC1 应连在交换机的什么类型端口上？
2. 交换机上起网关作用的逻辑接口名称是什么？
3. 请写出使 PC1 能够访问 PC2 的关键配置命令。

**标准答案：**
1.  **Access 端口**。
2.  **VLANIF 10** 和 **VLANIF 20**。
3.  **核心配置命令**：
    ```bash
    # 1. 创建 VLAN
    [SW1] vlan batch 10 20
    # 2. 配置接口
    [SW1] interface GigabitEthernet 0/0/1
    [SW1-GigabitEthernet0/0/1] port link-type access
    [SW1-GigabitEthernet0/0/1] port default vlan 10
    # 3. 配置三层网关
    [SW1] interface Vlanif 10
    [SW1-Vlanif10] ip address 192.168.10.254 24
    [SW1] interface Vlanif 20
    [SW1-Vlanif20] ip address 192.168.20.254 24
    ```

---

**实战逻辑总结**

在 eNSP 实验中，连接完成后一定要执行 `display ip interface brief`。
- 如果 VLANIF 状态是 **Down** 的，说明该 VLAN 内没有处于 Up 状态的物理接口。
- **排错秘籍**：PC 连交换机，PC 必须开启（物理口 Up），对应的 VLANIF 才会变 Up。

理解了设备间的互联，你是否想在 eNSP 里尝试一个“混合挑战”：用一台路由器连接两台三层交换机，让三台设备背后的四个网段全部互通？这不仅考验你的接口连接能力，更考验你对网关分布的终极理解。
