在实操综合大题中，正确配置 OSPFv 3 是必考的压轴项目。它的配置逻辑有着极其严格的顺序要求，只要你在脑海中牢牢刻下**“全局开 IPv 6 ➔ 进程配 Router-ID ➔ 接口绑区域”**这三步曲，这道题的分数就稳稳收入囊中。

假设我们有一个最经典的考试拓扑：**R 1** 和 **R 2** 通过 `GigabitEthernet 0/0/0` 互联（网段 `2001:DB8:12::/64`），并且 R 1 的 `GigabitEthernet 0/0/1` 下挂了一个局域网（网段 `2001:DB8:1::/64`）。

以下是华为 VRP 系统的满分标准配置流：

### 第一阶段：核心配置（以 R 1 为例）

Plaintext

```
<Huawei> system-view
[Huawei] sysname R1

! 【第一步：全局开启 IPv6】
! 阅卷老师抓的第一个采分点，如果不敲这句，后面的 IPv6 命令全部无法识别。
[R1] ipv6

! 【第二步：启动 OSPFv3 并手工指定 Router-ID】
[R1] ospfv3 1
[R1-ospfv3-1] router-id 1.1.1.1                ! 绝对红线：哪怕是纯 IPv6 网络，Router-ID 也必须是 32 位点分十进制格式！
[R1-ospfv3-1] quit

! 【第三步：进入互联接口，配置 IP 并宣告进 OSPFv3】
[R1] interface GigabitEthernet 0/0/0
[R1-GigabitEthernet0/0/0] ipv6 enable                                  ! 开启接口的 IPv6 能力（必敲，否则无法生成 FE80:: 地址）
[R1-GigabitEthernet0/0/0] ipv6 address 2001:db8:12::1 64               ! 配置互联全球单播地址
[R1-GigabitEthernet0/0/0] ospfv3 1 area 0                              ! 核心巨变：直接在接口下将其宣告进骨干区域 0
[R1-GigabitEthernet0/0/0] quit

! 【第四步：进入局域网接口，将其也宣告进 OSPFv3】
! 只有把局域网接口也宣告进 OSPFv3，R2 才能学到通往该局域网的路由。
[R1] interface GigabitEthernet 0/0/1
[R1-GigabitEthernet0/0/1] ipv6 enable
[R1-GigabitEthernet0/0/1] ipv6 address 2001:db8:1::254 64
[R1-GigabitEthernet0/0/1] ospfv3 1 area 0
[R1-GigabitEthernet0/0/1] quit
```

_(对端 R 2 的配置逻辑完全镜像对称，只需将 Router-ID 改为 `2.2.2.2`，并将互联接口 IP 设为 `2001:db8:12::2 64` 即可。)_

---

### 第二阶段：考场查漏排错“三板斧”

配置完成后，绝对不能直接交卷，必须用以下三条命令验证配置是否真正生效，这也是排错题最常考察的查看命令：

**1. 检查邻居状态（定生死）**

Plaintext

```
[R1] display ospfv3 peer
```

- **满分确认：** 在输出结果中，`State` 字段必须显示为 **`Full`**。如果是 `Init` 或空无一物，立刻回去检查两边的 `Router-ID` 是否冲突，或者接口有没有敲 `ipv6 enable`。
    

**2. 检查全局 IPv 6 路由表（看结果）**

Plaintext

```
[R1] display ipv6 routing-table protocol ospfv3
```

- **满分确认：** 你应该能在这里看到对端（R 2）下挂的网段。注意，这里的 `NextHop`（下一跳）必然是对端接口的 **链路本地地址（`FE80::` 开头）**，这是 OSPFv 3 的正常表现，千万别以为自己配错了。
    

**3. 检查 OSPFv 3 接口宣告状态（查遗漏）**

Plaintext

```
[R1] display ospfv3 interface
```

- **满分确认：** 确认 `G0/0/0` 和 `G0/0/1` 都在列表中，并且已经成功选举出了 DR 或 BDR。如果某个接口没在这个列表里，说明你忘了进那个接口敲 `ospfv3 1 area 0`。
