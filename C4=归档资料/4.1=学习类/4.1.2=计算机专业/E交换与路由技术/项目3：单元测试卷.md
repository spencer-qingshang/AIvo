## 附：参考答案与详细解析 (项目 3)

### 一、 选择题
1. **B** 【解析】路由器根据 IP 地址进行寻址和转发，属于 OSI 第三层（网络层）设备。
2. **A** 【解析】Console 接口是本地管理接口，用于初次配置或无法通过网络远程管理时的故障排查。
3. **B** 【解析】用户视图（<Huawei>）只能查看信息，执行 `system-view` 进入系统视图（[Huawei]）才能进行配置。
4. **D** 【解析】`display ip routing-table` 是路由器最核心的命令，用于显示当前有效的路由条目。
5. **C** 【解析】带小数点的接口号（如 .10）通常表示逻辑子接口，用于单臂路由或 VLAN 间路由。
6. **A** 【解析】子接口需要通过 `dot1q termination vid 10` 命令封装 802.1Q 协议并终结对应的 VLAN 标签。
7. **B** 【解析】默认情况下子接口不响应 ARP 请求。配置 `arp broadcast enable` 后，路由器才能处理来自该子接口网段的 ARP 广播。
8. **C** 【解析】全局地址池（Global Pool）在系统视图下定义，可以被路由器上任何使能了 `dhcp select global` 的接口调用。
9. **A** 【解析】华为设备中，静态路由的默认 Preference 为 60；OSPF 为 10；RIP 为 100。数值越小优先级越高。
10. **D** 【解析】默认路由（Default Route）用于匹配所有未知的目的网段，表示形式为全 0。
11. **B** 【解析】浮动路由通过配置两条目的相同但 Preference 不同的静态路由实现，高优先级的条目失效后，低优先级的条目会自动出现在路由表中。
12. **A** 【解析】RIP 属于距离矢量路由协议，其度量值（Metric）仅通过跳数计算，最大为 15。
13. **B** 【解析】RIPv1 是有类协议（广播发送），RIPv2 是无类协议（组播发送），支持子网划分（VLSM）和 CIDR。
14. **C** 【解析】RIP 协议中，15 跳为可达的最大值，16 跳则被定义为不可达。
15. **B** 【解析】OSPF（最短路径优先）是链路状态路由协议，路由器之间交换 LSA（链路状态通告）并建立拓扑图。
16. **A** 【解析】OSPF 采用两级区域架构，Area 0 是骨干区域，所有非骨干区域（非 0 区域）必须连接到 Area 0 才能互通。
17. **B** 【解析】OSPF 宣告网络使用“反掩码”（Wildcard Mask），255.255.255.0 对应的反掩码是 0.0.0.255。
18. **A** 【解析】手动配置的路由在路由表中标记为 Static；接口直接相连的标记为 Direct。
19. **B** 【解析】DHCP 服务器在分配地址前会先发送 ICMP Echo 请求（Ping），如果没有响应则认为该地址未被占用。
20. **D** 【解析】路由器的核心任务是根据路由协议建立路由表，并在不同子网（异构网络）间选择最优路径转发数据。

### 二、 填空题
21. **路由表** 【解析】路由表是路由器的“地图”，记录了目的网段、下一跳和出接口。
22. **用户** 【解析】用户视图提示符为 `< >`，是进入系统的第一个视图。
23. **接口 (或 interface)** 【解析】接口地址池直接使用接口本身的 IP 网段作为分配范围。
24. **子接口** 【解析】通过在物理接口上模拟出逻辑子接口，实现一根网线传输多个 VLAN 流量。
25. **不占用网络带宽 (或 安全性高)** 【解析】静态路由不需要像动态路由那样频繁交换路由信息报文。
26. **0.0.0.0/0** 【解析】表示所有网段。
27. **组播** 【解析】RIPv2 的组播地址是 224.0.0.9，相比 RIPv1 的广播更节省非运行 RIP 设备的 CPU。
28. **SPF (或 Dijkstra)** 【解析】最短路径优先（Shortest Path First）算法保证了网络内部无环路。
29. **Loopback** 【解析】环回接口是逻辑接口，状态极其稳定，最适合作为 Router ID。
30. **浮动路由** 【解析】主链路失效后的自动备份技术。

### 三、 简答题 (参考要点)
31. **工作原理**：解封装链路层 -> 查路由表（最长匹配原则） -> 找不到则丢弃，找到则重新封装（源/目MAC变化） -> 从出接口发出。
32. **单臂路由**：在路由器一个物理口配置多个子接口。解决了二层交换机划分 VLAN 后导致的广播域隔离、无法互通的问题。
33. **DHCP对比**：接口池配置快，范围小；全局池适合大型网络，可支持中继和更复杂的选项设置。
34. **路由对比**：静态适合出口或小网络，人工维护累但安全；动态适合大网络，自动学习但费资源。
35. **OSPF区域**：将大型网络分割，LSDB 变小，计算快。骨干区域是所有流量的汇聚中心，防区域间环路。

### 四、 应用题
36. **静态路由与浮动路由配置 (5分)**：
```shell
# 配置主路由（优先级默认60）
ip route-static 172.16.1.0 255.255.255.0 10.1.1.2
# 配置备份路由（优先级设为100，数值大则平时不生效）
ip route-static 172.16.1.0 255.255.255.0 20.1.1.2 preference 100
```

37. **OSPF 基础配置 (5分)**：
```shell
# 启动进程并设ID
ospf 1 router-id 1.1.1.1
# 进入骨干区域
 area 0
# 宣告网段（反掩码）
  network 192.168.10.0 0.0.0.255
```
