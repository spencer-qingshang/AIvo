在华为 VRP 系统中，**全局地址池（Global Address Pool）** 是 DHCP 的另一种核心配置方式。与之前提到的“接口地址池”不同，它是独立于接口存在的，具有更强的灵活性和管理范围。

以下是为你整理的硬核笔记，可以直接复制到你的 DHCP 专题中。

# 核心考点笔记：DHCP 全局地址池 (Global Address Pool)

## 一、什么是全局地址池？

全局地址池是在系统的全局视图下定义的 IP 地址集合。它不预先绑定到任何特定接口，而是由管理员手动创建一个“资源池”，并定义其属性（网段、网关、DNS 等）。

- **核心逻辑：** 路由器先在全局创建一个“水池”，然后在需要的接口上通过命令“调用”这个水池。
    
- **分配灵活性：** 一个全局地址池可以为多个不同的接口提供服务（只要逻辑匹配），也支持 **DHCP 中继（Relay）** 场景。
    
- **参数自定义：** 必须手动指定网段、子网掩码、网关和 DNS。
    

## 二、全局地址池 vs 接口地址池 (快速对比)

|**特性**|**全局地址池 (Global)**|**接口地址池 (Interface)**|
|---|---|---|
|**定义位置**|系统视图 `[Huawei]` |接口视图 `[Huawei-GE0/0/x]` |
|**配置复杂度**|较高，需手动指定所有参数|极简，自动继承接口参数|
|**网关配置**|手动命令 `gateway-list` |自动关联接口 IP|
|**适用场景**|大型网络、跨网段分配、DHCP 中继|小型简单网络、直连终端|
|**开启方式**| `dhcp select global` | `dhcp select interface` |

## 三、全局地址池必考配置代码 (华为 VRP)

假设你需要创建一个名为 `office_pool` 的地址池，为 `192.168.10.0/24` 网段的办公区提供 IP。

### 1. 第一步：全局开启 DHCP

Plaintext

```
[R1] dhcp enable
```

### 2. 第二步：创建并配置地址池属性

Plaintext

```
[R1] ip pool office_pool                     // 创建全局地址池，命名为 office_pool
[R1-ip-pool-office_pool] network 192.168.10.0 mask 24     // 设置地址池网段和掩码
[R1-ip-pool-office_pool] gateway-list 192.168.10.254      // 设置分配给客户端的默认网关
[R1-ip-pool-office_pool] dns-list 114.114.114.114 8.8.8.8 // 设置分配给客户端的 DNS
[R1-ip-pool-office_pool] excluded-ip-address 192.168.10.1 192.168.10.50 // 排除保留地址
[R1-ip-pool-office_pool] lease day 8                      // 设置租期为 8 天
[R1-ip-pool-office_pool] quit
```

### 3. 第三步：在接口下调用全局地址池

只有在接口下明确“调用”，该接口接收到的 DHCP Discover 报文才会去全局池子里查找 IP。

Plaintext

```
[R1] interface GigabitEthernet 0/0/1
[R1-GigabitEthernet0/0/1] ip address 192.168.10.254 24    // 接口 IP 必须与池子网段一致
[R1-GigabitEthernet0/0/1] dhcp select global              // 核心命令：关联全局地址池
```

## 四、核心查错命令 (Troubleshooting)

在笔记末尾建议记下这几条高频维护命令：

- **`display ip pool name office_pool`**：查看特定全局地址池的详细使用情况（包含已用地址、冲突地址、剩余地址）。
    
- **`display ip pool name office_pool used`**：查看哪些 MAC 地址租用了哪些 IP。
    
- **`display ip pool name office_pool conflict`**：查看网络中发生了 IP 冲突的记录。
