在掌握了 VLAN 的工作原理后，接下来的核心任务是：**如何在交换机上通过命令行（CLI）实现这些逻辑？**

在华为 VRP 平台上，配置接口类型的基本逻辑是“先定性，再定量”——即先定义接口的链路类型，再指定其所属或允许的 VLAN。

### 1. Access 端口配置：连接终端的“最后一百米”

Access 端口通常连接 PC。配置时，你需要明确告诉交换机：这个接口属于哪个“小隔间”。

**配置步骤：**
1.  进入接口视图：`interface GigabitEthernet 0/0/1`
2.  **定性**（设为 Access）：`port link-type access`
3.  **定量**（划入 VLAN）：`port default vlan 10`

> **考试提醒**：在将接口划入 VLAN 之前，必须先在全局模式下创建该 VLAN（命令：`vlan 10`），否则系统会报错。

### 2. Trunk 端口配置：设备间的“通信主动脉”

Trunk 端口连接交换机或路由器。由于它要承载多个 VLAN 的流量，配置命令稍微复杂一些。

**配置步骤：**
1.  进入接口视图：`interface GigabitEthernet 0/0/24`
2.  **定性**（设为 Trunk）：`port link-type trunk`
3.  **放行**（核心步骤）：`port trunk allow-pass vlan 10 20` 
    - 此命令指定哪些 VLAN 的数据帧可以带着标签通过。
    - 若要允许所有 VLAN：`port trunk allow-pass vlan all`
4.  **修改 PVID**（可选/进阶）：`port trunk pvid vlan 10`
    - 修改该口在收到不带标签帧时的默认归属。

### 3. 验证命令：如何检查配置是否生效？

在 eNSP 实验中，配置完后务必执行以下命令查看结果：

- **查看所有 VLAN 及成员接口**：
  `display vlan`
  （通过此命令可以直观看到哪些端口是 Access 成员，哪些是 Trunk 成员。）

- **查看特定接口的详细 VLAN 属性**：
  `display port vlan`
  （这是排错的神器，能清楚看到端口的 **PVID** 和 **Allow List**。）

### 4. 常见的配置误区与“坑”

- **顺序错误**：如果你试图在一个已经是 Trunk 的接口上直接敲 `port default vlan`，或者在 Access 口上敲 `port trunk allow-pass`，系统会提示命令冲突。你必须先用 `undo` 恢复默认，或者直接改变 `link-type`。
- **默认 VLAN 1**：所有华为交换机接口默认都是 `link-type hybrid` 且属于 `vlan 1`。在考试中，如果题目没要求，通常需要将其改为更明确的 Access 或 Trunk。

**考点与实战的逻辑总结**

在综合实训题中，配置流程通常是：
**创建 VLAN** -> **配置 Access 口（连 PC）** -> **配置 Trunk 口（连对端交换机）** -> **Ping 测试**。

你可以尝试在 eNSP 里完成这样一个挑战，并对照下方的**标准答案**验证你的理解：

1. 创建 VLAN 10 和 20。
2. 将 GE0/0/1 设为 Access 并归入 VLAN 10。
3. 将 GE0/0/2 设为 Trunk 并只允许 VLAN 10 通过。
4. **思考**：此时连接在 GE0/0/1 的 PC 能 Ping 通对端交换机下同样在 VLAN 10 的 PC 吗？如果是 VLAN 20 呢？

---

### 5. 挑战解答与命令实现

针对上述实验挑战，以下是标准的操作命令及逻辑分析：

**操作命令（以 SW1 为例）：**
1.  **创建 VLAN**：
    ```bash
    [SW1] vlan batch 10 20
    ```
2.  **配置 Access 端口**：
    ```bash
    [SW1] interface GigabitEthernet 0/0/1
    [SW1-GigabitEthernet0/0/1] port link-type access
    [SW1-GigabitEthernet0/0/1] port default vlan 10
    ```
3.  **配置 Trunk 端口（仅放行 VLAN 10）**：
    ```bash
    [SW1] interface GigabitEthernet 0/0/2
    [SW1-GigabitEthernet0/0/2] port link-type trunk
    [SW1-GigabitEthernet0/0/2] port trunk allow-pass vlan 10
    # 注意：华为交换机默认 VLAN 1 也是放行的，除非手动执行 undo 命令。
    ```

**思考题解答：**
- **VLAN 10 的 PC**：**可以 Ping 通**。
  - **理由**：数据帧从 Access 口进入 SW1 被打上 VLAN 10 标签，到达 Trunk 出口时，交换机检查 `allow-pass` 列表，确认 VLAN 10 在允许范围内，于是带着标签转发。对端 SW2 收到后剥离标签发给对应的 Access 口，链路打通。
- **VLAN 20 的 PC**：**无法 Ping 通**。
  - **理由**：虽然 SW1 内部可以处理 VLAN 20，但在跨越 Trunk 链路时，由于 `allow-pass` 列表里**没有** VLAN 20，交换机会在出接口处直接将该帧丢弃（Drop），导致通信失败。

---

**考点与实战的逻辑总结**

在综合实训题中，配置流程通常是：
**创建 VLAN** -> **配置 Access 口（连 PC）** -> **配置 Trunk 口（连对端交换机）** -> **Ping 测试**。

通过这种“有选择性放行”的实验，你能更深刻理解 `allow-pass` 命令在控制园区网络拓扑结构中的实战意义。如果你想进一步验证，可以尝试在 Trunk 口敲下 `undo port trunk allow-pass vlan 1`，看看原本默认连通的管理流量会发生什么？
