三层 WLAN 组网的配置逻辑，本质上就是：**二层配置 + 底层路由打通 + DHCP Option 43 指路**。

为了在综合大题中拿满分，请在脑海中牢牢刻下这套标准的**“满分四步曲”**代码流（以旁挂式+直接转发为例）：

### 第一步：打通底层路由与 DHCP（网关设备配置）

这是三层组网与二层组网**唯一也是最大的区别**。AP 和 AC 不在同一个网段，必须靠 DHCP 的 Option 43 告诉 AP 去哪找 AC，且底层的路由必须通。

Plaintext

```
! 1. 假设核心交换机作为 AP (VLAN 10) 和 用户 (VLAN 20) 的网关
<Core_SW> system-view
[Core_SW] dhcp enable

! 2. 配置 AP 的地址池（核心重头戏：Option 43）
[Core_SW] ip pool Pool_AP
[Core_SW-ip-pool-Pool_AP] network 192.168.10.0 mask 24
[Core_SW-ip-pool-Pool_AP] gateway-list 192.168.10.254
! 👇 三层组网的灵魂所在！告诉 AP 去找 AC（假设 AC 的 IP 是 10.1.1.1）
[Core_SW-ip-pool-Pool_AP] option 43 sub-option 2 ip-address 10.1.1.1 
[Core_SW-ip-pool-Pool_AP] quit

! 3. 在网关接口调用全局地址池
[Core_SW] interface Vlanif 10
[Core_SW-Vlanif10] ip address 192.168.10.254 24
[Core_SW-Vlanif10] dhcp select global
[Core_SW-Vlanif10] quit

! 4. 确保底层路由可达（非常重要，考卷上通常需要配 OSPF 或静态路由连通 AC）
! AP 拿到 IP 后，必须能顺着路由 Ping 通 10.1.1.1
```

---

### 第二步：宣告 AC 源接口与 AP 注册（AC 设备配置）

网络通了，现在轮到 AC 做好接待准备。

Plaintext

```
! 1. 宣告建立 CAPWAP 隧道的源接口（AC 必须声明自己用哪个 IP 和 AP 对话）
<AC> system-view
[AC] capwap source interface Vlanif 100   ! 假设 Vlanif 100 配了 10.1.1.1

! 2. 进入 WLAN 视图并离线添加 AP
[AC] wlan
[AC-wlan-view] ap auth-mode mac-auth      ! 开启 MAC 认证
[AC-wlan-view] ap-id 1 ap-mac 00e0-fc12-3456
[AC-wlan-ap-1] ap-name Office-AP1
[AC-wlan-ap-1] quit
```

---

### 第三步：组装三大业务模板（AC 设备的 WLAN 视图）

这段配置和二层组网完全一样，保持“安全 ➔ SSID ➔ VAP”的流水线顺序。

Plaintext

```
! 1. 安全模板（设密码）
[AC-wlan-view] security-profile name Sec_Profile
[AC-wlan-sec-prof-Sec_Profile] security wpa2 psk pass-phrase Huawei@123 aes
[AC-wlan-sec-prof-Sec_Profile] quit

! 2. SSID 模板（设 Wi-Fi 名字）
[AC-wlan-view] ssid-profile name SSID_Profile
[AC-wlan-ssid-prof-SSID_Profile] ssid Company-WiFi
[AC-wlan-ssid-prof-SSID_Profile] quit

! 3. VAP 模板（终极融合）
[AC-wlan-view] vap-profile name VAP_Profile
[AC-wlan-vap-prof-VAP_Profile] forward-mode direct-forward  ! 大题通常要求直接转发
[AC-wlan-vap-prof-VAP_Profile] service-vlan vlan-id 20      ! 绑定用户的业务 VLAN
[AC-wlan-vap-prof-VAP_Profile] security-profile Sec_Profile
[AC-wlan-vap-prof-VAP_Profile] ssid-profile SSID_Profile
[AC-wlan-vap-prof-VAP_Profile] quit
```

---

### 第四步：下发业务到 AP 射频

最后一步，将做好的 VAP 模板挂载到 AP 的天线上进行发射。

Plaintext

```
[AC-wlan-view] ap-group name default
! 👇 将 VAP_Profile 绑定到 2.4G 和 5G 天线上
[AC-wlan-ap-group-default] vap-profile VAP_Profile wlan 1 radio all
[AC-wlan-ap-group-default] quit
```

---

### 🌟 考场终极排错思路（三层专属）

如果在大题中，AP 死活不亮（不上线），立刻排查这三个“三层专属死穴”：

1. **Option 43 查漏：** 去 DHCP 服务器（通常是核心交换机）的 `ip pool` 里，看有没有敲错 AC 的 IP，或者漏了 `sub-option 2`。
    
2. **底层路由查通：** 在汇聚/核心交换机上，看能不能 `ping` 通 AC 的 IP 且能 `ping` 通 AP 获取到的 IP。如果 OSPF 忘了宣告网段，隧道绝对建不起来。
    
3. **接口放行查错：** 由于是直接转发，连接 AP 的那个最底层的接入交换机接口，它的 Trunk/Hybrid 必须**同时放行 AP 的 VLAN 10 和用户的 VLAN 20**。
