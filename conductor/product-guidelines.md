# AIvo 笔记编写与管理准则

## 1. 文体与叙述风格
- **学术严谨性：** 对于自考课程中的核心概念、法律条文或算法逻辑，必须严格遵循教材定义的术语，确保复习时的专业性。
- **结构化表达：** 拒绝长篇累牍。笔记主体应采用层级列表（Bullet points）、对比表格或大纲形式，核心知识点必须清晰独立。

## 2. 知识组织逻辑
- **层级归档：** 遵循现有的 `C1-C5` 文件夹体系，确保每篇新笔记都有明确的学科归属。
- **双向链接：** 积极利用 `[[笔记名]]` 语法。当一个知识点（如 Java 集合）涉及其他领域（如数据结构）时，必须建立逻辑关联。
- **元数据标签：** 统一使用功能性标签，例如：
  - `#真题`：标记历年考点。
  - `#重点`：标记需要强制背诵的内容。
  - `#错题`：标记在模拟练习中失分的知识点。

## 3. 任务执行与复盘
- **每日反馈循环：** 在当天的“日程”或“日记”笔记末尾，必须包含一个【今日复盘】板块。
- **记录内容：** 简要记录学习任务的完成度、未完成的原因、以及在自考知识理解上取得的突破或遇到的瓶颈。

## 4. 格式与视觉呈现
- **Callouts 强化：** 使用 Obsidian 的标注语法突出显示特定信息：
  - `> [!abstract] 核心定义`：用于教材原始定义。
  - `> [!bug] 避坑指南`：用于易错考点或解题陷阱。
  - `> [!quote] 考试套路`：总结历年真题的出题规律。
- **多维可视化：** 
  - 复杂逻辑优先使用 Mermaid 流程图或脑图展现。
  - 库内已有的 MarkMind 脑图应作为学科知识体系的索引。
- **重点高亮：** 段落中的关键词必须使用 **加粗** 或 ==高亮== 进行强调，确保扫视时能抓住重心。
