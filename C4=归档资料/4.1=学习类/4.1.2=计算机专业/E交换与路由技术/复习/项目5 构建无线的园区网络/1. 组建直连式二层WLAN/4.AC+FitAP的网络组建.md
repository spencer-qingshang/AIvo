今天是 4 月 6 日，距离《交换与路由技术》自考开考只剩几天的时间了！在华为 VRP 体系中，**AC + Fit AP 的网络组建**是综合实操大题中的“终极 BOSS”。

不要被长串的命令吓倒，这道题的得分秘诀在于**“逻辑框架”**。只要你在脑海里把组网分为标准的“四步曲”，这道大题就能稳拿高分。

我们以最主流的**“旁挂式组网 + 直接转发（本地转发）”**为例，来梳理这套必背的配置逻辑：

---

### 第一步：打通底层网络与 DHCP (修桥铺路)

瘦 AP 出厂就像个“白痴”，它第一件事就是要找 DHCP 服务器要个 IP 地址，并打听 AC 的地址。

**核心考点与命令：**

1. **划分 VLAN：** 通常将管理流量（AP 的 IP）和业务流量（用户的 IP）分开。比如 VLAN 10 给 AP 用，VLAN 20 给无线用户用。
    
2. **配置 DHCP 与 Option 43：** 极其重要！AP 拿到 IP 后，怎么知道 AC 在哪？全靠 DHCP 里的 `option 43` 字段告诉它。
    

Plaintext

```
! 在核心交换机或 DHCP 服务器上配置（给 AP 分配 IP，并告知 AC 地址）
[Huawei] dhcp enable
[Huawei] ip pool for-ap
[Huawei-ip-pool-for-ap] network 192.168.10.0 mask 255.255.255.0  ! AP 的管理网段
[Huawei-ip-pool-for-ap] gateway-list 192.168.10.254
[Huawei-ip-pool-for-ap] option 43 sub-option 2 ip-address 10.1.1.1 ! 核心重点：10.1.1.1 就是 AC 的管理 IP，指引 AP 去找 AC
[Huawei-ip-pool-for-ap] quit
```

---

### 第二步：配置 AC 基础参数与 AP 上线 (建交认亲)

这一步要在 AC 控制器上操作。你需要告诉 AC：“你是谁？允许哪些 AP 来找你？”

**核心考点与命令：**

1. **指定 CAPWAP 源接口：** 告诉 AC 用哪个 IP 去和 AP 建立隧道。
    
2. **配置 AP 认证模式：** 通常采用 MAC 认证，防止别人拿着来历不明的 AP 接入公司网络。
    

Plaintext

```
! 在 AC 上配置
[AC] wlan                                         ! 进入 WLAN 视图
[AC-wlan-view] ap auth-mode mac-auth              ! 设置 AP 认证模式为 MAC 认证
[AC-wlan-view] ap-id 0 ap-mac 00e0-fc12-3456      ! 录入合法 AP 的 MAC 地址，并编号为 0
[AC-wlan-ap-0] ap-name Office-AP1                 ! 给这个 AP 起个好记的名字
[AC-wlan-ap-0] quit
[AC] capwap source interface Vlanif 100           ! 极其容易漏掉！指定 AC 建立隧道的源接口（假设 Vlanif100 的 IP 是 10.1.1.1）
```

---

### 第三步：配置 WLAN 业务模板 (制定规矩)

这是最繁琐也是最容易出填空题的地方。AC 采用“模板化”管理，你要依次创建安全模板、SSID 模板，最后把它们融合成一个 VAP（虚拟 AP）模板。

**核心考点与命令：**在 AC 的 `wlan` 视图下逐一创建：

Plaintext

```
[AC] wlan
! 1. 创建安全模板（设密码）
[AC-wlan-view] security-profile name sec-emp
[AC-wlan-sec-prof-sec-emp] security wpa2 psk pass-phrase Huawei@123 aes  ! 设置 WPA2 密码
[AC-wlan-sec-prof-sec-emp] quit

! 2. 创建 SSID 模板（设 Wi-Fi 名字）
[AC-wlan-view] ssid-profile name ssid-emp
[AC-wlan-ssid-prof-ssid-emp] ssid Company-WiFi    ! 手机上搜到的就是这个名字
[AC-wlan-ssid-prof-ssid-emp] quit

! 3. 创建 VAP 模板（终极融合）
[AC-wlan-view] vap-profile name vap-emp
[AC-wlan-vap-prof-vap-emp] forward-mode direct-forward   ! 考试压轴题常考：设为直接转发（本地转发）模式
[AC-wlan-vap-prof-vap-emp] service-vlan vlan-id 20       ! 指定无线用户连上后进入 VLAN 20
[AC-wlan-vap-prof-vap-emp] security-profile sec-emp      ! 绑定刚才的安全模板
[AC-wlan-vap-prof-vap-emp] ssid-profile ssid-emp         ! 绑定刚才的 SSID 模板
[AC-wlan-vap-prof-vap-emp] quit
```

---

### 第四步：将模板绑定到 AP 射频 (发射信号)

规矩定好了，最后一步就是把这套“规矩”（VAP 模板）下发给具体的 AP，让它的天线（Radio）开始发射信号！

Plaintext

```
[AC] wlan
[AC-wlan-view] ap-group name default-group               ! 进入 AP 组（新上线的 AP 默认都在 default-group 里）
[AC-wlan-ap-group-default-group] vap-profile vap-emp wlan 1 radio all ! 将 VAP 模板绑定到所有射频（2.4G 和 5G 天线都发射这个 WiFi）
[AC-wlan-ap-group-default-group] quit
```

---

### 考场终极排错口诀

如果在考场模拟器上做完这四步，手机还是连不上 Wi-Fi，立刻默念这“排错三板斧”：

1. **查 AP 状态：** 在 AC 上敲 `display ap all`。如果 State 不是 **`nor`** (Normal)，说明 AP 连 IP 都没拿到或者 MAC 认证没过。重点查第一步和第二步。
    
2. **查 CAPWAP 源：** 检查 AC 上到底有没有敲 `capwap source interface`，没有这句隧道绝对建不起来。
    
3. **查转发模式连通性：** 如果是直接转发（本地转发），连接 AP 的那台接入交换机接口，必须要放行管理 VLAN（打 Tag/Untag）和业务 VLAN（必须打 Tag），千万别忘了配 Trunk。
