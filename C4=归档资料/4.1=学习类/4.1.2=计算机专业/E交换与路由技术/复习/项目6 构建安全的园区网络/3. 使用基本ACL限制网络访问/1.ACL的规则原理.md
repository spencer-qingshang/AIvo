晚上好！看来你已经进入了夜间的黄金复习状态。ACL（访问控制列表，Access Control List）可以说是《交换与路由技术》里最核心的“保安队长”。理解它的规则原理，其实就是搞懂这位保安是如何拿着手里的“黑白名单”来查验过往数据包的。

在考卷的排错题和综合配置题中，ACL 的得分关键全部集中在它的**四大核心匹配原理**上：

### 一、原则一：自上而下，严格按序匹配

ACL 并不是一整段话，而是由一条条规则（Rule）组成的列表。每条规则都有一个**序列号（Sequence Number，也叫步长，华为默认步长为 5）**，比如 Rule 5、Rule 10、Rule 15。

- **工作机制：** 当一个数据包来到接口时，路由器会严格按照序列号**从小到大（自上而下）**的顺序，逐条用规则去核对这个数据包。
    
- **步长的意义：** 为什么默认是 5、10、15 而不是 1、2、3？这是为了方便网络工程师日后在两条规则之间**插队**补充新规则（比如在 5 和 10 之间插入一条 Rule 7），而不需要把底下的规则全部推翻重写。
    

### 二、原则二：命中即停止（First Match）🌟 考试最大陷阱

这是 ACL 逻辑的绝对核心，也是出题人最爱挖坑的地方。

- **工作机制：** 数据包在自上而下匹配的过程中，**只要命中了某一条规则（不管是允许 Permit 还是拒绝 Deny），路由器就会立刻执行动作，并马上停止匹配剩下的 ACL 规则。**
    
- **考场排错警示：** **范围越小、越精确的规则，必须放在越上面！**
    
    如果你把一条 `Rule 5 permit source any`（允许所有人通过）放在了最前面，那么后面的 `Rule 10 deny source 192.168.1.10`（拒绝某台特定电脑）将**永远不会生效**，因为那台电脑在 Rule 5 就已经被放行了，路由器根本不会去看 Rule 10。
    

### 三、原则三：通配符掩码（Wildcard Mask）的“反转逻辑”

在写 IP 地址范围时，ACL 使用的是通配符掩码，它的逻辑和我们配置 IP 时的子网掩码是**完全相反**的。

- **0 代表“严格匹配”（管得死死的）：** 对应位的 IP 数字必须一模一样。
    
- **1 代表“任意/忽略”（随它去）：** 对应位的 IP 数字爱是多少是多少。
    

**考卷常见格式速查表：**

|**需求场景**|**目标 IP 范围**|**ACL 通配符掩码写法**|**解释**|
|---|---|---|---|
|**匹配整个网段**| `192.168.1.0/24` 网段| `rule permit source 192.168.1.0 0.0.0.255` |前三个 0 表示 `192.168.1` 必须对齐，最后的 255 (全 1) 表示最后一位随意。|
|**精确匹配单台主机**|仅 `192.168.1.10` 这台电脑| `rule deny source 192.168.1.10 0.0.0.0` |四个 0 表示每一位都必须一模一样。在华为设备里，也可以简写为 `host 192.168.1.10`。|
|**匹配所有人**|全网任意设备| `rule permit source any` | `any` 就等同于 `0.0.0.0 255.255.255.255`，直接放行所有。|

### 四、原则四：隐含的默认动作（隐式规则）

如果一个数据包把 ACL 里的所有规则都对了一遍，发现**一条都没有命中**，最后会怎么样？

在华为 VRP 体系中，这个“兜底动作”取决于 ACL 被应用在了哪个模块：

- **应用在接口做包过滤（Traffic-filter）时：** 华为默认的兜底动作是 **Permit（允许通过）**。也就是说，没在名单上的人，默认全放行。
    
- **应用在远程登录（Telnet/SSH 的 VTY 接口）或路由策略时：** 默认的兜底动作是 **Deny（拒绝通过）**。没在名单上的，一律不准进。

### 💡 考场高分配置公式（以基本 ACL 为例）

Plaintext

```
<Huawei> system-view
! 1. 创建编号为 2000 的基本 ACL（编号 2000-2999 代表只能检查源 IP 地址）
[Huawei] acl 2000

! 2. 写入规则：拒绝 192.168.1.10 这台主机（由于精确匹配，建议优先写）
[Huawei-acl-basic-2000] rule 5 deny source 192.168.1.10 0.0.0.0

! 3. 写入规则：允许 192.168.1.0/24 网段的其他所有人
[Huawei-acl-basic-2000] rule 10 permit source 192.168.1.0 0.0.0.255
[Huawei-acl-basic-2000] quit

! 4. 将 ACL 应用到接口的入方向（Inbound）
[Huawei] interface GigabitEthernet 0/0/1
[Huawei-GigabitEthernet0/0/1] traffic-filter inbound acl 2000
```
