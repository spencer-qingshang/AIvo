进入到二层交换与三层路由的交汇处，**单臂路由（Router-on-a-Stick）** 绝对是自考和真实网络工程中的一座“大山”。

简单来说，**单臂路由是解决“不同 VLAN 之间如何通信”的经典技术。**

在之前学习中我们知道：

1. 路由器是为了连接**不同网段**的。
    
2. 交换机划分 VLAN 后，不同 VLAN 之间是**绝对隔离**的（VLAN 10 的电脑 ping 不通 VLAN 20 的电脑）。
    
3. 如果你想让 VLAN 10 和 VLAN 20 通信，就必须借助路由器的“跨网段寻路”功能。
    

**传统的笨办法：** 路由器出两个物理接口，拉两根网线，一根接交换机的 VLAN 10，一根接交换机的 VLAN 20。但如果公司有 50 个 VLAN 呢？路由器的接口根本不够插！

**单臂路由的巧妙解法：**

路由器只出一个物理接口（也就是那只“单臂”），连接到交换机。我们在路由器内部，把这一个物理接口**逻辑上切分成多个“子接口 (Sub-interface)”**。每个子接口对应一个 VLAN，充当那个 VLAN 的默认网关。

# 核心考点笔记：单臂路由 (Router-on-a-Stick)

## 一、什么是单臂路由？

**定义：** 在路由器的一个物理接口上，通过配置多个**逻辑子接口（Sub-interface）**，来实现连接在同一台交换机上、属于不同 VLAN 的终端之间的互相通信。

**核心思想：“一根网线，当多根用”。**

- 物理链路上只有一条（单臂）。
    
- 数据包在这条链路上进行路由时，是先上去（进路由器），查完路由表后，再下来（出路由器）。
    

## 二、优缺点考点

- **优点：** 极大节省了路由器昂贵的物理接口；拓扑结构简单。
    
- **缺点：** 1. **带宽瓶颈：** 所有 VLAN 间的数据都要挤在这一根网线上，容易拥堵。
    
    2. **单点故障：** 这根网线一旦断了，所有 VLAN 之间的通信全部瘫痪。
    
    _(注：在现代中大型网络中，单臂路由已经被“三层交换机”技术全面取代，但它依然是学习网络底层原理的绝对经典，且属于必考范围。)_
    

## 三、华为 eNSP 单臂路由必考配置命令

配置单臂路由是一场“双簧戏”，交换机和路由器必须配合。假设交换机连接路由器的接口是 G 0/0/1，路由器的接口是 G 0/0/0。

### 1. 交换机侧的配合 (必须是 Trunk 链路)

交换机连接路由器的接口，**必须配置为 Trunk 模式**，并允许所有相关 VLAN 通过，因为这条网线要承载带有不同 VLAN 标签的数据包。

Plaintext

```
[Switch] vlan batch 10 20                          // 创建 VLAN 10 和 20
[Switch] interface GigabitEthernet 0/0/1           // 进入连接路由器的接口
[Switch-GigabitEthernet0/0/1] port link-type trunk // 核心：设为 Trunk 模式
[Switch-GigabitEthernet0/0/1] port trunk allow-pass vlan 10 20 // 允许对应 VLAN 通过
```

### 2. 路由器侧的配置 (开启子接口)

物理接口不需要配 IP，直接保持开启状态即可。所有的工作都在**子接口**完成。

Plaintext

```
[Router] interface GigabitEthernet 0/0/0.10             // 1. 创建并进入子接口 .10
[Router-GigabitEthernet0/0/0.10] dot1q termination vid 10 // 2. 核心命令：让子接口识别、剥离并封装 VLAN 10 的标签
[Router-GigabitEthernet0/0/0.10] ip address 192.168.10.254 24 // 3. 配置 IP (作为 VLAN 10 的网关)
[Router-GigabitEthernet0/0/0.10] arp broadcast enable     // 4. 华为设备大坑必考：必须手动开启 ARP 广播，否则无法通信！
[Router-GigabitEthernet0/0/0.10] quit

[Router] interface GigabitEthernet 0/0/0.20             // 创建 VLAN 20 的子接口
[Router-GigabitEthernet0/0/0.20] dot1q termination vid 20 // 封装 VLAN 20
[Router-GigabitEthernet0/0/0.20] ip address 192.168.20.254 24
[Router-GigabitEthernet0/0/0.20] arp broadcast enable     // 开启 ARP 广播
[Router-GigabitEthernet0/0/0.20] quit
```

## 四、黄金排错口诀

如果在 eNSP 中 PC 之间 ping 不通，请按此顺序自查：

1. **查交换机：** 连路由器的口是不是 Trunk？有没有放行 VLAN？
    
2. **查路由器：** 物理口是不是 Shutdown 了？
    
3. **查子接口：** `dot1q` 后面的 VID 填对了吗？**有没有敲 `arp broadcast enable`**？（90% 的初学者死在最后这一步）。
    

---

`dot1q termination vid 10` 和 `arp broadcast enable` 这两句代码是华为体系下单臂路由的灵魂。如果这两句理解了，单臂路由的大题就基本拿下满分了。
