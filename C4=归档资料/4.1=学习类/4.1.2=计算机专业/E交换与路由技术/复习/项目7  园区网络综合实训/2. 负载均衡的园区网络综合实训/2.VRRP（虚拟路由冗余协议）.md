下午的冲刺节奏依然凶猛！刚拿下二层防环的“天花板” MSTP，现在直接杀向三层网关的高可用“神技”——**VRRP（虚拟路由冗余协议）**。

如果在《交换与路由技术》的综合大题里，MSTP 负责的是二层线路的备份，那么 **VRRP 负责的就是三层网关的备份**。它们俩简直就是考卷上雷打不动的“黄金搭档”。

要彻底搞懂 VRRP，我们得先看它解决了什么致命痛点。

### 一、为什么必须要有 VRRP？（单点故障的噩梦）

在普通的网络里，局域网内所有电脑的“默认网关”通常只能指向一台路由器（比如 `192.168.1.254`）。

- **痛点：** 这叫**“网关单点故障”**。一旦这台路由器死机了、断电了，或者这根网线断了，整个内网的几百台电脑将瞬间全部断网，哪怕你旁边还有另一台备用路由器也没用，因为电脑的网关地址没法自动改。
    

### 二、 VRRP 的核心原理：打造一个“幽灵网关”

VRRP 的精髓在于**“欺骗”与“接管”**。

它把两台（或多台）物理路由器，在逻辑上捆绑成一台**“虚拟路由器”**。

1. **生成虚拟 IP (VIP)：** 这台虚拟路由器会拥有一个虚拟的 IP 地址（比如 `192.168.1.250`）和一个虚拟的 MAC 地址。
    
2. **电脑的配置：** 内网所有电脑的网关，不再指向真实的物理路由器，而是**全部指向这个虚拟 IP**。
    
3. **角色分工（考点）：**
    
    - **Master（主路由器）：** 真正干活的设备。它持有虚拟 IP，负责转发底下电脑发来的上网数据。同时，它会每隔 1 秒向外发送“VRRP 心跳报文”，告诉兄弟们“我还活着”。
        
    - **Backup（备用路由器）：** 纯纯的替补。它平时不转发数据，就竖起耳朵听 Master 的心跳。一旦连续 3 秒（也就是 3 个周期）听不到心跳，它就会立刻大喊一声：“大哥挂了，我来接管！”然后瞬间把虚拟 IP 和虚拟 MAC 抢到自己身上，接替转发工作。
        
    - **结果：** 整个切换过程极其迅速，底下的电脑根本感觉不到网关曾经出过事。
        

### 三、考场必背：决定命运的参数

在配置或排错题中，VRRP 的角色谁主谁次，全看这几个核心参数：

1. **VRID (虚拟路由器 ID)：** 一组路由器必须配置相同的 VRID（如 1），才能算作同一个虚拟路由器里的成员。
    
2. **优先级 (Priority)：** **默认值是 100。数字越大，越优先成为 Master。**（范围 1-254）
    
3. **抢占模式 (Preempt)：** 默认开启。假设原来的大哥（Master）修好重启了，只要它的优先级比现在的替补高，它就会强行把 Master 的位子抢回来。
    

### 四、考场进阶大招：上行链路追踪 (Track)🌟压轴考点

这是大题中最爱挖坑的地方！

**场景：** Master 路由器本身没死，连着内网的接口也是好的，所以它还在拼命发心跳。但是，**它连接互联网的那根外网线被挖断了**！

**致命后果：** 备用路由器听得到心跳，不敢篡权；但底下电脑发给 Master 的数据包，全都死在了 Master 出不去的外网口上。全网瘫痪！

**破解之法 (Track)：** 让 VRRP 监视（Track）上行外网接口。一旦发现外网接口断了，立刻**主动降低自己的优先级**（比如自降 30），让 Backup 的优先级反超自己，从而逼迫 Backup 强行接管网关！

### 五、华为满分配置流（考卷必写代码）

**【考题背景】**

路由器 R 1 和 R 2 互为备份。规划虚拟网关 IP 为 `192.168.1.250`。要求 R 1 为主设备（Master），R 2 为备用设备（Backup）。

**1. R 1（主网关）配置：**

Plaintext

```
<R1> system-view
[R1] interface GigabitEthernet 0/0/1             ! 进入连接内网的接口
[R1-GigabitEthernet0/0/1] ip address 192.168.1.253 24
! 核心指令开始
[R1-GigabitEthernet0/0/1] vrrp vrid 1 virtual-ip 192.168.1.250  ! 配置 VRID 和虚拟 IP
[R1-GigabitEthernet0/0/1] vrrp vrid 1 priority 120              ! 提高优先级让它当老大 (默认100)
[R1-GigabitEthernet0/0/1] quit
```

**2. R 2（备用网关）配置：**

Plaintext

```
<R2> system-view
[R2] interface GigabitEthernet 0/0/1
[R2-GigabitEthernet0/0/1] ip address 192.168.1.254 24
! 核心指令开始
[R2-GigabitEthernet0/0/1] vrrp vrid 1 virtual-ip 192.168.1.250  ! VRID 和 VIP 必须和主设备完全一致
! 备用设备不需要配优先级，让它保持默认的 100 即可，自然竞争失败成为 Backup
[R2-GigabitEthernet0/0/1] quit
```

**排错验证命令：** `display vrrp`。只要在回显中看到 `State : Master` 或者 `State : Backup`，这道题的满分你就已经稳稳拿下了！
