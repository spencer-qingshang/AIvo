前面我们聊了高级 ACL 的内部逻辑（同时满足多条件），现在我们就来看看当一个真实的数据包冲进路由器时，路由器是如何拿着高级 ACL 去给它“搜身”的。

在《交换与路由技术》的考试中，高级 ACL 的规则匹配流程，本质上就是一个**“五级漏斗+命中即停”**的查验流水线。

### 核心流水线：五级查验漏斗

当路由器提取出某一条高级 ACL 规则（例如 Rule 5）准备和数据包进行比对时，它会在不到一毫秒的时间内，严格按照以下顺序进行“灵魂拷问”（这也就是五元组的真实匹配顺序）：

- **第一级漏斗：查协议（Protocol）**
    
    - _动作：_ 路由器先剥开网络层，看数据包是 TCP、UDP、ICMP 还是 OSPF？
        
    - _匹配结果：_ 如果规则写的是 `tcp`，而数据包是个 `icmp` (Ping)，**第一关直接出局**！路由器立刻判定该规则不匹配，跳去查下一条规则（Rule 10）。如果协议对上了，继续往下查。
        
- **第二级漏斗：查源 IP 地址（Source IP）**
    
    - _动作：_ 结合规则里的源通配符掩码，核对发件人地址。
        
    - _匹配结果：_ 如果源 IP 不在规则允许的范围内，出局，看下一条规则。如果对上了，继续查。
        
- **第三级漏斗：查目的 IP 地址（Destination IP）**
    
    - _动作：_ 结合规则里的目的通配符掩码，核对收件人地址。
        
    - _匹配结果：_ 对不上就出局；对上了，如果协议是 IP 或 ICMP，匹配基本结束；如果是 TCP/UDP，还要继续往下看！
        
- **第四/五级漏斗：查端口号（Source/Destination Port）**
    
    - _动作：_ 如果规则里写了 `destination-port eq 80`，路由器就会去拆解传输层，看这个包是不是去访问网页的。
        
    - _匹配结果：_ 如果连这最后一关都严丝合缝地对上了，**恭喜，这条高级 ACL 规则被正式命中！**
        

---

### 匹配成功后的“绝对指令”：命中即停

一旦数据包成功闯过了上面这五级漏斗，完全符合了某一条规则的所有条件，路由器就会立刻执行**最终裁决**，并且触发“停止”机制。

- 如果是 **Permit（允许）**：数据包被盖上放行章，扔进路由表继续转发。**路由器立刻合上 ACL 名单，后面的规则看都不看！**
    
- 如果是 **Deny（拒绝）**：数据包直接被丢弃（俗称丢进黑洞）。**同样，路由器直接合上名单，匹配强制结束！**

### 🎯 考场终极沙盘推演（大题排错必考）

为了把这个流水线彻底砸实，我们来看一道非常经典的考试陷阱题。

**接口配置了如下高级 ACL：**

Plaintext

```
rule 5 deny tcp source 192.168.1.10 0 destination 10.1.1.100 0 destination-port eq 80
rule 10 permit ip source 192.168.1.0 0.0.0.255 destination 10.1.1.0 0.0.0.255
```

**现在有三个数据包来到该接口，请判断它们的命运：**

1. **数据包 A：** `192.168.1.10` 发给 `10.1.1.100`，想去访问网页（**TCP 80**）。
    
    - _匹配过程：_ 过 Rule 5。协议 (TCP) 对齐 -> 源对齐 -> 目的对齐 -> 端口 (80) 对齐。**全中！**
        
    - _结果：_ 命中 Rule 5，执行 **Deny**（丢弃），不看 Rule 10。
        
2. **数据包 B：** `192.168.1.10` 发给 `10.1.1.100`，想去下载文件（**TCP 21**）。
    
    - _匹配过程：_ 过 Rule 5。前三关都过了，最后一关发现**端口是 21，不是 80。匹配失败！**
        
    - _继续匹配：_ 路由器去过 Rule 10。协议 (ip) 包含 TCP，源/目的都在网段内。**命中 Rule 10！**
        
    - _结果：_ 执行 **Permit**（放行）。
        
3. **数据包 C：** `192.168.1.10` 发给 `10.1.1.100` 的 **Ping** 包（**ICMP** 协议）。
    
    - _匹配过程：_ 过 Rule 5。第一关查协议，**ICMP 不等于 TCP，瞬间出局！**
        
    - _继续匹配：_ 路由器去过 Rule 10。Ip 包含 ICMP，源/目的对齐。**命中 Rule 10！**
        
    - _结果：_ 执行 **Permit**（放行）。
