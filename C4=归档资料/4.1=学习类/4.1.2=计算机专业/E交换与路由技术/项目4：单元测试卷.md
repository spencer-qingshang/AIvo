## 附：参考答案与详细解析 (项目 4)

### 一、 选择题
1. **C** 【解析】IPv6 使用 128 位地址长度，彻底解决了 IPv4 的 32 位地址枯竭问题。
2. **B** 【解析】IPv6 采用 8 组、每组 4 个十六进制数表示，各组之间用冒号“:”分隔。
3. **D** 【解析】缩写规则：(1) 省略每组前导 0；(2) 连续全 0 组缩写为 `::`（全地址仅限一次）。B 选项双冒号在中间，A 选项没缩写到位，C 选项虽然也合法但通常优先缩写最长一段。
4. **A** 【解析】`::1` 等同于 IPv4 的 `127.0.0.1`。
5. **B** 【解析】链路本地地址（Link-Local）用于同一链路内的通信，以 `FE80::/10` 为前缀。
6. **C** 【解析】IPv6 取消了广播，使用组播（Multicast）来执行邻居发现等任务，提高了网络效率。
7. **B** 【解析】IPv6 报头固定为 40 字节，去除了校验和等字段，使路由器转发更高效。
8. **C** 【解析】流标签（Flow Label）是 IPv6 特有的字段，用于标识属于同一个业务流的数据包，便于 QoS 处理。
9. **A** 【解析】在华为设备上，全局输入 `ipv6` 开启 IPv6 协议栈是后续所有 IPv6 配置的前提。
10. **D** 【解析】IPv6 默认路由的目的网络前缀为 `::`（全 0），前缀长度为 `/0`。
11. **B** 【解析】汇总路由通过提取共同前缀，将多条具体的明细路由压缩为一条。
12. **B** 【解析】RIPng 是针对 IPv6 的 RIP 版本，使用 UDP 521 端口；RIPv2 使用 520 端口。
13. **C** 【解析】RIPng 不再使用 `network` 宣告，而是直接在接口视图下通过命令 `ripng 1 enable` 来使能。
14. **D** 【解析】RIPng 使用组播地址 `FF02::9` 发送更新信息。
15. **A** 【解析】尽管是 IPv6 协议，OSPFv3 的 Router ID 仍遵循 32 位点分十进制格式以保持兼容性和管理习惯。
16. **D** 【解析】Type 9 LSA 是 OSPFv3 新增的，用于描述区域内与特定接口相关的 IPv6 前缀信息。
17. **B** 【解析】OSPFv3 与 RIPng 类似，通过在接口视图下绑定进程（如 `ospfv3 1 area 0`）来参与路由计算。
18. **C** 【解析】ICMPv6 包含了 NDP（邻居发现协议），其中的 RS（路由请求）和 RA（路由通告）报文是 SLAAC 的核心。
19. **B** 【解析】静态路由命令最后一位参数通常是下一跳的 IPv6 全球单播地址。
20. **C** 【解析】IPsec 在 IPv6 协议簇中是强制性支持的（虽然不一定强制开启），提供了原生的端到端安全。

### 二、 填空题
21. **4** 【解析】128 位 / 32 位 = 4。
22. **NDP (邻居发现协议)** 【解析】基于 ICMPv6，实现地址解析、重复地址检测等。
23. **FE80** 【解析】只要接口开启 IPv6，系统会自动基于 EUI-64 或随机算法生成链路本地地址。
24. **校验和 (Checksum)** 【解析】IPv6 认为二层和四层已有校验，三层不再重复计算，以提升转发性能。
25. **::/0** 【解析】IPv6 的万能匹配路由。
26. **15** 【解析】RIP 协议的核心限制，超过 15 跳被认为环路或不可达。
27. **接口** 【解析】IPv6 动态路由协议（RIPng/OSPFv3）普遍改在接口下直接使能。
28. **IPv6** 【解析】直接封装在 IPv6 报文内，协议号为 89。
29. **骨干** 【解析】Area 0 在所有 OSPF 版本中都是层级结构的核心。
30. **一** 【解析】多次出现双冒号会导致地址长度歧义，无法解析。

### 三、 简答题 (参考要点)
31. **IPv6优势**：地址多、报头简（转发快）、自动配置（插拔即用）、内置安全（IPsec）、流标签（QoS好）。
32. **缩写规则**：前导0省去；全0组变::；::全地址仅现一次。
33. **RIPng/RIPv2区别**：IPv6 vs IPv4；接口配置 vs Network 宣告；FF02::9 vs 224.0.0.9；UDP 521 vs 520。
34. **汇总路由**：合并明细路由。意义：缩减路由表规模，掩盖网络局部故障带来的路由震荡。
35. **OSPFv3/v2对比**：算法同为SPF；v3基于链路（Link）运行，v2基于网段（Subnet）；v3支持多实例。

### 四、 应用题
36. **IPv6 静态路由配置 (5分)**：
```shell
ipv6 # 全局开启
```

37. **OSPFv3 基础配置 (5分)**：
```shell
# 1. 全局配置进程和ID
ospfv3 1

# 2. 进入接口绑定到区域
interface GigabitEthernet 0/0/0

```
