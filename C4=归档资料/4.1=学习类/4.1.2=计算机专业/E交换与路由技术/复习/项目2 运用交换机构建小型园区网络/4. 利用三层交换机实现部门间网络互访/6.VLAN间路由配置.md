在三层交换机上实现 VLAN 间路由，本质上是为每一个 VLAN 建立一个逻辑上的“出口”。这比传统的路由器单臂路由更简洁、更高效。

以下是实现“部门 A 访 部门 B”的标准三部曲配置流程。

### 1. 第一步：基础建设（创建 VLAN 并划分接口）

在配置路由之前，二层的底座必须打牢。

```bash
# 1. 创建业务 VLAN
[SW1] vlan batch 10 20

# 2. 将连接终端的接口划入对应 VLAN
[SW1] interface GigabitEthernet 0/0/1
[SW1-GigabitEthernet0/0/1] port link-type access
[SW1-GigabitEthernet0/0/1] port default vlan 10

[SW1] interface GigabitEthernet 0/0/2
[SW1-GigabitEthernet0/0/2] port link-type access
[SW1-GigabitEthernet0/0/2] port default vlan 20
```

### 2. 第二步：灵魂注入（配置 VLANIF 虚拟接口）

这是实现三层转发的关键。我们要给每个 VLAN 开启一个“虚拟路由器接口”。

```bash
# 配置 VLAN 10 的网关
[SW1] interface Vlanif 10
[SW1-Vlanif10] ip address 192.168.10.254 24

# 配置 VLAN 20 的网关
[SW1] interface Vlanif 20
[SW1-Vlanif20] ip address 192.168.20.254 24
```

### 3. 第三步：验证与检查

配置完成后，三层交换机会在路由表中自动生成对应的**直连路由**。

- **查看路由表**：`display ip routing-table`
    - 你应该能看到两条 `Direct`（直连）路由，目的地分别是 192.168.10.0/24 和 192.168.20.0/24。
- **Ping 测试**：在 PC1 上 Ping PC2 的 IP 地址。

---

### 4. 考点挑战：排错实战

**场景**：你配置好了 SW1 的 VLANIF 10 (192.168.10.254) 和 VLANIF 20 (192.168.20.254)，PC1 和 PC2 也分别设好了对应的网关。但 PC1 依然 Ping 不通 PC2。

**请列出 3 个可能的排查点及对应的检查命令：**

**标准答案：**
1.  **物理层/二层未就绪**：检查 VLAN 10 内是否有接口处于 Up 状态（只有该 VLAN 内至少有一个物理口 Up，VLANIF 才会 Up）。
    - **命令**：`display ip interface brief`
2.  **网关配置错误**：检查 PC 侧填写的“默认网关”是否与交换机 VLANIF 的 IP 完全一致。
    - **检查项**：PC 的 IP 设置界面。
3.  **VLAN 划分错误**：检查接口是否真的被正确划入了 VLAN 10/20。
    - **命令**：`display vlan`

---

**实战逻辑总结**

在 eNSP 大型组网中，三层交换机通常位于**汇聚层**或**核心层**。
- **向下**：通过 Trunk 链路连接接入层交换机，透传多个 VLAN。
- **向上**：通过物理三层口或专用的互联 VLAN 连接出口路由器。

**终极口诀：**
> 先建 VLAN 再划口，VLANIF 设网关。
> 路由表里看直连，Ping 通对面才算完。
