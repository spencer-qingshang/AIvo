# 第一章：初识Java - 必备关键知识点

本章是Java编程的入门，旨在建立对Java语言的宏观认识，了解其特点和开发环境。

---

## 1. Java语言发展

### 1.1 计算机语言分类 (识记)

*   **机器语言**：计算机直接识别和执行的二进制指令。
*   **汇编语言**：使用助记符代替机器指令，与硬件紧密相关。
*   **高级语言**：更接近自然语言，抽象程度高，与硬件无关 (如 Java)。

### 1.2 Java发展史与特点 (识记)

*   **发展史**：
    *   由 `Sun` 公司推出，James Gosling（Java之父）领导开发。
    *   原名 `Oak` (1991年)，1995年正式改名为 `Java`。
    *   2009年被 `Oracle (甲骨文)` 公司收购。
*   **主要特点 (6个关键词)**：
    *   **简单易学**：去除了C++的复杂特性（如指针、多继承）。
    *   **混合型 (半编译、半解释)**：`.java` 源码先**编译**成字节码（`.class`），再由 `JVM` **解释执行**。
    *   **面向对象**：纯面向对象，支持**封装、继承、多态**。
    *   **平台无关性 (跨平台)**：**“一次编译，到处运行”** (Write Once, Run Anywhere)，依靠不同系统的 `JVM` 实现。
    *   **安全稳健**：具有垃圾回收机制（GC），无指针操作，强异常处理。
    *   **多线程**：内置多线程支持，可同时处理多个任务。

### 1.3 Java体系 (了解)

*   **Java SE (Standard Edition)**：**核心和基础**，用于桌面应用、服务器开发。包含 `JDK`、`JRE` 及核心类库。
*   **Java EE (Enterprise Edition)**：企业版，用于开发**企业级应用**（如网站、后台服务）。在 `Java SE` 基础上增加了 `Servlet`、`JSP`、`EJB` 等技术。
*   **Java ME (Micro Edition)**：微型版，专为**嵌入式消费电子设备**（如老式手机）提供。

## 2. Java开发环境

### 2.1 JDK、JRE与JVM概念及关系 (识记)

*   **JVM (Java Virtual Machine)**：Java虚拟机，是Java程序运行的**核心**，负责解释执行字节码。
*   **JRE (Java Runtime Environment)**：Java运行环境，**包含JVM**和程序运行所需的核心类库。**只安装JRE可以运行Java程序**。
*   **JDK (Java Development Kit)**：Java开发工具包，**包含JRE**和编译器(`javac`)等开发工具。**开发Java程序必须安装JDK**。
*   **关系总结**：`JDK` > `JRE` > `JVM` (范围上包含关系)。

### 2.2 JDK主要目录结构 (领会)

*   **`bin`**：存放可执行程序（命令），如 `javac.exe` (编译器) 和 `java.exe` (Java运行程序)。
*   **`jre`**：Java运行环境的根目录。
*   **`lib`**：存放Java类库文件。

### 2.3 环境变量配置 (`JAVA_HOME`, `Path`) (应用)

*   **`JAVA_HOME`**：指向JDK的安装路径，许多Java相关工具会依赖此变量。
*   **`Path`**：将 `%JAVA_HOME%\bin` 加入Path变量，使得操作系统能在任何目录下找到`javac`和`java`等命令。

## 3. Java程序编写

### 3.1 Java程序结构 (识记)

*   **基本单位**：Java程序以**类 (`class`)** 为基本单位。
*   **文件名规则 (必考)**：如果一个类被声明为 `public`，那么**文件名必须与类名完全一致** (包括大小写)，扩展名为 `.java`。一个源文件中可以有多个类，但**只能有一个 `public` 类**。
*   **`main` 方法 (程序入口)**：
    *   **地位/作用**：程序的**入口点 (Entry Point)**，`JVM` 从这里开始执行代码。
    *   **标准格式 (必背)**：`public static void main(String[] args)`
        *   `public`：访问修饰符，公开，供 `JVM` 调用。
        *   `static`：静态，无需创建对象即可直接运行。
        *   `void`：返回值类型，表示该方法无返回值。
        *   `main`：方法名，`JVM` 识别的特定名称。
        *   `String[] args`：命令行参数，一个字符串数组。

### 3.2 编译与运行 (应用)

*   **编译 (`javac` 命令)**：
    *   **作用**：将**源代码文件** (`.java`) 翻译成**字节码文件** (`.class`)。
    *   **格式**：`javac 文件名.java` (必须带 `.java` 后缀)。
*   **运行 (`java` 命令)**：
    *   **作用**：启动 `JVM`，加载字节码文件并运行程序。
    *   **格式**：`java 类名` (**绝对不能带后缀名**，`JVM` 会自动找对应的 `.class` 文件)。
