# 核心考点笔记：RIP 三大核心定时器

RIP 协议主要依靠三个定时器来协作完成路由的更新、老化和删除。

## 一、定时器详解 (⚠️ 必背参数)

|**定时器名称**|**默认时间**|**核心作用与行为**|
|---|---|---|
|**更新定时器 (Update)**|**30 s**|**发传单：** 路由器每隔 30 秒向邻居发送一次完整的路由表更新。为了防止全网同步拥塞，通常会有 ±5 s 的随机偏差。|
|**老化定时器 (Age)**|**180 s**|**怀疑：** 如果 180 秒内没收到某条路由的更新，路由器就认为它“老了、不可靠了”，将其跳数设为 **16 (不可达)**，但暂不删除。|
|**垃圾收集 (Garbage-collect)**|**120 s**|**清扫：** 路由变 16 跳后，触发此闹钟。如果再过 120 秒还没收到更新，该路由条目将从路由表中**彻底删除**。|

> **💡 考点解析：** > * 一条路由从失联到被彻底删除，总共需要 **180 s + 120 s = 300 s**。这也是为什么说 RIP 收敛速度慢的原因。
> 
> - **抑制定时器 (Hold-down)：** 部分实现中还有这个定时器（通常也是 180 s），当收到路由不可达信息时，进入抑制状态，拒绝接收该路由更优的更新，防止环路。
>

## 二、华为 VRP 查错与配置命令 (纯简写)

在 eNSP 实验中，如果你想验证或修改这些时间：

### 1. 查看当前定时器数值

Plaintext

```
<R1> dis rip 1             // 查看 RIP 进程 1 的详细运行状态，包含所有定时器
```

### 2. 修改定时器 (不建议随意改，除非题目要求)

**指令格式：** `timers rip <update> <age> <garbage-collect>`

Plaintext

```
[R1] rip 1
[R1-rip-1] tim r 20 120 80 // 简写: timers rip。将更新改为20s, 老化120s, 垃圾收集80s
```

## 三、定时器与防环机制

RIP 容易产生**路由环路**，定时器配合以下机制可以有效缓解：

1. **水平分割 (Split Horizon)：** 从一个接口学到的路由，不再从该接口发回去。
    
2. **毒性逆转 (Poison Reverse)：** 发现路由不可达，立即回发一个 16 跳的包告诉对方“这路死了”。
    
3. **触发更新 (Triggered Update)：** 只要拓扑一变，**不等 30 s 闹钟**，立刻发包通知邻居。
