### 一、考场绝对核心：CHAP 的“三次握手”与“密文机制”

CHAP 在考卷上的出场率极高，选择题只要提到“高安全性”、“MD 5 哈希”或者“密文传输”，闭着眼睛选 CHAP 就对了。它的工作机制是经典的**三次握手（3-Way Handshake）**：

1. **挑战（Challenge）- 主认证方发起：** 保安（主认证方 R 1）拦下你，不让你报密码，而是扔给你一个随机生成的数字（比如 `8848`）和自己的主机名。
    
2. **响应（Response）- 被认证方计算并回复：** 你（被认证方 R 2）拿到随机数 `8848` 后，把它和你的本地密码（比如 `Huawei@123`）放在一起，扔进一台名叫 **MD 5 的绞肉机（哈希算法）**里，绞出一串乱码（Hash 值）。你把这串乱码发给保安。**（注意：真实的密码从来没有在网线上出现过！）**
    
3. **成功/失败（Success/Failure）- 主认证方核对：** 保安在本地用一模一样的方法，把 `8848` 和数据库里的密码也放进 MD 5 绞肉机算一次。如果两边算出来的乱码一模一样，说明你确实掌握了正确的密码，保安放行。

### 二、实战满分配置流（华为 VRP）

CHAP 的配置思路和 PAP 非常像，也是分“设关卡”和“交通行证”两步，只是接口下的命令关键字变了。

假设 R 1 是主认证方（总部），R 2 是被认证方（分部），通过 `Serial 4/0/0` 相连。

#### 第一步：配置主认证方（R 1 - 设关卡并发起挑战）

Plaintext

```
<R1> system-view
! 1. AAA 建立合法名单（和 PAP 完全一模一样）
[R1] aaa
[R1-aaa] local-user admin password cipher Huawei@123
[R1-aaa] local-user admin service-type ppp
[R1-aaa] quit

! 2. 进入串口，开启 CHAP 认证大门
[R1] interface Serial 4/0/0
[R1-Serial4/0/0] link-protocol ppp
[R1-Serial4/0/0] ip address 10.1.1.1 24
! 👇 核心指令：要求对方使用 CHAP 认证
[R1-Serial4/0/0] ppp authentication-mode chap
[R1-Serial4/0/0] quit
```

#### 第二步：配置被认证方（R 2 - 提交哈希响应）

R 2 需要告诉路由器，用哪个用户名和密码去配合那个“MD 5 绞肉机”进行计算。

Plaintext

```
<R2> system-view
[R2] interface Serial 4/0/0
[R2-Serial4/0/0] link-protocol ppp
[R2-Serial4/0/0] ip address 10.1.1.2 24
! 👇 核心指令：配置发送给认证方的 CHAP 用户名
[R2-Serial4/0/0] ppp chap user admin
! 👇 核心指令：配置用于计算哈希值的本地密码
[R2-Serial4/0/0] ppp chap password cipher Huawei@123
[R2-Serial4/0/0] quit
```

### 三、 PAP vs CHAP 考前终极速记表

这绝对是大题和选择题的采分点，考前在脑子里过一遍：

|**对比维度**|**PAP (密码认证协议)**|**CHAP (挑战握手认证协议)**|
|---|---|---|
|**握手次数**|**2 次**|**3 次**|
|**发起方**|被认证方（主动递交密码）|主认证方（主动发起挑战）|
|**密码传输形式**|**明文 (Plaintext)**|**密文 / 哈希值 (Hash)**|
|**安全性**|极差，易被窃听|**极高**，防止重放攻击|
|**适用场景**|安全要求极低的内部测试链路|**互联网、企业广域网核心链路（现网标配）**|
