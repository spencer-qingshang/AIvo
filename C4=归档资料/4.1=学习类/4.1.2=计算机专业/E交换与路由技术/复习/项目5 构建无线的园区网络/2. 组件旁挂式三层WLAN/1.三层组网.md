在企业级 WLAN 架构中，**三层组网（Layer 3 Networking）** 是大型企业（如校园网、多层办公楼）的绝对标配，也是综合配置大题里最爱设置陷阱的地方。

理解三层组网，核心在于明白 **AP 和 AC 不在同一个网段（VLAN）里**。

以下是应对三层组网必考点的“通关指南”：

### 一、为什么非要用三层组网？

如果是小公司（二层组网），AP 和 AC 都在一个网段，AP 插上电后在局域网里大喊一声（发送广播报文）：“谁是 AC？”，AC 听到后直接回应，俩人就顺利“建交”了。

但在大型企业中，如果有 500 个 AP，把它们全放在一个 VLAN 里会引发恐怖的“广播风暴”，直接让网络瘫痪。因此，必须用三层交换机把它们划分到不同的 VLAN（网段）中。**这就带来了三层组网最致命的考点：广播报文是无法跨越路由器的。**

AP 启动后在自己的网段里大喊，但声音被网关（路由器/三层交换机）拦截了。AP 找不到 AC，成了“孤儿”。

### 二、破局的“寻路神器”：DHCP Option 43（绝对高频考点）

为了让 AP 在茫茫三层网络中精准找到 AC，网络工程师引入了 **DHCP Option 43** 这个神仙属性。

在三层组网中，DHCP 服务器不仅仅是给 AP 分配 IP 地址、子网掩码和网关，它还会通过 Option 43 字段，悄悄递给 AP 一张“纸条”，上面写着：“**你的顶头上司 AC，它的 IP 地址是 10.1.1.1，你直接单播去找它！**”

AP 拿到这张纸条后，不再盲目广播，而是直接打包装载着单播目标 IP 的报文，顺着底层的路由协议，跨越千山万水精准命中 AC，建立起 CAPWAP 隧道。

### 三、满分配置实战：如何写下 Option 43

这步配置在咱们刚刚复习的 DHCP 全局地址池中进行。请务必记住这段华为 VRP 的标准格式：

Plaintext

```
<Huawei> system-view
[Huawei] ip pool AP_Pool
[Huawei-ip-pool-AP_Pool] network 192.168.10.0 mask 24
[Huawei-ip-pool-AP_Pool] gateway-list 192.168.10.254

! 👇 这就是三层组网的灵魂所在！
! sub-option 2 代表里面装的是 IP 地址，后面紧跟 AC 的真实管理 IP
[Huawei-ip-pool-AP_Pool] option 43 sub-option 2 ip-address 10.1.1.1 
[Huawei-ip-pool-AP_Pool] quit
```

### 四、考场排错的“底层陷阱”

在做三层组网的综合大题时，很多考生明明把 Option 43 敲对了，AP 依然死活上不了线，通常是掉进了以下两个坑：

1. **底层路由不通：**
    
    既然是三层组网，AP 和 AC 之间隔着路由器。**AP 必须能 Ping 通 AC 的 IP 地址。** 如果底层的 OSPF 或静态路由没配好，AP 拿着“纸条”也找不到路。所以在查 WLAN 故障前，先用底层网络互 Ping 一下。
    
2. **AC 忘记指定源接口：**
    
    你虽然告诉了 AP 来 `10.1.1.1` 找 AC，但如果你在 AC 上忘记敲 `capwap source interface Vlanif X`（宣告自己用这个 IP 建立隧道），AC 就会对 AP 的请求视而不见。
    

理清了三层组网的“跨网段寻址”逻辑，WLAN 最难的网络互通部分就被你彻底踩在脚下了！
