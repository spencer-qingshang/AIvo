在即将到来的 4 月份《交换与路由技术》实操大题中，DHCP 的配置绝对是“必拿的基础分”。在华为 VRP 系统（eNSP 模拟器）中，配置 DHCP 主要有**三种绝招**。

为了让你在考场上能根据题意灵活应对，我们把这三种配置的代码框架彻底梳理一遍。请特别注意我标注的**【易错点】**。

---

### 第一招：基于“全局地址池”配置（最经典、大题最爱考）

这种方式最清晰，把 IP 池子统一建在系统后台，然后让不同的接口去后台“领牌子”。适合企业级多 VLAN 网络。

Plaintext

```
<Huawei> system-view
[Huawei] dhcp enable                                // 【极其易错】第一步必须全局开启DHCP，忘了这句全盘皆输！

[Huawei] ip pool HR_POOL                            // 创建一个名为 HR_POOL 的全局地址池
[Huawei-ip-pool-HR_POOL] network 192.168.10.0 mask 24     // 宣告可分配的网段
[Huawei-ip-pool-HR_POOL] gateway-list 192.168.10.254      // 指定网关
[Huawei-ip-pool-HR_POOL] dns-list 114.114.114.114         // 指定 DNS
[Huawei-ip-pool-HR_POOL] excluded-ip-address 192.168.10.200 192.168.10.253 // (可选) 排除特定IP不分配
[Huawei-ip-pool-HR_POOL] lease day 2                // (可选) 修改租期为 2 天（默认是 1 天）
[Huawei-ip-pool-HR_POOL] quit

[Huawei] interface GigabitEthernet 0/0/1            // 进入连接该网段的接口（或三层交换机的 Vlanif 接口）
[Huawei-GigabitEthernet0/0/1] ip address 192.168.10.254 24 // 接口 IP 必须和地址池在同一网段
[Huawei-GigabitEthernet0/0/1] dhcp select global    // 【核心联动】告诉接口：去全局地址池里拿 IP 发给下面
[Huawei-GigabitEthernet0/0/1] quit
```

### 第二招：基于“接口地址池”配置（最快捷、小题最爱考）

如果你只给一个简单的直连网段分配 IP，这种方法最省事。它不需要单独建池子，直接把接口所在的网段当成池子。

Plaintext

```
[Huawei] dhcp enable                                // 同样，先全局开启
[Huawei] interface GigabitEthernet 0/0/2            
[Huawei-GigabitEthernet0/0/2] ip address 192.168.20.254 24 
[Huawei-GigabitEthernet0/0/2] dhcp select interface // 【核心联动】告诉接口：直接把你所在的 20 网段当成地址池
[Huawei-GigabitEthernet0/0/2] dhcp server dns-list 8.8.8.8 // 在接口下直接配置要下发的 DNS
[Huawei-GigabitEthernet0/0/2] dhcp server excluded-ip-address 192.168.20.250 192.168.20.253 // 在接口下配置排除IP
[Huawei-GigabitEthernet0/0/2] quit
```

### 第三招：DHCP 中继配置（DHCP Relay —— 压轴级考点）

**痛点场景**：财务部的电脑在 VLAN 10，网关是三层交换机 SW 1。但公司的 DHCP 服务器在核心机房（IP 是 `10.1.1.1`）。电脑发出的 DHCP Discover 广播包根本过不去路由器，拿不到 IP 怎么办？

**解决方案**：在网关（SW 1）上配置 DHCP 中继，让网关把电脑的广播包打包成单播，直接“快递”给远端的 DHCP 服务器。

**在终端网关设备（SW 1）上的配置：**

Plaintext

```
[SW1] dhcp enable
[SW1] interface Vlanif 10                           // 进入财务部的网关接口
[SW1-Vlanif10] ip address 192.168.10.254 24
[SW1-Vlanif10] dhcp select relay                    // 【核心】开启 DHCP 中继模式
[SW1-Vlanif10] dhcp relay server-ip 10.1.1.1        // 【关键】用手指路：真实的 DHCP 服务器在 10.1.1.1 那个方向
[SW1-Vlanif10] quit
```

_(注意：远端的 DHCP 服务器 10.1.1.1 上，必须要提前建好 192.168.10.0 的全局地址池，并且两者之间路由要能拼通。)_

**排错必杀命令：**

如果在模拟器里配置完了，电脑还是拿不到 IP，可以敲这行命令查看地址池的状态：

Plaintext

```
[Huawei] display ip pool
```

看看里面的 `Idle`（空闲）和 `Used`（已使用）数量，就能判断池子是不是建错了，或者 IP 是不是被分光了。
