这绝对是毅力的极限挑战了！凌晨三点还在死磕 NAT 家族的拼图。既然你点名了**动态 NAT（Dynamic NAT）**，咱们就一鼓作气，用最直观的方式把它彻底拿下，为今天的路由交换复习画上一个完美的句号！

在网络考试中，动态 NAT 常常作为 NAPT 的“对照组”出现。理解它的核心，全在一个词：**“排队”**。

### 一、核心原理：只换 IP，不带端口（公共停车场模式）

- **工作机制：** 路由器手里捏着一个**公网 IP 地址池（Address Pool）**，比如里面有 5 个公网 IP。当内网的电脑想要上外网时，路由器会从池子里拿出一个空闲的公网 IP 借给它。
    
- **致命瓶颈（考点）：** 因为它**不转换端口（No-PAT）**，所以一个公网 IP 在同一时刻只能给一台内网电脑使用。如果池子里的 5 个 IP 都被借出去了，第 6 台电脑就必须等前面的人下线、把 IP 还回池子后，才能“排队”上网。
    
- **现网地位：** 由于它根本没有解决公网 IP 严重浪费的问题，现网工程中几乎已经被 NAPT 彻底淘汰。但在考试的排错题和选择题中，出题人特别喜欢拿它来挖坑。
    

### 二、满分配置流：灵魂参数 `no-pat`

在华为设备的配置中，动态 NAT 和主流的 NAPT 配置极其相似，**唯一的区别就在于最后一行命令里的一个后缀参数。**

**【考场实战场景】**

研发部网段 `192.168.1.0/24` 需要上网。公司向运营商购买了 5 个公网 IP，范围是 `200.1.1.10` 到 `200.1.1.14`。要求配置动态 NAT。

Plaintext

```
<Huawei> system-view
! 1. 建立公网 IP 地址池（编号为 1）
[Huawei] nat address-group 1 200.1.1.10 200.1.1.14

! 2. 用基本 ACL 抓取允许上网的私网网段（发通行证）
[Huawei] acl 2000
[Huawei-acl-basic-2000] rule 5 permit source 192.168.1.0 0.0.0.255
[Huawei-acl-basic-2000] quit

! 3. 进入连接公网的出接口
[Huawei] interface GigabitEthernet 0/0/1

! 4. 核心指令：将 ACL 2000 与地址池 1 绑定，并强制不转换端口！
[Huawei-GigabitEthernet0/0/1] nat outbound 2000 address-group 1 no-pat
[Huawei-GigabitEthernet0/0/1] quit
```

### 三、考场排错“秒杀”技巧

在排错大题中，如果题目要求配置“动态 NAT”（或者明确说明“不能进行端口复用”），你只需要盯死接口下的这行调用命令：

- **带了 `no-pat`：** `nat outbound 2000 address-group 1 no-pat` ➡️ 这是**纯动态 NAT**（多对多，只换 IP，浪费地址）。
    
- **没带 `no-pat`：** `nat outbound 2000 address-group 1` ➡️ 这就瞬间变成了强大的 **NAPT**（多对一，连 IP 带端口一起换，全公司共享上网）。
    

“No-PAT” 的字面意思就是“不要进行端口地址转换（No Port Address Translation）”。只要认准这个后缀，动态 NAT 的考点就绝对跑不出你的手掌心。
