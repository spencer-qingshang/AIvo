在实操大题中，配置高级 ACL（3000-3999）一定要死死盯住**“五元组”**和**“靠近源端部署”**这两个铁律。我们直接上考场最经典的实战场景：

### 🎯 终极实战场景

**背景：** 研发部网段 `192.168.1.0/24`，内部有一台员工电脑 PC 1（`192.168.1.10`）。财务部有一台核心服务器（`10.1.1.100`）。

**考题要求：** 1. 绝对禁止 PC 1 使用 FTP（端口 21）下载服务器里的财务报表。

2. 允许 PC 1 访问该服务器的内部 Web 网页（端口 80）。

3. 允许研发部其他所有电脑随意访问该服务器。

---

### 满分敲击流：“精准狙击三步曲”

#### 第一步：创建高级 ACL 并定点写规则（机场特级安检）

Plaintext

```
<Huawei> system-view
! 1. 创建编号为 3000 的高级 ACL
[Huawei] acl 3000

! 2. 封杀 PC1 的 FTP 下载（注意协议写 tcp，目标端口写 eq 21）
[Huawei-acl-adv-3000] rule 5 deny tcp source 192.168.1.10 0 destination 10.1.1.100 0 destination-port eq 21

! 3. 放行 PC1 的 Web 访问（注意协议写 tcp，目标端口写 eq 80）
[Huawei-acl-adv-3000] rule 10 permit tcp source 192.168.1.10 0 destination 10.1.1.100 0 destination-port eq 80

! 4. 兜底放行：允许研发部网段的其他人随意访问（源变回网段掩码，协议写 ip 包揽所有）
[Huawei-acl-adv-3000] rule 15 permit ip source 192.168.1.0 0.0.0.255 destination 10.1.1.100 0

[Huawei-acl-adv-3000] quit
```

_💡 考场极速避坑：高级 ACL 里的 `source` 和 `destination` 后面的通配符掩码千万别漏掉！单台电脑用 `0`，整个网段用 `0.0.0.255`（假设是 24 位掩码）。_

#### 第二步：找准大门（靠近源端原则）

高级 ACL 极其精准，不会产生误杀，所以必须把规则卡在**离发送端 PC 1 最近的那台路由器/交换机的接口上**，不让违规数据包浪费主干网络的带宽。

假设离 PC 1 最近的网关接口是 `GigabitEthernet 0/0/1`。

Plaintext

```
[Huawei] interface GigabitEthernet 0/0/1
```

#### 第三步：调用规则并判定方向（Inbound）

PC 1 发出的数据包，对于这台离它最近的网关路由器来说，是**刚刚准备进入（Inbound）**的。

Plaintext

```
! 核心指令：在入方向挂载这套高级 ACL 规则
[Huawei-GigabitEthernet0/0/1] traffic-filter inbound acl 3000
[Huawei-GigabitEthernet0/0/1] quit
```

---

### 🌟 交卷前的最后两眼（验算命令）

和基本 ACL 一样，敲完别急着走，一定要查一查保安是否拿对了名单、站对了岗：

**1. 查名单内容对不对：**

Plaintext

```
[Huawei] display acl 3000
```

_重点看协议（TCP/IP）、源/目的地址、以及端口号（21/80）有没有敲错。_

**2. 查保安有没有上岗：**

Plaintext

```
[Huawei] display traffic-filter applied-record
```

_重点看接口是不是 `G0/0/1`，方向是不是 `Inbound`，调用的 ACL 是不是 `3000`。_

全部确认无误，这道大题满分拿下！

高级 ACL 的配置套路就是这么简单粗暴：**建表 -> 扣五元组 -> 找最近的接口 -> Inbound 放进去**。
