“动态 NAT 的工作原理”可以用一个词来精准概括：“**借用与归还**”。

与静态 NAT（终身绑定）不同，动态 NAT 不会为内网主机固定分配公网 IP，而是建立一个“公网 IP 地址池”。我们可以跟着一个数据包，完整走一遍这个动态转换的流水线：

### 第一步：触发请求与“查库存”

当内网的一台主机（例如 IP：`192.168.1.10`）想要访问互联网时，它会将数据包发送给网关路由器。

路由器收到数据包后，检查路由表发现这是一个需要访问公网的请求，于是立刻去自己的**公网 IP 地址池（Address Pool）**里查看：现在还有没有空闲（未被占用）的公网 IP？

- **如果池子空了：** 数据包直接被丢弃，内网主机只能排队等待，直到有人下线让出 IP。
    
- **如果有空闲：** 进入下一步。
    

### 第二步：分配 IP 与“登记造册”（去程出网）

- **替换源 IP：** 路由器挑出一个处于空闲状态的公网 IP（假设为 `200.1.1.10`），临时“借”给这台内网主机。路由器拆开数据包，将原本的私有源 IP（`192.168.1.10`）擦除，替换为这个公网 IP（`200.1.1.10`）。
    
- **记录账本：** 路由器会在内部的 NAT 映射表（NAT Table）中立刻生成一条动态记录：`192.168.1.10 <---> 200.1.1.10`。
    
- **重点注意：** 纯动态 NAT **只记录 IP 映射，绝对不记录端口号（No-PAT）！** 此时，这个公网 IP 被该主机独占。
    

### 第三步：回程查表与“完璧归赵”（回程入网）

- 互联网上的目标服务器收到请求后，会将回复的数据包发给那个借来的公网 IP（`200.1.1.10`）。
    
- 数据包顺着互联网来到公司的网关路由器。路由器收到入站包时，立刻去翻阅刚才登记的 NAT 映射表。
    
- **精准还原：** 查表确认公网 IP `200.1.1.10` 目前正在借给私网 IP `192.168.1.10` 使用。于是，路由器将数据包的目的 IP 替换回真实的私网 IP，并顺利转发给内网的那台主机。
    

### 第四步：“销毁账本”与回收 IP（释放资源）

这是动态 NAT 循环运作的收尾动作。

当这台内网主机上网结束，主动关闭了网络连接（例如发出了 TCP FIN 报文），或者经过了一段设定的老化时间（Aging Time）没有再发送任何数据包时，路由器会毫不留情地执行两个操作：

1. **抹除记录：** 在 NAT 映射表中彻底删除 `192.168.1.10 <---> 200.1.1.10` 这条映射关系。
    
2. **IP 回收：** 将公网 IP `200.1.1.10` 重新放回地址池中，状态从“占用”改回“空闲”，等待下一台需要上网的内网主机来借用。
    

动态 NAT 的底层逻辑就是这样一个**“排队借用，用完即还”**的循环系统。由于它无法实现端口级别的复用（一个公网 IP 同时只能服务一台主机），因此在现代网络架构中，它的工作原理更多地是被当作理解高级 NAPT 技术的一块理论垫脚石。
