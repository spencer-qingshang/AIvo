# 项目1：认识eNSP及VRP的基础操作 (考试复习精简版)

## 一、 必背核心知识点 (识记/领会)

### 1. eNSP 环境依赖 (常考填空/选择)
*   **eNSP** 是华为的企业网络仿真平台。
*   **三大核心组件**：
    1.  **VirtualBox**：核心虚拟机引擎（设备运行的载体，建议 5.2.x 版本）。
    2.  **Wireshark**：抓包分析工具。
    3.  **WinPcap**：底层驱动，用于数据包捕获。

### 2. 网络线缆辨析 (常考识记)
*   **Ethernet (双绞线)**：连接 PC 与交换机/路由器，最通用。
*   **Serial (串口线)**：连接广域网（WAN）接口，模拟器中通常显示为**红色**。
*   **Console 线**：用于现场配置设备，连接 PC 的串口和设备的 Console 口。

### 3. VRP 视图切换 (核心必考)
| 视图 | 提示符 | 核心作用 | 切换命令 |
| :--- | :--- | :--- | :--- |
| **用户视图** | `<Huawei>` | 查看运行状态、文件操作 | 登录即进入 |
| **系统视图** | `[Huawei]` | 配置全局参数、进入子视图 | `system-view` |
| **接口视图** | `[Huawei-G0/0/0]` | 配置接口 IP、物理属性 | `interface G0/0/0` |

### 4. VRP 基本规律
*   **撤销命令**：在原命令前加 `undo`（如 `undo shutdown` 开启接口）。
*   **保存配置**：必须在**用户视图**下执行 `save`。
*   **配置即生效**：VRP 命令敲完回车即生效，无需额外重启。

## 二、 模拟试题预测 (考试怎么出)

### 1. 单项选择题 (考察深度)
*   **Q**: eNSP 设备无法启动，报错“错误代码 40”，通常与哪个组件有关？
    *   A. Wireshark  B. **VirtualBox**  C. WinPcap  D. eNSP 主程序
*   **Q**: 从接口视图直接回退到用户视图的快捷键是？
    *   A. `quit`  B. `exit`  C. **`Ctrl + Z`** (或输入 `return`)  D. `Ctrl + C`

### 2. 填空题 (考察精准度)
*   华为 VRP 平台中，输入命令时按 \_\_\_ 键可以自动补全。 (答案：**Tab**)
*   配置设备名称为 R1 的命令是 \_\_\_。 (答案：**sysname R1**)
*   查看当前接口配置的命令是 \_\_\_。 (答案：**display this**)

### 3. 简答题 (考察逻辑)
*   **Q**: 简述 `quit` 和 `return` 命令的区别。
    *   **参考答案**：`quit` 是逐级回退（如从接口视图退到系统视图）；`return` 是直接回退到最顶层的用户视图（同 `Ctrl+Z`）。

### 4. 综合应用题 (笔试答题模板)
*   **题目**：请写出将路由器名称改为 `Core-R1`，并给 `G0/0/0` 接口配置 IP `10.1.1.254/24` 的完整操作过程。
*   **答题思路**：
    ```shell
    <Huawei> system-view                 # 1. 进入系统视图
    [Huawei] sysname Core-R1             # 2. 修改设备名
    [Core-R1] interface G0/0/0           # 3. 进入目标接口
    [Core-R1-G0/0/0] ip address 10.1.1.254 24  # 4. 配置IP与掩码
    [Core-R1-G0/0/0] quit                # 5. 退出接口
    [Core-R1] quit
    <Core-R1> save                       # 6. 保存(必须在用户视图)
    ```

## 三、 考前最后 5 分钟 (必杀技)

1.  **分清括号**：`<>` 是用户视图（看），`[]` 是系统视图（改）。
2.  **报错辨析**：
    *   *Unrecognized command*：命令敲错了或视图进错了。
    *   *Incomplete command*：参数没写全（比如有 IP 没掩码）。
    *   *Ambiguous command*：简写太短，系统不知道你想选哪个（比如只敲个 `s`）。
3.  **万能命令 `display`**：
    *   `display ip interface brief`：查接口活没活（物理和协议状态）。
    *   `display this`：在哪个视图下，就查那个视图的配置，极好用。
4.  **保存保存再保存**：笔试中如果涉及到配置题，最后一定要写上 `<Huawei> save`，否则不符合工程规范。
