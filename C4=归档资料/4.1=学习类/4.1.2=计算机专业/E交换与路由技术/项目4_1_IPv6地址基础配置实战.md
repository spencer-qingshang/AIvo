# 项目4_1：IPv6地址基础配置实战 (小白保姆级教程)

> **实验目标**：在路由器 R1 上开启 IPv6 功能，并手动给它的接口配上 IPv6 地址。
> **适用人群**：第一次接触 IPv6 的网络小白。

---

## 一、 生活化比喻：快递员换了“超长身份证”

在 IPv4 时代，路由器的身份证（IP）只有 4 段数字（如 `12.1.1.1`），这就像旧式的 15 位身份证。
到了 IPv6 时代，因为人口（设备）太多，我们要给路由器换发 **“128 位超长身份证”**（如 `2001::1`）。

这个实验就是教你：
1.  **激活新证办理处**（全局开启 IPv6 功能）。
2.  **给具体大门贴门牌号**（在接口下配置地址）。

---

## 二、 核心配置步骤 (R1)

### 1. 全局开启 IPv6
```shell
<Huawei> system-view
[Huawei] ipv6        # 【命令拆解】告诉路由器：从现在起，你要开始理解 IPv6 这种语言了。
```

### 2. 进入接口并配置
我们假设电脑连在 `GE 0/0/0` 接口。
```shell
[R1] interface GigabitEthernet 0/0/0
[R1-GigabitEthernet0/0/0] ipv6 enable      # 【命令拆解】在这个具体的门口，把 IPv6 模块“通电”。
[R1-GigabitEthernet0/0/0] ipv6 address 2001:1::1 64
# 【逐词拆解】：
# ipv6 address：我要配地址了
# 2001:1::1：这是门牌号。中间的 :: 是省略了一堆 0，全写是 2001:0001:0000:0000...:0001
# 64：前缀长度（相当于掩码）。IPv6 默认通常都是 64。
```

---

## 三、 避坑必杀技 (新手必看)

1.  **忘记“通电”**：
    *   很多小白直接敲 `ipv6 address ...`，结果报错。
    *   **记住**：必须先在接口下敲 `ipv6 enable`，大门才会变色（逻辑上激活）。
2.  **地址写错**：
    *   IPv6 地址是十六进制（0-9，A-F）。如果你写了个 `2001:G::1`，路由器会报错，因为没有 `G`。
3.  **:: 只能用一次**：
    *   一个地址里只能有一个 `::`。如果你写 `2001::1::2`，路由器会懵掉，因为它不知道中间到底省了多少个 0。

---

## 四、 实验自测 (检查方法)

在 R1 命令行输入：
`display ipv6 interface brief`

*   **预期结果**：
    *   你应该能看到 `GigabitEthernet0/0/0` 那一列。
    *   `Physical` 是 **up**，`Protocol` 是 **up**。
    *   下面会显示你配的 `2001:1::1`。
    *   还会额外多出一个以 `FE80::` 开头的地址，那是路由器自动生成的“小名”（链路本地地址），别紧张，那是正常的。

---

## 五、 总结 (笔记)
*   IPv6 不是默认开启的，需要先在全局敲 `ipv6`。
*   配置地址前，接口必须先 `ipv6 enable`。
*   IPv6 地址看起来很吓人，但其实就是更长、带字母的门牌号而已。
