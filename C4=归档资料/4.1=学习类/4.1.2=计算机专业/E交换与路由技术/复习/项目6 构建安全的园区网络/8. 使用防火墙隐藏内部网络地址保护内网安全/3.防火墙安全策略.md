前面的“安全区域”划分了防火墙的地盘，而**安全策略（Security Policy）**就是决定数据包能不能跨越这些地盘的“最高判决书”。它是传统 ACL 的究极进化版。

### 一、核心逻辑：条件匹配与命中即停

安全策略的匹配逻辑和 ACL 非常像，但它查验的维度要丰富得多。防火墙收到数据包后，会**自上而下（从规则 1 到规则 N）**逐条核对，**命中即停**。

如果所有你自己写的规则都没有命中，防火墙就会执行**默认策略（Default Policy）**。在现网和考场中，绝大多数防火墙的默认策略都是**“拒绝所有（Deny All）”**。也就是说，没有明确被允许的，统统不准过！

### 二、考场必背：安全策略的“匹配六要素”

在配置或排错题中，一条完整的下一代防火墙安全策略通常包含以下查验条件：

1. **源安全区域 (Source Zone)：** 比如 `Trust`。
    
2. **目的安全区域 (Destination Zone)：** 比如 `Untrust`。
    
3. **源地址 / 目的地址 (Source/Destination IP)：** 具体的 IP 或网段（继承了 ACL 的功能）。
    
4. **服务 / 应用 (Service / Application)：** 比如 `HTTP`、`FTP` 或者具体的端口（如 TCP 80）。
    
5. **用户 (User)：** 现代防火墙可以直接基于员工账号（而不是冰冷的 IP）来做策略。
    
6. **动作 (Action)：** 允许（`permit`）还是拒绝（`deny`）。
    

_💡 考点提示：如果某一个要素（比如目的地址）你没有敲命令去限制，防火墙默认该条件为 `any`（匹配所有）。_

### 三、满分实战配置流（华为 USG 防火墙）

**【考题背景】**

允许研发部网段（`192.168.1.0/24`，属于 Trust 区域）访问互联网（Untrust 区域）。

防火墙的配置和路由器有点不一样，它是进入专门的 `security-policy` 视图去写规则的：

Plaintext

```
<Huawei> system-view
! 1. 进入安全策略视图
[Huawei] security-policy

! 2. 创建一条规则，并给它起个好记的名字（比如允许内网出外网）
[Huawei-policy-security] rule name Allow_Trust_to_Untrust

! 3. 核心指令：扣上面说的要素！
[Huawei-policy-security-rule-Allow_Trust_to_Untrust] source-zone trust               ! 指定源区域
[Huawei-policy-security-rule-Allow_Trust_to_Untrust] destination-zone untrust        ! 指定目的区域
[Huawei-policy-security-rule-Allow_Trust_to_Untrust] source-address 192.168.1.0 24   ! 指定源 IP 网段

! 4. 最终裁决：放行
[Huawei-policy-security-rule-Allow_Trust_to_Untrust] action permit

[Huawei-policy-security-rule-Allow_Trust_to_Untrust] quit
[Huawei-policy-security] quit
```

**排错检查命令：** `display security-policy rule all` 可以查看所有策略及命中次数。如果在考场上发现不通，先看这三点：区域写没写对、网段写没写对、最后有没有加 `action permit`。
