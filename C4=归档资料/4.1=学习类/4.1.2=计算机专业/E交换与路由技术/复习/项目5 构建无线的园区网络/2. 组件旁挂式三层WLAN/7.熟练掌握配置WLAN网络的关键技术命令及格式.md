### 一、核心基础：给 AP 分配 IP 与指路（网关设备）

这部分的核心格式是 DHCP 地址池的建立与 Option 43 字段的严格语法。

Plaintext

```
[Huawei] dhcp enable                                  # 1. 全局开启 DHCP
[Huawei] ip pool {pool-name}                          # 2. 创建地址池
[Huawei-ip-pool-{name}] network {ip} mask {mask}      # 3. 宣告网段（必须带 mask 关键字）
[Huawei-ip-pool-{name}] gateway-list {ip}             # 4. 指定网关
[Huawei-ip-pool-{name}] option 43 sub-option 2 ip-address {AC-IP} # 5. 绝对高频考点：配置 Option 43
```

### 二、建交准备：AC 源接口与 AP 录入（AC 设备全局）

考卷上的填空题经常在这里挖坑，特别是源接口的声明。

Plaintext

```
[AC] capwap source interface {vlanif-id}              # 1. 声明建隧道源接口（必背首发命令）
[AC] wlan                                             # 2. 进入 WLAN 视图
[AC-wlan-view] ap auth-mode {mac-auth | no-auth}      # 3. 设置认证模式（通常为 mac-auth）
[AC-wlan-view] ap-id {id} ap-mac {mac-address}        # 4. 离线绑定 AP 的 MAC 地址
[AC-wlan-ap-{id}] ap-name {name}                      # 5. 给 AP 命名（可选，但实操必做）
```

---

### 三、三大模板的创建与拼装（AC 设备的 WLAN 视图）

这是最容易搞混配置层级的地方，牢记**“安全 ➔ SSID ➔ VAP 融合”**的顺序。

**1. 安全模板 (Security Profile)**

Plaintext

```
[AC-wlan-view] security-profile name {sec-name}
[AC-wlan-sec-prof-{name}] security wpa2 psk pass-phrase {password} {加密算法, 如 aes}
```

**2. SSID 模板 (SSID Profile)**

Plaintext

```
[AC-wlan-view] ssid-profile name {ssid-name}
[AC-wlan-ssid-prof-{name}] ssid {Wi-Fi_Name}          # 这是用户真正能搜到的网络名称
```

**3. VAP 模板 (VAP Profile) —— 终极装配车间**

Plaintext

```
[AC-wlan-view] vap-profile name {vap-name}
[AC-wlan-vap-prof-{name}] forward-mode {direct-forward | tunnel}  # 指定转发模式
[AC-wlan-vap-prof-{name}] service-vlan vlan-id {vlan-id}          # 指定业务 VLAN
[AC-wlan-vap-prof-{name}] security-profile {sec-name}             # 挂载安全模板
[AC-wlan-vap-prof-{name}] ssid-profile {ssid-name}                # 挂载 SSID 模板
```

---

### 四、终极下发：绑定到射频 (AP Group 或具体 AP)

做完前面所有的工作，如果没有这行“发射”命令，空中依然没有信号。

Plaintext

```
[AC-wlan-view] ap-group name {group-name}                         # 进入 AP 组（默认通常是 default）
[AC-wlan-ap-group-{name}] vap-profile {vap-name} wlan {wlan-id} radio {all | 0 | 1} 
# 参数解析：
# wlan-id: 虚拟局域网编号（通常填 1 即可）
# radio: all 代表双频同时发射；0 代表 2.4G；1 代表 5G。
```

### 🌟 考场排错必背的三条“保命”命令

配置全部敲完，不要急着交卷，按照顺序依次敲下这三条命令，确保没有纰漏：

1. **查状态：** `display ap all`
    
    - 看 AP 是否上线，状态列（State）必须是 **`nor`**。
        
2. **查模板：** `display vap-profile all`
    
    - 看 VAP 模板里的东西有没有绑齐，有没有漏掉安全配置或业务 VLAN。
        
3. **查信号：** `display vap ssid {Wi-Fi_Name}`
    
    - 确认指定的 SSID 确实已经在空口中广播出去了。
