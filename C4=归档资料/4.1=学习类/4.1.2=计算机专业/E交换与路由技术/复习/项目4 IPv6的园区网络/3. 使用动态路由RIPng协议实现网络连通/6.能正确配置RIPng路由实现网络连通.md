为了实现全网连通，我们来推演一个最标准的考试拓扑：两台路由器（R 1、R 2）通过专线互联，并各自下挂一个 IPv 6 局域网。目标是让两端的 PC 能够互相 Ping 通。

**拓扑地址规划：**

- **R 1 下挂局域网 (PC 1 所在网段)：** `2001:DB8:1::/64` (R 1 接口 IP 为 `::254`)
    
- **R 1 与 R 2 互联直连网段：** `2001:DB8:12::/64` (R 1 为 `::1`，R 2 为 `::2`)
    
- **R 2 下挂局域网 (PC 2 所在网段)：** `2001:DB8:2::/64` (R 2 接口 IP 为 `::254`)
    

以下是实现网络连通的完整华为 VRP 标准配置流程：

### 第一阶段：配置 R 1（左侧路由器）

Plaintext

```
<Huawei> system-view
[Huawei] sysname R1

! 1. 全局开启 IPv6 报文转发（考试最容易忘的第一步）
[R1] ipv6

! 2. 创建 RIPng 进程
[R1] ripng 1
[R1-ripng-1] quit

! 3. 配置互联接口 (G0/0/0) 并激活 RIPng
[R1] interface GigabitEthernet 0/0/0
[R1-GigabitEthernet0/0/0] ipv6 enable                          ! 开启接口 IPv6 能力
[R1-GigabitEthernet0/0/0] ipv6 address 2001:db8:12::1 64       ! 配置公网互联地址
[R1-GigabitEthernet0/0/0] ripng 1 enable                       ! 将该接口加入 RIPng 进程
[R1-GigabitEthernet0/0/0] quit

! 4. 配置局域网网关接口 (G0/0/1) 并激活 RIPng
[R1] interface GigabitEthernet 0/0/1
[R1-GigabitEthernet0/0/1] ipv6 enable
[R1-GigabitEthernet0/0/1] ipv6 address 2001:db8:1::254 64      ! 配置局域网网关地址
[R1-GigabitEthernet0/0/1] ripng 1 enable                       ! 同样必须将此接口加入 RIPng，否则路由传不出去
[R1-GigabitEthernet0/0/1] quit
```

### 第二阶段：配置 R 2（右侧路由器）

逻辑与 R 1 镜像对称。

Plaintext

```
<Huawei> system-view
[Huawei] sysname R2
[R2] ipv6

[R2] ripng 1
[R2-ripng-1] quit

[R2] interface GigabitEthernet 0/0/0
[R2-GigabitEthernet0/0/0] ipv6 enable
[R2-GigabitEthernet0/0/0] ipv6 address 2001:db8:12::2 64
[R2-GigabitEthernet0/0/0] ripng 1 enable
[R2-GigabitEthernet0/0/0] quit

[R2] interface GigabitEthernet 0/0/1
[R2-GigabitEthernet0/0/1] ipv6 enable
[R2-GigabitEthernet0/0/1] ipv6 address 2001:db8:2::254 64
[R2-GigabitEthernet0/0/1] ripng 1 enable
[R2-GigabitEthernet0/0/1] quit
```

### 第三阶段：连通性验证与排错三板斧

在答题纸上写完配置或在模拟器敲完回车后，必须通过以下步骤验证网络是否真正“活”了过来。

**第一斧：检查本端是否学到了远端路由**

在 R 1 上执行以下命令，查看全局 IPv 6 路由表：

Plaintext

```
[R1] display ipv6 routing-table protocol ripng
```

- **得分点确认：** 必须能看到目标网段 `2001:DB8:2::/64`。并且它的 `NextHop`（下一跳）显示的是 R 2 接口的 `FE80::` 链路本地地址，`Cost`（跳数）应为 `1` 或 `2`。
    

**第二斧：检查接口的 RIPng 报文收发状态**

如果在路由表里没看到路由，立刻检查接口状态：

Plaintext

```
[R1] display ripng 1 interface
```

- **得分点确认：** 确认 `G0/0/0` 和 `G0/0/1` 都在列表中，且状态正常，说明激活命令没有漏敲。
    

**第三斧：终极连通性测试 (Ping)**

模拟 PC 1 访问 PC 2（在路由器上带源地址进行 Ping 测试），这是证明网络连通的最直接证据：

Plaintext

```
[R1] ping ipv6 -a 2001:db8:1::254 2001:db8:2::254
```

- **注意：** 敲 `ping ipv6` 是华为设备的强制语法要求。如果回显 `5 packets transmitted, 5 packets received, 0.00% packet loss`，则代表配置完美无瑕，这道配置大题的分数已稳稳收入囊中。
