# 项目 7：园区网络综合实训 (考纲深度对标版)

> **项目定位**：本章是全书综合度最高的实训，旨在将前六个项目的零散技术（VLAN、OSPF、WLAN、ACL、NAT等）整合为一套完整的企业级解决方案。

---

## 一、 核心学习目标 (考纲提炼)
1. **全设备管理**：熟练操作交换机、路由器、防火墙、AC/AP等设备。
2. **三层冗余架构**：掌握 MSTP + VRRP 实现网关冗余与负载均衡。
3. **多协议路由**：掌握静态路由与 OSPF 的混合应用。
4. **全业务接入**：实现 DHCP 自动分配、WLAN 无线覆盖及 NAT 互联网出口。
5. **边界安全控制**：应用防火墙安全策略与 PPP 广域网安全认证。

---

## 二、 考核知识点精讲

### 1. 网络设备管理与维护 (识记/应用)
*   **状态检验**：
    *   `display ip interface brief`：检查接口 IP 和物理状态。
    *   `ping -a [源IP] [目的IP]`：指定源地址进行测试，排查回程路由故障。
    *   `tracert`：追踪路由，是定位园区网跳数过多、路由环路问题的必用工具。
*   **配置文件管理**：
    *   **备份**：通过 `tftp` 或 `ftp` 将 `vrpcfg.zip` 上传至服务器。
    *   **还原**：从服务器下载配置，并使用 `startup saved-configuration [文件名]` 指定下次启动文件。

### 2. 负载均衡的园区网络 (重点/难点)
> **考核核心**：如何利用二三层技术协同，让不同 VLAN 的流量走不同的核心交换机。

*   **MSTP (多生成树协议)**：
    *   **领会**：通过多实例将不同 VLAN 流量映射到不同实例，打破环路的同时实现负载分担。
    *   **应用**：域内 `region-name`、`revision-level`、`instance vlan` 必须全域一致。
*   **VRRP (虚拟路由冗余协议)**：
    *   **识记**：多台设备虚拟一个 IP 地址，实现网关备份。
    *   **应用**：通过调整 `vrrp vrid [id] priority`（默认100，越大越优）来确定 Master 角色。
*   **协同逻辑**：
    *   **关键原则**：实例（Instance）的根桥与 VRRP 的 Master 设备必须物理一致。
    *   **目的**：避免数据流在二层交换机和核心交换机之间产生次优路径或往返绕路。

### 3. 企业网综合组网技术 (应用/领会)
*   **DHCP 服务**：
    *   三层架构中，DHCP 通常部署在核心层。
    *   **DHCP 中继**：当 DHCP 服务器与终端不在同一 VLAN 时，必须在 Vlanif 接口配置 `dhcp select relay`。
*   **出口 NAT**：
    *   **Easy-IP**：适用于公网 IP 固定的场景（直接使用出接口 IP）。
    *   **NAT Server**：将内网服务器（80、21端口等）发布到互联网。
*   **无线网络 (WLAN)**：
    *   **CAPWAP 隧道**：AC 管理 AP 的核心协议。
    *   **配置流**：创建 SSID 模板 -> 安全模板 -> VAP 模板 -> 绑定 AP。

## 三、 本章重点：综合实训配置标准流程

| 阶段 | 核心任务 | 关键配置命令 |
| :--- | :--- | :--- |
| **阶段 1：二层互通** | VLAN、Trunk、链路聚合、**MSTP** | `port link-type trunk`, `stp region-conf` |
| **阶段 2：三层网关** | Vlanif IP、**VRRP**、DHCP 中继 | `vrrp vrid 1 virtual-ip`, `dhcp select relay` |
| **阶段 3：内网路由** | **OSPF** 区域划分、通告内网段 | `ospf 1 area 0`, `network 192.168.0.0 0.0.255.255` |
| **阶段 4：出口安全** | 静态默认路由、**NAT**、ACL 访问控制 | `ip route-static 0.0.0.0 0.0.0.0`, `nat outbound` |
| **阶段 5：无线与防火墙** | AC 业务下发、防火墙 **安全区域** 策略 | `capwap source interface`, `security-policy` |

## 四、 故障排除 Checklist (难点攻克)

> **考纲建议**：遇到复杂故障，需从物理层、链路层、网络层、应用层等多角度分析。

1.  **终端不通网关**：
    *   检查二层 VLAN 是否透传。
    *   检查 VRRP 状态是否为 Master/Backup。
    *   检查终端 IP/掩码/网关配置是否正确。
2.  **内网通但上不了网**：
    *   检查出口路由器是否有默认路由指向 ISP。
    *   检查 NAT 配置中绑定的 ACL 是否正确包含了内网地址。
    *   检查出口链路是否配置了 PPP CHAP 认证且认证通过。
3.  **无线搜索不到 SSID**：
    *   检查 AP 是否在 AC 上成功上线 (`display ap all`)。
    *   检查 AC 上的 VAP 状态是否为 Up。

## 五、 重点词汇与概念
*   **Eth-Trunk (链路聚合)**：逻辑捆绑物理链路，增加带宽并备份。
*   **SVI / Vlanif**：实现 VLAN 间通信的三层逻辑接口。
*   **CHAP (挑战握手认证协议)**：PPP 中安全性较高的三次握手认证。
*   **五元组**：源IP、源端口、目的IP、目的端口、协议号（ACL过滤的核心依据）。
