# 核心考点笔记：DHCP 接口地址池 (Interface Address Pool)

## 一、什么是接口地址池？

接口地址池是指在路由器的指定接口下直接开启 DHCP 功能。它不需要像全局地址池那样起名字、划网段，而是直接**“就地取材”**。

- **核心逻辑：** 路由器直接利用该接口当前的 **IP 地址**和**子网掩码**，自动创建一个地址池。
    
- **分配范围：** 接口所属网段内，除去接口 IP 以外的所有可用地址。
    
- **默认网关：** 自动将该**接口自身的 IP** 作为分配给下属 PC 的默认网关。
    

## 二、接口地址池 vs 全局地址池 (考试对比项)

|**特性**|**接口地址池 (Interface)**|**全局地址池 (Global)**|
|---|---|---|
|**配置复杂度**|极简，几行代码搞定|较复杂，需定义 Pool 名称、Network 等|
|**适用场景**|接口直接连接终端的小型局域网|复杂网络、DHCP 中继、跨网段分配|
|**网关设置**|自动关联接口 IP|需手动敲 `gateway-list` 指定|
|**管理效率**|只能在本接口生效|可以在多个不同的接口调用同一个池子|

## 三、接口地址池必考配置代码 (华为 VRP)

假设 R 1 的 `GE0/0/0` 接口连接着你正在设计的地图开发小组，接口 IP 为 `192.168.1.254/24`。

### 1. 基础开启流程

Plaintext

```
[R1] dhcp enable                            // 1. 全局开启 DHCP 服务
[R1] interface GigabitEthernet 0/0/0         // 2. 进入目标接口
[R1-GigabitEthernet0/0/0] ip address 192.168.1.254 24 // 3. 配置接口IP（即未来的网关）
[R1-GigabitEthernet0/0/0] dhcp select interface       // 4. 开启接口地址池 (核心命令)
```

### 2. 接口池下的高级控制 (常用填空题考点)

有时候你并不想把整个网段都分出去（比如要预留几个固定 IP 给服务器），或者想改租期。

Plaintext

```
// 排除掉不参与分配的 IP（例如 1.1 到 1.10）
[R1-GigabitEthernet0/0/0] dhcp server excluded-ip-address 192.168.1.1 192.168.1.10

// 修改租期（默认是 1 天，改为 2 天 12 小时）
[R1-GigabitEthernet0/0/0] dhcp server lease day 2 hour 12

// 指定分配的 DNS 服务器
[R1-GigabitEthernet0/0/0] dhcp server dns-list 8.8.8.8
```

## 四、故障排查与验证

在 eNSP 实验中，配置完后如果在 PC 上获取不到 IP，请按此顺序检查：

1. **`display this`**：在接口下检查是否有 `dhcp select interface`。
    
2. **`display ip pool interface gigabitethernet 0/0/0`**：查看该接口池的具体状态（有多少地址被租出去了，有没有冲突）。
    
3. **`display dhcp statistics`**：查看 DHCP 报文统计（看看有没有收到客户端发的 Discover 包）。
