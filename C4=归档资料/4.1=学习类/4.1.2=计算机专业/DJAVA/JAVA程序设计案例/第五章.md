# 1. 面向对象特征

## 识记：

### 三个特征:封装性、多态性、继承性。

- **封装性 (Encapsulation)**

- **继承性 (Inheritance)**

- **多态性 (Polymorphism)**

## 领会：

### 三个特征含义及作用。

**A. 封装性 (Encapsulation)**

- **含义**：将对象的属性和方法**包装**在一起，**隐藏**内部实现细节，只对外提供公共的访问接口（如 get/set 方法）。

- **作用**：

- **安全性**：保护数据，防止外部随意修改（比如年龄不能设为负数）。

- **易维护**：修改内部逻辑不影响外部使用者。

- *比喻*：像**ATM机**，你看不见里面的钱和电路（隐藏细节），只能通过插卡口和键盘操作（对外接口）。

**B. 继承性 (Inheritance)**

- **含义**：子类自动拥有父类的非私有属性和方法。

- **作用**：

- **代码复用**：相同的代码写在父类里，子类直接拿来用，不用重写。

- **扩展性**：子类可以在父类基础上增加新功能。

- *比喻*：**子承父业**。你继承了父亲的房子（复用），也可以自己再买辆车（扩展）。

**C. 多态性 (Polymorphism)**

- **含义**：**同一个行为**（方法），作用于**不同的对象**上，产生**不同的结果**。

- **作用**：

- **灵活性**：代码更加通用。

- **解耦**：降低代码之间的紧密程度，方便扩展。

- **前提**：必须要有**继承**关系和**方法重写**。

- *比喻*：**花木兰**。在家是女儿（一种形态，织布）；

- 在战场是将军（另一种形态，杀敌）。

- 或者：按下“打印”键（同一个行为），彩色打印机打出彩色，黑白打印机打出黑白（不同的结果）。

# 2. 类与对象

## 识记：

### 类的结构及相关关键字表示，class、new、构造方法、this、static等概念；

- **类 (Class)**：对象的**模板**（图纸）。使用 class 关键字定义。

- **对象 (Object)**：类的**实例**（成品）。使用 new 关键字创建。

- **构造方法 (Constructor)**：

- **作用**：初始化对象。

- **特征（必考）**：**方法名与类名相同**，且**没有返回值类型**（连 void 都不能写）。

- **this**：代表**当前对象**（谁调用这个方法，this 就代表谁）。

- **static**：代表**静态**的，属于**类**的，被所有对象**共享**。

## 领会：

### 类的设计方法、对象的创建和使用。

- **设计公式**：**类 = 属性（成员变量） + 行为（成员方法）**。

- **对象创建公式**：

- 类名 对象名 = new 构造方法();

- *例*：Student s1 = new Student();

- **对象使用**：

- 使用 **点 (****.****)** 操作符来访问属性或方法。

- *例*：s1.name = "Tom"; 或 s1.study();

## 应用：

### 能根据应用场景完成类设计与使用，包括：定义类；

**A. **

**解决重名**

```java
public void setName(String name) {
    this.name = name; // 左边是成员变量，右边是参数
}
//调用本类构造器：this()。必须放在构造方法的第一行。
```

**B. **

- **静态变量**：static int count;

- **特点**：全类共享一份。

- **调用**：推荐用 类名.变量名（如 Student.count）。

- **静态方法**：public static void show() {}

- **特点**：工具方法（如 Math.random()）。

- **铁律（必考）**：**静态方法只能访问静态成员**（Static 不能直接访问非 Static，因为还没对象呢）。

- **静态代码块**：static { ... }

- **时机**：类加载时执行，**只执行一次**。通常用于加载驱动或初始化资源。

**C. 访问权限修饰符 (Access Modifiers)**

请背诵这个权限从

- **private**：**私有**。只有**本类**能访问。（封装的核心）。

- **default** (不写)：**默认**。只有**本包**能访问。

- **protected**：**受保护**。本包 + **外包子类**能访问。

- **public**：**公共**。**所有地方**都能访问。

### 创建对象；使用对象；this的使用；static定义静态变量、静态方法与静态代码块；使用权限声明变量和方法。

```java
// 1. 定义类
public class Student {
    // 2. 成员变量 (通常 private 封装)
    private String name;
    public static String school = "清华大学"; // 静态变量，大家共用

    // 3. 构造方法 (无返回值，名同类)
    public Student(String name) {
        this.name = name; // this 区分重名
    }

    // 4. 成员方法
    public void introduce() {
        System.out.println("我是" + this.name + "，学校是" + school);
    }
    
    // 5. 静态代码块
    static {
        System.out.println("类加载了，只打一次");
    }
}

// 6. 静态方法
// Student.school 访问静态变量
// new Student("张三").introduce() 创建并使用对象
```

# 3. 继承与多态

## 识记：

### 继承与多态的概念及应用场景；**extends继承、super等关键字。**

- **继承 (Inheritance)**

- **概念**：子类继承父类的特征和行为（代码复用）。

- **关系**：**IS-A** 关系（Cat **is a** Animal）。

- **关键字**：**extends**。

- **限制**：Java 只支持**单继承**（一个儿子只能有一个亲爹），但支持多层继承。

- **多态 (Polymorphism)**

- **概念**：同一个行为具有多个不同表现形式或形态的能力。

- **关键字**：**super**（代表父类引用）。

## 领会：

### 子类的设计方法；**向上转型。**

**A. 向上转型 (Upcasting) —— 多态的核心**

- **格式**：**父类类型 变量名 = new 子类类型();**

- *例*：Animal a = new Dog();

- **作用**：提高程序的扩展性。方法参数写父类，传子类对象也能接收。

- **后果**：只能调用父类有的方法，**不能**调用子类特有的方法（除非强转回去）。

**B. 子类设计**

- **原则**：将通用的属性/方法提取到父类，子类只写自己特有的，或者重写父类的。

## 应用：

### 能根据特定场景进行子类设计，包括：使用extends定义子类；使用super调用父类方法和成员变量；

```java
class Dog extends Animal { // 1. 定义子类
    public Dog() {
        super(); // 2. 调用父类构造方法 (必须在第一行)
    }
    
    public void eat() {
        super.eat(); // 3. 调用父类普通方法
        System.out.println("狗吃骨头");
    }
}
```

### **方法重写 vs 方法重载**

| 特性 | 方法重载 | 方法重写 | 
| -- | -- | -- |
| 别名 | 静态多态 | 动态多态 | 
| 位置 | 同一个类中 | 父子类之间 | 
| 方法名 | 必须相同 | 必须相同 | 
| 参数列表 | 必须不同 | 必须相同 | 
| 返回值 | 无关 | 必须相同 (或兼容) | 
| 权限 | 无关 | 子类权限>=父类 | 


### instanceof运算符的使用方法。

- **作用**：判断一个对象是否属于某种类型（通常在向下转型前用来保命，防止报错）。

- **格式**：对象 instanceof 类名 （返回 boolean）。

- **代码**：

```java
Animal a = new Dog(); // 向上转型
// a.watchDoor(); // 报错！因为 Animal 没这个方法

if (a instanceof Dog) { // 先判断
    Dog d = (Dog) a;    // 再向下强转
    d.watchDoor();      // 安全调用子类特有方法
}
```
