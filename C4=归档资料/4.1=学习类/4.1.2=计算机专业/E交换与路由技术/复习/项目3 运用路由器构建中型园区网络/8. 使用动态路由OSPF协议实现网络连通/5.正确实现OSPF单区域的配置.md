### 一、华为 R 1 路由器完整配置

Plaintext

```
<Huawei> system-view                                   ! 进入系统视图
[Huawei] sysname R1                                    ! 修改设备名称为 R1

! 1. 配置 LoopBack 接口（推荐用于稳定 OSPF Router-ID）
[R1] interface LoopBack 0
[R1-LoopBack0] ip address 1.1.1.1 255.255.255.255
[R1-LoopBack0] quit

! 2. 配置物理接口 IP 地址
[R1] interface GigabitEthernet 0/0/0
[R1-GigabitEthernet0/0/0] ip address 10.1.1.1 255.255.255.252
[R1-GigabitEthernet0/0/0] quit
[R1] interface GigabitEthernet 0/0/1
[R1-GigabitEthernet0/0/1] ip address 192.168.1.254 255.255.255.0
[R1-GigabitEthernet0/0/1] quit

! 3. 核心步骤：配置 OSPF
[R1] ospf 1 router-id 1.1.1.1                          ! 启动 OSPF 进程 1，并绑定 Router-ID
[R1-ospf-1] area 0                                     ! 进入骨干区域 0
[R1-ospf-1-area-0.0.0.0] network 10.1.1.0 0.0.0.3      ! 宣告直连互联网段（注意使用反掩码）
[R1-ospf-1-area-0.0.0.0] network 192.168.1.0 0.0.0.255 ! 宣告局域网段
[R1-ospf-1-area-0.0.0.0] quit
[R1-ospf-1] silent-interface GigabitEthernet 0/0/1     ! （可选/推荐）将连接 PC 的接口设为静默接口
[R1-ospf-1] quit

[R1] quit
<R1> save                                              ! 保存配置（重要）
```

### 三、华为专属 OSPF 验证与排错命令清单

在华为设备上完成配置后，请严格按照以下顺序进行验证（这也是实验排错的标准流程）：

**1. 检查 OSPF 邻居关系是否建立（最关键）**

Plaintext

```
<R1> display ospf peer brief
```

- **查看重点：** 状态（State）必须是 **`Full`**。如果是 `Init`、`2-Way` 或 `ExStart` 停滞，说明参数不匹配（如 MTU、掩码、认证或 Router-ID 冲突）。
    

**2. 检查 OSPF 路由表（看是否学到了对方的网段）**

Plaintext

```
<R1> display ip routing-table protocol ospf
```

- **查看重点：** 是否出现了到达 `192.168.2.0/24` 的路由，且下一跳（NextHop）指向 R 2 的接口 IP `10.1.1.2`。华为 OSPF 内部路由的默认优先级（Preference）为 **10**。
    

**3. 检查 OSPF 接口参数与状态**

Plaintext

```
<R1> display ospf interface
```

- **查看重点：** 可以看到接口的 Cost（开销值）、DR/BDR 角色、以及 Hello/Dead 定时器时间。
    

**4. 检查 LSDB（链路状态数据库）**

Plaintext

```
<R1> display ospf lsdb
```

- **查看重点：** 用于查看本路由器收集到了哪些 LSA（如 Router LSA、Network LSA）。这在进阶的多区域排错中非常有用。
