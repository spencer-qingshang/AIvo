在华为交换机上配置 RSTP，我们的目标是：**开启协议、确立老大、优化边缘**。

以下是标准的三步走配置手册：

### 1. 第一步：全局开启并切换模式

华为交换机默认运行的是 MSTP（多生成树），为了兼容性或特定要求，我们通常需要明确指定模式为 RSTP。

```bash
[SW1] stp mode rstp          # 将生成树模式切换为快速生成树
[SW1] stp enable             # 全局开启 STP（默认已开启，但建议确认）
```

### 2. 第二步：手动干预“老大”选举（确立根桥）

如果不配置，交换机会根据 MAC 地址随机选出根桥。在实际工程中，我们必须让最核心、性能最强的交换机当老大。

- **方法 A（指定等级）**：
  ```bash
  [SW1] stp root primary     # 将此交换机设为“主根桥”（优先级自动变为 0）
  [SW2] stp root secondary   # 将此交换机设为“备份根桥”（优先级自动变为 4096）
  ```
- **方法 B（指定具体优先级）**：
  ```bash
  [SW1] stp priority 4096    # 优先级必须是 4096 的倍数，越小越优先
  ```

### 3. 第三步：性能优化（配置边缘端口）

这是 RSTP 实战中最容易被忽视但**最能体现专业水平**的一步。

- **操作对象**：所有直接连接 PC、服务器、路由器的接口。
- **作用**：让这些口跳过生成树计算，插上线秒通。
- **配置命令**：
  ```bash
  [SW1] interface GigabitEthernet 0/0/1
  [SW1-GigabitEthernet0/0/1] stp edged-port enable
  ```

---

### 4. 验证命令：如何看懂生成树状态？

在 eNSP 中，配置完后必敲的检查命令：

- **查看生成树摘要**：
  `display stp brief`
  - **重点观察**：
    - `Role`：`ROOT`（根口）、`DESI`（指定口）、`ALTE`（阻塞口）。
    - `STP State`：`FORWARDING`（转发）或 `DISCARDING`（丢弃）。

- **查看谁是根桥**：
  `display stp`
  - 寻找 `CIST Bridge`（本设备 ID）和 `CIST Root`（根桥 ID）。如果两者一样，说明你成功成为了全网老大。

---

### 5. 考点挑战：排错实战

**场景**：
你在 SW1 和 SW2 之间配置了 RSTP，两台交换机之间连了两根线。你希望 SW1 成为根桥，并且希望阻塞掉 GE0/0/2 这条物理链路。

**请写出关键配置思路：**

**标准答案：**
1.  **确定根桥**：在 SW1 上执行 `stp root primary`。
2.  **人工干预阻塞点**：
    - 由于 SW1 是根桥，它的端口全是指定端口（DP），不会被阻塞。
    - 阻塞点必然在 SW2 上。
    - 进入 SW2 的 GE0/0/2 接口，手动调大路径开销：`stp cost 200000`。
    - **逻辑**：RSTP 会自动选择开销大的那条路径进行阻塞（Discarding）。

---

**实战逻辑总结**

在综合实验中，RSTP 的配置顺序通常是：
**全网改模式** -> **核心定主备** -> **接入设边缘**。

掌握了防环技术，你的二层网络已经坚如磐石。但如果网络太稳了，导致所有流量都走同一条路，带宽利用率又不高了。为了在稳的基础上更进一步，我们需要学习 **项目 2.7：提高网络稳定性（VRRP）**，实现网关级别的冗余和负载均衡。
