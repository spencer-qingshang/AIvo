# 《交换与路由技术 (13833)》必背考点精华手册 (大纲深度还原版)

## 一、 识记类核心考点 (常考：单选/填空)

### 1. 基础概念
*   **VRP (Versatile Routing Platform)**：通用路由平台。
*   **eNSP**：华为企业网络仿真平台。
*   **设备管理**：首次配置用 **Console**，远程管理用 **Telnet/SSH**。
*   **配置存盘**：必须在用户视图 `< >` 使用 `save`。
*   **IP 地址冲突**：指局域网内不同设备分配了相同的 IP。
*   **DNS 作用**：将域名解析为 IP 地址。

### 2. 交换机与端口
*   **VLAN Tag (802.1Q)**：插入在原始以太网帧中的 **4 字节**。
*   **VLAN ID 范围**：1~4094。
*   **链路类型**：Access 链路（连电脑）、Trunk 链路（连交换机）。
*   **端口双工模式**：Full-duplex (全双工，同时收发)；Half-duplex (半双工)。

## 二、 领会类核心考点 (常考：简答)

### 1. VLAN 端口收发报文的处理方式 (必背！)
*   **Access 端口**：
    *   **接收**：收到不带 Tag 帧，打上 PVID 接收。
    *   **发送**：剥掉 Tag 后发送给 PC。
*   **Trunk 端口**：
    *   **接收**：带 Tag 且在允许列表中则接收；不带 Tag 则打上 PVID 接收。
    *   **发送**：VLAN ID 等于 PVID 时剥掉 Tag 发送；不等则带 Tag 发送。

### 2. 生成树协议 (STP/RSTP)
*   **原理**：通过逻辑阻塞特定端口，将环路网络变成树状，消除广播风暴。
*   **RSTP 优势**：收敛速度比 STP 快得多（秒级）。

### 3. 三层交换机与路由器的区别
*   **三层交换机**：硬件转发（ASIC芯片），效率高，“一次路由，多次交换”，主要用于园区网内部。
*   **路由器**：功能丰富（NAT/VPN/安全），软件转发，主要用于广域网连接或企业出口。

## 三、 应用类：全网互通综合应用题 (必背标准答案)

### 题目要求：实现全网 OSPF 互通
*(规划参考：R1 连接网段 192.168.1.0/24，R2 连接网段 192.168.2.0/24)*

#### 1. 规划表设计 (得分项)
| 节点 | 接口 | IP 地址/掩码 | 默认网关 |
| :--- | :--- | :--- | :--- |
| **PC1** | 网卡 | 192.168.1.10 /24 | 192.168.1.1 |
| **PC2** | 网卡 | 192.168.2.10 /24 | 192.168.2.1 |
| **R1** | GE0/0/0 | 192.168.1.1 /24 | - |
| **R1** | GE0/0/1 | 10.1.1.1 /24 | - |
| **R2** | GE0/0/0 | 192.168.2.1 /24 | - |
| **R2** | GE0/0/1 | 10.1.1.2 /24 | - |

#### 2. 配置过程详解 (按步骤得分)

**步骤一：配置接口 IP 地址**
```shell
# 路由器 R1
<Huawei> system-view
[Huawei] sysname R1
[R1] interface GigabitEthernet 0/0/0
[R1-GigabitEthernet0/0/0] ip address 192.168.1.1 24
[R1] interface GigabitEthernet 0/0/1
[R1-GigabitEthernet0/0/1] ip address 10.1.1.1 24
```

**步骤二：配置 OSPF 协议实现互通**
```shell
# R1 配置过程
[R1] ospf 1 router-id 1.1.1.1             # 1.启动进程并设ID
[R1-ospf-1] area 0                        # 2.进入骨干区域
[R1-ospf-1-area-0.0.0.0] network 192.168.1.0 0.0.0.255 # 3.宣告业务网段
[R1-ospf-1-area-0.0.0.0] network 10.1.1.0 0.0.0.255    # 4.宣告互联链路
```
*(注意点：network 后紧跟的是 **反掩码**，即 255.255.255.0 写作 0.0.0.255)*

#### 3. 故障排除常用命令
*   `display ip routing-table`：查看路由表，确认是否有 `OSPF` 标志的路径。
*   `ping [目标IP]`：验证端到端连通性。
*   `tracert [目标IP]`：追踪路径，确定数据包在哪一跳中断。
