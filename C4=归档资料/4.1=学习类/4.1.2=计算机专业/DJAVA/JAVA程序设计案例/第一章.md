# java语言发展

## 识记：

### 计算机语言分类；

**机器语言：** 计算机能直接识别和执行的二进制指令。

**汇编语言：** 使用助记符代替机器指令，比机器语言易读，但仍与硬件紧密相关。

**高级语言：** 更接近自然语言和数学表达式，抽象程度高，与硬件无关，例如 Java。

### java发展史与特点；

针对考试简答题，最核心的考点通常是

**一、Java 发展史 (3个考点)**

- 出身： 由 Sun 公司推出，James Gosling（Java之父）领导开发。

- 前身： 原名 Oak（1991年），1995年正式改名为 Java。

- 归属： 2009年被 Oracle (甲骨文) 公司收购。

**二、Java 主要特点 (6个关键词)**

- **简单易学**： 也就是“C++--”，去掉了指针和多继承等复杂内容。

- **混合型（半编译、半解释）**：Java 首先将源代码（.java）**编译**成与平台无关的字节码（.class），然后由 JVM（虚拟机）对字节码进行**解释**执行。为了提升性能，现代 JVM 会使用 **JIT (Just-In-Time) 编译器**将热点代码直接编译成机器码运行。

- **面向对象**： 纯面向对象，支持封装、继承、多态。

  - **封装**：将数据和操作数据的方法捆绑在一起，隐藏内部细节。

  - **继承**：允许一个类获取另一个类的属性和方法，实现代码复用。

  - **多态**：允许不同类的对象对同一消息做出不同的响应。

- **平台无关性 (跨平台)**： 核心卖点。一次编译，到处运行 (依靠不同系统的 JVM 实现)。

- **安全稳健**： 拥有垃圾回收机制 (GC)，无指针操作，强异常处理。

- **多线程**： 内置多线程支持，可同时处理多个任务。

### 了解java体系。

- **Java SE (标准版/Standard Edition)**：

- **地位**：是三个平台的**核心和基础**。

- **用途**：用于开发桌面应用、服务器及嵌入式环境。

- **包含**：JDK、JRE 以及核心类库（如 UI、集合、I/O、线程等）。

- **Java EE (企业版/Enterprise Edition)**：

- **用途**：解决**企业级应用**的开发、部署和管理等复杂问题。

- **关系**：在保留 Java SE 特性的基础上，增加了 **Servlet、JSP、EJB、XML** 等技术支持。

- **Java ME (微型版/Micro Edition)**：

- **用途**：专为**嵌入式消费电子设备**（如机顶盒、手机、PDA）提供。

- **特点**：包含虚拟机和一系列标准化的 Java API，拥有自己的类库和安全模型。

# java开发环境

## 识记：

### jdk概念。

- **定义**：**Java 开发工具包**。它是开发 Java 程序的核心组件。

- **开发工具**：包括**编译工具**、**运行工具**、文档生成工具、文件打包工具等。

## 领会：	

jdk主要目录结构。

**1. bin (Binary)**

- **功能**：存放**可执行程序**（各种命令）。

- **核心文件**：javac.exe：**Java 编译器**（把代码变成字节码）。java.exe：**Java 运行程序**（启动 JVM 跑程序）。javadoc.exe：**文档生成器**。

classpath、path环境变量配置;

- **Path 环境变量的作用**：让操作系统能在任何目录下找到 `javac` 和 `java` 等命令。
- **Classpath 环境变量的作用**：告诉 Java 虚拟机去哪里找程序运行时所需的类文件（虽然现代 Java 开发中，更多是通过构建工具自动管理）。

**2. db (Database)**

- **功能**：存放 Java 自带的**数据库管理系统**。

- **用途**：用于学习 **JDBC**（数据库互连）时使用，免去安装额外数据库的麻烦。

**3. include**

- **功能**：存放 **C 语言头文件**。

- **原因**：JDK 底层是通过 C 和 C++ 实现的，启动时需要引入这些文件。

**4. jre (Java Runtime Environment)**

- **功能**：**Java 运行环境**的根目录。

- **包含**：**Java 虚拟机 (JVM)** 和程序运行时所需的类库。

**5. lib (Library)**

- **功能**：存放 **Java 类库文件**。

### JDK、JRE与JVM的关系

- **JVM (Java Virtual Machine)**：Java虚拟机，是程序运行的核心，负责执行字节码。
- **JRE (Java Runtime Environment)**：Java运行环境，**包含JVM**和程序运行所需的核心类库。只安装JRE可以运行Java程序。
- **JDK (Java Development Kit)**：Java开发工具包，**包含JRE**和编译器（javac）等开发工具。开发Java程序需要安装JDK。
- **关系总结**：**JDK > JRE > JVM**。

## 应用：

### jdk安装与环境变量配置（Windows）

在命令行使用`javac`和`java`命令前，必须正确安装JDK并配置环境变量。

**1. 下载JDK**

-   **官方渠道**: 前往 Oracle 官网的 Java 下载页面。
-   **版本选择**: 选择一个长期支持（LTS）版本，如 Java 11, 17 或 21。初学者推荐 Java 17。
-   **系统**: 选择 Windows x64，下载 `.exe` 格式的安装程序。

**2. 安装JDK**

-   双击下载的 `.exe` 文件。
-   遵循安装向导，一直点击“下一步”即可。
-   **重要**: 记住你的JDK安装路径，例如 `C:\Program Files\Java\jdk-17`。这个路径稍后要用。

**3. 配置环境变量**

这是最关键的一步，目的是让操作系统能在任何位置找到 JDK 的命令。

-   在 Windows 搜索框中搜索“编辑系统环境变量”并打开它。
-   在弹出的“系统属性”窗口中，点击右下角的“环境变量”按钮。

-   **配置 `JAVA_HOME` (非常重要)**
    -   在“系统变量”区域，点击“新建”。
    -   **变量名**: `JAVA_HOME`
    -   **变量值**: 填入你刚才记住的JDK安装路径（例如 `C:\Program Files\Java\jdk-17`）。
    -   **作用**: `JAVA_HOME` 是一个行业标准，很多Java相关的工具（如Maven, Tomcat）都会通过它来找到JDK的位置。

-   **配置 `Path`**
    -   在“系统变量”区域，找到并双击 `Path` 变量。
    -   在弹出的编辑窗口中，点击“新建”。
    -   输入 `%JAVA_HOME%\bin`。
    -   **作用**: `bin` 目录存放了 `javac.exe` 和 `java.exe` 等核心命令。将这个路径加入 Path，可以让我们在任何目录下直接从命令行调用这些命令。

**4. 验证安装**

-   打开一个新的命令行窗口（CMD或PowerShell）。
-   分别输入 `java -version` 和 `javac -version`。
-   如果都成功显示版本号，说明你的JDK已安装并配置成功！

**关于 `CLASSPATH` 的说明**

-   在早期的Java版本中，你需要手动配置 `CLASSPATH` 环境变量来告诉JVM去哪里寻找 `.class` 文件。
-   **现在基本不再需要手动配置**。从 Java 5 开始，JVM 默认会在当前目录 (`.`) 下寻找类文件。而使用 Eclipse、IDEA 等现代 IDE 时，它们会自动管理项目的 `classpath`，你完全无需关心。

# java程序编写

## 识记：

### java类结构；

- **基本单位**：Java 程序是以 **类 (class)** 为基本单位的。

- **定义格式**：使用关键字 class 定义，代码块必须包裹在 **大括号 ****{}** 中。

- **文件名规则（必考）**：如果一个类被声明为 public，那么**文件名必须与类名完全一致**（包括大小写），扩展名为 .java。

- **数量限制**：一个源文件中可以有多个类，但**只能有一个 ****public**** 类**。这是因为 `public` 类需要被包外的代码访问，Java 规定文件名必须与 `public` 类名相同，以便 JVM 能够快速定位。

```java
// 文件名必须为 MyPublicClass.java
public class MyPublicClass {
    // ...
}

// 这是允许的，因为 Dog 类不是 public 的
class Dog {
    // ...
}

/*
// 这是错误的，会导致编译失败！
// 一个 .java 文件中不能有两个 public 类。
public class MyOtherPublicClass {
    // ...
}
*/
### main方法;

- **地位/作用**：程序的**入口**（Entry Point）。JVM（虚拟机）从这里开始执行代码。

- **标准格式（一字不差背下来）**：public static void main(String[] args)

- **关键词拆解（常考选择/填空）**：

- public：**访问修饰符**，表示公开，JVM 才有权限调用。

- static：**静态**，表示不需要创建对象就可以直接运行。

- void：**返回值类型**，表示该方法执行完没有任何返回值。

- main：**方法名**，特定名称，JVM 认准这四个字母。

- String[] args：**参数**，一个字符串数组，用于接收命令行参数。下面的例子演示了如何使用它。

**代码 (`ShowArgs.java`):**
```java
public class ShowArgs {
    public static void main(String[] args) {
        // 检查用户是否传递了参数
        if (args.length == 0) {
            System.out.println("您没有输入任何参数。");
            return; // 结束程序
        }

        System.out.println("您总共输入了 " + args.length + " 个参数，它们分别是：");

        // 遍历并打印所有参数
        for (int i = 0; i < args.length; i++) {
            System.out.println("参数 " + (i + 1) + ": " + args[i]);
        }
    }
}
```

**如何编译和运行：**

1.  **编译**：`javac ShowArgs.java`
2.  **运行（带参数）**：`java ShowArgs Hello World 123`
3.  **预期输出**：
    ```
    您总共输入了 3 个参数，它们分别是：
    参数 1: Hello
    参数 2: World
    参数 3: 123
    ```

## 领会：

### 类的概念;

- **地位**：Java 程序的**基本单位**（Java 程序是由一个个类组成的）。
- **本质**：类是**对象的模板**（或图纸），是对现实生活中事物的抽象。它描述了一类事物共同的**属性（成员变量）**和**行为（成员方法）**。
- **组成**：主要包含**成员变量**（属性/状态）和**成员方法**（行为/功能）。
- **关键词**：使用 **class** 关键字定义。

**示例：将“类”和“对象”具体化**

把**类（Class）**想象成一个“汽车设计图”，它定义了汽车应该有什么。
把**对象（Object）**想象成根据这张图纸制造出来的“一辆辆具体的汽车”，它们是实实在在的个体。

```java
// 1. 定义一个 "Person" 类，这就是“模板”
class Person {
    // 成员变量 (属性)
    String name;
    int age;

    // 成员方法 (行为)
    void sayHello() {
        System.out.println("大家好，我叫 " + name + "，今年 " + age + " 岁。");
    }
}

// 2. 在主程序中使用这个模板来创建具体的“对象”
public class School {
    public static void main(String[] args) {
        // 根据 Person 模板，创建一个叫 p1 的具体的人（对象）
        Person p1 = new Person();
        
        // 给这个对象的属性赋值
        p1.name = "张三";
        p1.age = 18;
        
        // 调用这个对象的行为
        p1.sayHello(); // 输出: 大家好，我叫 张三，今年 18 岁。

        // 还可以根据同一个模板，创建另一个不同的对象
        Person p2 = new Person();
        p2.name = "李四";
        p2.age = 20;
        p2.sayHello(); // 输出: 大家好，我叫 李四，今年 20 岁。
    }
}
```
这个例子清晰地展示了：`Person` 是**类/模板**，而 `p1` 和 `p2` 是两个独立的**对象/实例**。

### main方法作用；

- **核心作用**：程序的**入口**（Entry Point）。没有 main 方法，Java 程序无法独立运行。

- **调用者**：由 **JVM (Java 虚拟机)** 直接调用。

- **特性**：

- **public**：保证 JVM 能从外部访问。

- **static**：保证 JVM 不用创建对象就能直接调用。

- **void**：执行结束不需要给 JVM 返回结果。

### java程序编写原则；

- **大小写敏感**：Java 严格区分大小写

- **命名规则**：

- **类名**：首字母**大**写。

- **方法/变量名**：首字母**小**写，后面单词首字母大写

- **文件名一致性**：public 修饰的类名必须和**文件名**完全一致（包括大小写）。

- **符号规范**：所有的标点符号（分号;、括号()、引号""）必须是**英文半角**状态，**严禁使用中文标点**。

- **层级结构**：类包裹方法，方法包裹语句。代码块必须用 **{}** 配对。

- **语句结束**：每条可执行语句必须以**分号 ****;** 结尾。

## 应用：

### 类文件编写方法;

- **源文件后缀**：必须是 **.java**。

- **文件名规则**：如果类被 public 修饰，**文件名必须与类名完全一致**（区分大小写）。

- **基本结构**：

```java
public class Hello {  // 类名
    public static void main(String[] args) { // main 方法
        System.out.println("Hello");
    }
}
```

### 编译命令javac；

- **作用**：将**源文件** (.java) 翻译成 **字节码文件** (.class)。

- **命令格式**：**javac 文件名.java**

- **考点细节**：命令后面**必须带后缀名 ****.java**。javac 的 c 代表 **Compiler (编译器)**。编译成功后，会生成同名的 .class 文件（给机器看的）。

### 运行java类命令 java。

- **作用**：启动 JVM，加载字节码文件并运行程序。

- **命令格式**：**java 类名**

- **考点细节（必考坑点）**：命令后面**绝对不能带后缀名**（不能写 .class，也不能写 .java）。系统会自动找对应的 .class 文件执行。

# eclipse使用

## 识记：

### eclispe概念。

- **定义**：Eclipse 是一个开放源代码的、基于 Java 的**集成开发环境 (IDE)**。
- **特点**：核心很小，主要通过**插件 (Plugins)** 来扩展功能（“一切皆插件”）。
- **地位**：Java 开发中最主流的工具之一，与 IntelliJ IDEA 齐名。

## 领会 & 应用： Eclipse 从安装到运行

下面是一份在 Eclipse 环境下创建并运行第一个 Java 程序的完整流程。

### 1. Eclipse安装

-   **下载**: 前往 [Eclipse 官方下载页面](https://www.eclipse.org/downloads/)。
-   **版本**: 直接下载 "Eclipse IDE for Java Developers" 的最新版即可。它会提供一个安装器。
-   **安装**:
    -   运行下载的安装器。
    -   在列表中选择 "Eclipse IDE for Java Developers"。
    -   选择安装路径后，点击 "INSTALL"。
    -   安装完成后，启动 Eclipse。首次启动时，它会让你选择一个工作空间（Workspace），这是存放你所有项目代码的根目录，选择一个你方便管理的文件夹即可。

### 2. Eclipse环境下Java项目创建

-   **启动透视图 (Perspective)**: 确保你处于 Java 透视图。通常首次启动会默认进入。如果没有，可以点击右上角的 "Open Perspective" 按钮（像一个带加号的小窗口）选择 "Java"。
-   **创建项目**:
    1.  在左侧的 "Package Explorer" 视图中右键，选择 `New` -> `Java Project`。
    2.  **项目名 (Project name)**: 输入一个有意义的名字，例如 `MyFirstProject`。
    3.  **JRE**: 确保 "Use an execution environment JRE" 选项选择了你安装的 JDK 版本（如 JavaSE-17）。
    4.  点击 `Finish`。你的项目就会出现在左侧列表中。

### 3. 类的创建与运行

-   **创建包 (Package)**:
    1.  在 `MyFirstProject` 的 `src` 文件夹上右键，选择 `New` -> `Package`。
    2.  为包命名，这是一种组织代码的方式，规范要求使用反向域名，例如 `com.mycompany.hello`。初学可以简单命名为 `hello`。
    3.  点击 `Finish`。

-   **创建类 (Class)**:
    1.  在刚刚创建的 `hello` 包上右键，选择 `New` -> `Class`。
    2.  **类名 (Name)**: 输入 `HelloWorld` (遵循首字母大写的规范)。
    3.  **重要**: 勾选 `public static void main(String[] args)` 这个选项。这会让 Eclipse 自动为你生成 `main` 方法的框架。
    4.  点击 `Finish`。

-   **编写代码**:
    -   在 `HelloWorld.java` 文件中，Eclipse 已经为你生成了类和 `main` 方法的结构。
    -   在 `main` 方法的大括号 `{}` 内部，输入以下代码：
        ```java
        System.out.println("Hello, Eclipse!");
        ```
    -   Eclipse 拥有强大的代码提示功能，你输入 `Syso` 然后按 `Ctrl + Space` 就可以快速补全 `System.out.println();`。

-   **运行程序**:
    1.  在 `HelloWorld.java` 的代码编辑区内，直接右键。
    2.  选择 `Run As` -> `1 Java Application`。
    3.  程序会立刻执行，你会在下方的 "Console" (控制台) 视图中看到输出结果：`Hello, Eclipse!`。

至此，你已经完成了在 Eclipse 中的完整开发流程。IDE 极大地简化了编译和运行的步骤，让你能更专注于代码逻辑本身。
