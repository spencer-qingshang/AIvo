在华为 VRP 路由器或三层交换机上配置 DHCP 服务器，主要有两种方式：**接口地址池（快捷省事）**和**全局地址池（灵活强大，大题最爱考）**。

无论用哪种方式，永远记住**第一步绝对不能漏**，这是阅卷老师抓的第一个采分点：

Plaintext

```
<Huawei> system-view
[Huawei] dhcp enable       ! 极其重要：全局开启 DHCP 服务，没敲这句后面全白搭
```

---

### 方式一：基于接口的地址池（Interface Pool）

**适用场景：** 路由器直接作为内网用户的网关，且网关接口 IP 所在的网段就是分配给用户的网段。

**特点：** 配置极其简单，系统会自动把接口所在的网段作为地址池，接口的 IP 自动作为默认网关。

**实战配置：**

假设路由器 `G0/0/0` 接口连接局域网，网关 IP 为 `192.168.1.254/24`。

Plaintext

```
[Huawei] interface GigabitEthernet 0/0/0
[Huawei-GigabitEthernet0/0/0] ip address 192.168.1.254 24
[Huawei-GigabitEthernet0/0/0] dhcp select interface             ! 核心命令：告诉接口，用你自己的网段建立地址池
[Huawei-GigabitEthernet0/0/0] dhcp server dns-list 8.8.8.8      ! （可选）给终端分配 DNS 服务器
[Huawei-GigabitEthernet0/0/0] dhcp server excluded-ip-address 192.168.1.200 192.168.1.253 ! （常考排错）排除一段不想分配的 IP
[Huawei-GigabitEthernet0/0/0] dhcp server lease day 3           ! （可选）设置租期为 3 天，默认是 1 天
[Huawei-GigabitEthernet0/0/0] quit
```

### 方式二：基于全局的地址池（Global Pool）🌟（大题必考）

**适用场景：** 核心交换机集中管理多个 VLAN 的 IP 分配，或者配合 DHCP 中继（Relay）跨网段分配 IP。

**特点：** 地址池集中建在全局视图下，逻辑极其清晰，是考试中最常见的拓扑要求。

**实战配置：**

假设我们要建一个名为 `Pool_VLAN20` 的地址池，分配 `192.168.20.0/24` 网段，网关为 `192.168.20.254`。

Plaintext

```
! 1. 在全局视图下创建地址池，并配置各种参数
[Huawei] ip pool Pool_VLAN20                                    ! 创建名为 Pool_VLAN20 的全局地址池
[Huawei-ip-pool-Pool_VLAN20] network 192.168.20.0 mask 24       ! 宣告要分配的具体网段和掩码
[Huawei-ip-pool-Pool_VLAN20] gateway-list 192.168.20.254        ! 指定分配给客户端的默认网关
[Huawei-ip-pool-Pool_VLAN20] dns-list 114.114.114.114           ! 指定分配给客户端的 DNS
[Huawei-ip-pool-Pool_VLAN20] excluded-ip-address 192.168.20.250 192.168.20.253  ! 排除掉给打印机或服务器预留的静态 IP
[Huawei-ip-pool-Pool_VLAN20] quit

! 2. 进入网关接口，调用全局地址池（这步极易遗漏！）
[Huawei] interface Vlanif 20                                    ! 进入用户的网关接口（或者是物理接口）
[Huawei-Vlanif20] ip address 192.168.20.254 24
[Huawei-Vlanif20] dhcp select global                            ! 核心命令：告诉这个接口，去找全局地址池要 IP 发给用户
[Huawei-Vlanif20] quit
```

### 考场避坑与验证三板斧

在敲完配置后，考卷上经常会让你写出验证命令。熟练掌握这三步排错，保证万无一失：

**1. 检查地址池配置与使用情况：**

Plaintext

```
[Huawei] display ip pool name Pool_VLAN20
```

- **得分眼：** 敲下这条命令，能看到该地址池的总 IP 数（Total）、已经分配出去的 IP 数（Used）以及空闲数（Idle）。
    

**2. 致命易错点：掩码写错格式**

- 在全局地址池配置 `network` 时，华为 VRP 要求掩码必须明确使用 `mask` 关键字，例如 `network 192.168.1.0 mask 24` 或 `mask 255.255.255.0`。不能像 OSPF 那样写反掩码，也不能直接写 `/24`。
    

**3. 忘记挂载（Global 模式特有陷阱）**

- 很多考生在全局辛辛苦苦建好了 `ip pool`，配了网关配了 DNS，一测试发现 PC 还是拿不到 IP。原因就是忘了进到对应的 `Vlanif` 或物理接口下敲 **`dhcp select global`**。没有这根“导火索”，全局地址池就是个摆设。
