### 🎯 考场终极实战演练

**【考题背景】**

某企业深圳总部（R 1）与广州分部（R 2）刚刚拉通了一条跨市的广域网专线（接口均为 `Serial 4/0/0`）。由于该专线途经运营商的公共网络，为防止企业内部数据和账号密码在公网上被不法分子窃听，网络管理员决定在 R 1 和 R 2 之间启用身份认证。要求 R 1 作为主认证方，R 2 作为被认证方。

#### 第一步：场景判断（拿满理论分）

- **你的思考逻辑：** 题目中明确出现了**“途经运营商的公共网络”**和**“防止账号密码被窃听”**这两个致命关键词。
    
- **你的决断：** **必须使用 CHAP 认证！**
    
- **判断依据（答题话术）：** 因为 PAP 采用明文传输密码，在跨越不可信的公共广域网时存在极大的安全隐患。而 CHAP 采用三次握手机制，密码在网线上以 MD 5 哈希密文的形式传输，能有效防范抓包窃听和重放攻击，完全符合题目对高安全性的要求。
    

---

#### 第二步：实战配置代码流（拿满实操分）

既然敲定了 CHAP，我们立刻在脑海里调出对应的配置流。记住，主认证方（R 1）要建数据库并“设关卡”，被认证方（R 2）只负责“递交通行证”。

**1. 配置总部 R 1（主认证方）**

Plaintext

```
<R1> system-view

! 第一步：进入 AAA，建立合法的访客名单
[R1] aaa
! 创建专门用于 PPP 接入的账号 admin，配置密文密码，并限定服务类型为 ppp
[R1-aaa] local-user admin password cipher Huawei@123
[R1-aaa] local-user admin service-type ppp
[R1-aaa] quit

! 第二步：进入广域网物理接口，开启门禁
[R1] interface Serial 4/0/0
[R1-Serial4/0/0] link-protocol ppp                   ! 确认底层协议为 PPP（华为默认）
[R1-Serial4/0/0] ip address 10.1.1.1 255.255.255.0   ! 配置互联 IP
! 👇 核心指令：要求对端使用高安全的 CHAP 方式进行认证
[R1-Serial4/0/0] ppp authentication-mode chap
[R1-Serial4/0/0] quit
```

**2. 配置分部 R 2（被认证方）**

R 2 不需要建立 AAA，它只需要在接口上配置好自己的“身份证信息”，以便配合 R 1 发来的“随机数字”算出正确的哈希值。

Plaintext

```
<R2> system-view
[R2] interface Serial 4/0/0
[R2-Serial4/0/0] link-protocol ppp
[R2-Serial4/0/0] ip address 10.1.1.2 255.255.255.0

! 👇 核心指令：配置回应 CHAP 挑战时使用的用户名
[R2-Serial4/0/0] ppp chap user admin
! 👇 核心指令：配置配合 CHAP 计算哈希值的本地密码
[R2-Serial4/0/0] ppp chap password cipher Huawei@123

[R2-Serial4/0/0] quit
```

### 💡 考场极速反转（假设题目变了）

如果题目背景变成了：“两台路由器 R 1 和 R 2 放置在**同一个安全级别的核心机房内（或者你正在做 eNSP 模拟实验）**，为了**最快**拉通链路进行测试……”

- **你的决断：** **改为使用 PAP 认证。**
    
- **代码微调：**
    
    - R 1 接口下的门禁命令改为：`[R1-Serial4/0/0] ppp authentication-mode pap`
        
    - R 2 接口下的递交命令改为（把账号密码直接打包发过去）：`[R2-Serial4/0/0] ppp pap local-user admin password cipher Huawei@123`
