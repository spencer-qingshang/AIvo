# 项目3 运用路由器构建中型园区网络 (深度备考版)

## 一、 路由器基础与工作原理 (必考识记)

### 1.1 路由器的作用
*   **异构网互联**：连接不同类型的网络（如局域网与广域网）。
*   **路由转发**：根据**路由表**指导 IP 数据包选择最佳路径发送。
*   **隔离广播域**：路由器的每个接口都是一个独立的广播域。

### 1.2 视图模式切换 (基础)
*   **用户视图** `<Huawei>`：只能查看，不能配置。
*   **系统视图** `[Huawei]`：使用 `system-view` 进入，全局配置。
*   **接口视图** `[Huawei-GigabitEthernet0/0/0]`：配置 IP、使能协议。

## 二、 部门动态获取地址：DHCP 实战 (核心应用)

华为支持两种 DHCP 配置方式，考试常考区别：

| 特性 | 接口地址池 (Interface) | 全局地址池 (Global) |
| :--- | :--- | :--- |
| **配置位置** | 直接在接口下配置 | 在全局视图下配置，接口引用 |
| **适用场景** | 简单网络，一个接口对应一个池 | 复杂网络，AC/核心交换机统一管理多个池 |
| **灵活性** | 低，池子死磕在接口上 | 高，可跨接口、跨设备（配合中继）调用 |

### 核心命令：
```shell
# 方式1：接口地址池
[R1-G0/0/0] dhcp select interface
[R1-G0/0/0] dhcp server dns-list 8.8.8.8

# 方式2：全局地址池 (最常用)
[R1] ip pool pool1
[R1-ip-pool-pool1] network 192.168.1.0 mask 24
[R1-ip-pool-pool1] gateway-list 192.168.1.254
[R1] interface G0/0/0
[R1-G0/0/0] dhcp select global
```

## 三、 单臂路由：实现跨 VLAN 互访 (必考难点)

### 3.1 核心原理
由于路由器物理接口有限，通过在物理接口上创建**子接口 (Sub-interface)**，实现不同 VLAN 间的路由。

### 3.2 必背三行核心命令：
配置子接口时，这三行少一行都不通：
1.  **终结封装**：`dot1q termination vid [VLAN-ID]` (剥离并识别 VLAN 标签)。
2.  **配置 IP**：`ip address [IP] [Mask]` (作为对应 VLAN 的网关)。
3.  **开启 ARP 广播**：`arp broadcast enable` (**最易漏掉！** 路由器子接口默认不主动发 ARP，不敲此命令则无法获取 MAC 地址)。

## 四、 静态、默认与浮动路由 (重点应用)

### 4.1 静态路由 (Static)
*   **命令**：`ip route-static [目的网段] [掩码] [下一跳IP]`。
*   **特点**：手动指定，最稳定，但不可自动感知网络变化。

### 4.2 默认路由 (Default)
*   **命令**：`ip route-static 0.0.0.0 0.0.0.0 [下一跳IP]`。
*   **应用**：企业出口，去往互联网（所有未知目的地）的方向。

### 4.3 浮动路由 (Floating - 主备链路)
*   **原理**：配置两条静态路由，给其中一条设置较高的 **Preference (优先级)** 值（值越大优先级越低）。
*   **命令**：
    *   主线：`ip route-static 192.168.2.0 24 10.1.1.2` (默认优先级 60)
    *   备线：`ip route-static 192.168.2.0 24 20.1.1.2 preference 100`
*   **效果**：主线断开后，备线自动出现在路由表中。

## 五、 动态路由协议：RIPv2 与 OSPF (核心领会)

### 5.1 RIPv2 (距离矢量协议)
*   **度量值**：跳数 (Hop Count)，最大 15 跳，16 跳不可达。
*   **与 v1 区别**：v2 支持无类域间路由（携带掩码）、支持组播更新 (224.0.0.9)。
*   **核心配置**：
```shell
[R1] rip 1
[R1-rip-1] version 2
[R1-rip-1] undo summary  # 关闭自动汇总 (华为默认已关，但建议敲)
[R1-rip-1] network 192.168.1.0 # 必须宣告主类网段
```

### 5.2 OSPF (链路状态协议 - 现代网络主流)
*   **核心概念**：
    *   **Router ID**：路由器的唯一“身份证”，建议手动配置。
    *   **区域 (Area)**：Area 0 是**骨干区域**，所有其他非骨干区域必须连在 Area 0 上。
*   **核心配置 (单区域)**：
```shell
[R1] ospf 1 router-id 1.1.1.1
[R1-ospf-1] area 0
[R1-ospf-1-area-0.0.0.0] network 192.168.1.0 0.0.0.255 # 反掩码宣告
```

## 六、 考纲重点提炼：哪些必须背？

### 1. 必背逻辑
*   **单臂路由配置逻辑**：交换机侧设为 Trunk -> 路由器侧建子接口 -> 封装 dot1q -> 开启 ARP 广播。
*   **DHCP 获取逻辑**：客户端发送 Discover -> 服务端提供 Offer -> 客户端请求 Request -> 服务端确认 Ack。

### 2. 必背命令关键字 (华为)
*   `display ip routing-table` (查看路由表，分析网络必用)。
*   `dot1q termination vid` (子接口封装)。
*   `arp broadcast enable` (单臂路由生存之本)。
*   `preference` (浮动路由关键参数)。
*   `network [反掩码]` (OSPF 宣告方式)。

### 3. 常见故障分析点
*   **Ping 不通？**：
    1. 检查物理口是否 `undo shutdown`。
    2. 检查是否有回程路由。
    3. 单臂路由下，检查交换机 Trunk 链路是否放行了对应 VLAN。
    4. 检查 ARP 广播是否开启。
