# 项目 6：构建安全的园区网络 (深度备考版)

## 一、 接入安全：端口安全 (必考识记)

### 1.1 核心概念
*   **端口安全的作用**：通过限制接口学习到的 **MAC 地址数量**，防止黑客进行 MAC 地址溢出攻击（如交换机 CAM 表攻击）。
*   **动态 MAC (Dynamic)**：设备接入后自动学习，接口重启或老化后会丢失。
*   **Sticky MAC (粘性 MAC - 考纲重点)**：
    *   **特点**：将动态学习到的 MAC 地址转化为“粘性”地址。
    *   **优势**：保存配置后 **重启不丢失**。无需手动输入海量 MAC，又能固定设备接入，安全性与维护性的平衡。

### 1.2 保护动作 (Violation Mode - 必背)
当接入设备超过 `max-mac-num` 限制或 MAC 不匹配时：
1.  **Protect**：静默丢弃违规数据包，不发送告警。
2.  **Restrict (考纲应用重点)**：丢弃违规数据包，**发送 SNMP Trap 告警日志**，并记录违规次数。
3.  **Shutdown**：直接将物理接口 **Error-down** 关闭。需管理员手动 `undo shutdown` 或配置自动恢复。
    *   **自动恢复配置**：`error-down auto-recovery cause port-security interval 30` (单位：秒)

### 1.3 核心配置命令
```shell
[SW-G0/0/1] port-security enable             # 开启端口安全功能
[SW-G0/0/1] port-security max-mac-num 1      # 限制接口下最多学习1个MAC
[SW-G0/0/1] port-security mac-address sticky # 开启粘性MAC
[SW-G0/0/1] port-security protect-action restrict # 设置惩罚动作为restrict
```

## 二、 远程管理安全 (Telnet、SSH 与 AAA)

### 2.1 验证方式对比
*   **Password 验证**：只需输入一个全局密码。简单但不安全，无法追溯具体管理员。
*   **AAA 验证 (推荐)**：需要用户名 + 密码。支持权限分级（0-15级），审计清晰，安全性高。

### 2.2 协议对比 (必考)
*   **Telnet (TCP 23)**：**明文**传输数据，不安全。
*   **SSH / STelnet (TCP 22)**：**加密**传输数据，能够防止信息泄露和中间人攻击。在现网环境中应优先开启 `Stelnet server enable`。

### 2.3 AAA 核心配置命令
```shell
[R1] aaa
[R1-aaa] local-user admin password cipher Huawei@123
[R1-aaa] local-user admin service-type telnet ssh   # 授权Telnet和SSH登录
[R1-aaa] local-user admin privilege level 15        # 授权最高管理级别15

[R1] user-interface vty 0 4                         # 进入虚拟终端线路
[R1-ui-vty0-4] authentication-mode aaa               # 切换验证模式为AAA
```

## 三、 ACL 访问控制列表 (重难点 - 必须精通)

### 3.1 ACL 匹配规则与通配符
*   **匹配规则**：**从小到大，命中即止**。一旦匹配到某条 rule，后续规则不再检查。
*   **通配符 (Wildcard Mask)**：
    *   `0` 表示必须匹配（严格一致）；`1` 表示忽略。
    *   `0.0.0.255` 表示匹配前 24 位（网段）；`0.0.0.0` 表示精确匹配一个 IP。

### 3.2 分类与应用位置 (识记)
| 类型 | 编号范围 | 匹配依据 | 应用位置 |
| :--- | :--- | :--- | :--- |
| **基本 ACL** | 2000-2999 | **仅源 IP 地址** | **靠近目的地** |
| **高级 ACL** | 3000-3999 | 源/目 IP、**协议号、TCP/UDP 端口号** | **靠近源头** |

### 3.3 常用协议及端口号 (考填空/应用)
*   **FTP**: TCP 21 (控制) / 20 (数据)
*   **Telnet**: TCP 23
*   **HTTP**: TCP 80
*   **HTTPS**: TCP 443
*   **DNS**: UDP/TCP 53

### 3.4 ACL 应用方向
*   **Inbound (入方向)**：流量进入路由器接口之前进行过滤。
*   **Outbound (出方向)**：流量离开路由器接口之前进行过滤。

## 四、 广域网安全与 NAT 技术

### 4.1 PPP 认证对比 (必考简答)
| 认证方式 | 握手次数 | 安全性 | 特点 |
| :--- | :--- | :--- | :--- |
| **PAP** | 两次握手 | 低 | **明文**发送用户名和密码。 |
| **CHAP** | 三次握手 | 高 | **密文**挑战，密码不直接在链路上跑。 |

### 4.2 NAT (网络地址转换) 
#### 核心类型：
1.  **静态 NAT / NAT Server**：1对1。用于**内网服务器发布**。
    *   命令：`nat server protocol tcp global [公网IP] 80 inside [私网IP] 80`
2.  **Easy-IP (NAPT)**：多对1。内网用户共用路由器**出接口 IP** 上网。
    *   命令：接口下敲 `nat outbound 2000` (2000为定义了内网网段的ACL)。

#### NAT 的优点 (必考简答)：
1.  **节省公网 IP 地址资源**（最主要作用）。
2.  **隐藏内部网络拓扑**，增强内网安全性。
3.  **解决私网 IP 地址冲突**（重叠地址转换）。

## 五、 防火墙基础 (安全区域)

### 5.1 安全区域 (Zone) 与优先级
*   **Local (100)**：防火墙本体。
*   **Trust (85)**：内网信任区。
*   **DMZ (50)**：隔离区（放置服务器）。
*   **Untrust (5)**：外部非信任区（互联网）。

### 5.2 流量方向定义
*   **Outbound**：流量由 **高** 优先级区域向 **低** 优先级区域流动（如 Trust -> Untrust）。
*   **Inbound**：流量由 **低** 优先级区域向 **高** 优先级区域流动（如 Untrust -> DMZ）。

### 5.3 默认策略
*   **华为防火墙默认拒绝所有区域间的流量**。必须手动配置“安全策略（Security Policy）”才能放行。
