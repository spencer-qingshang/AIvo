理解 NAT 类型，核心就是抓准两个维度：**“几个私网对应几个公网”**，以及**“带不带端口转换”**。

在考卷的简答题或排错题中，请直接套用以下标准描述：

### 一、静态 NAT (Static NAT) —— “专属车位”模式

- **标准描述：** 将内部网络的私有 IP 地址与公网 IP 地址进行**一对一（1:1）的静态绑定**。转换时**不涉及端口号**的更改。
    
- **深度理解：** 极其奢侈。一旦绑定，不管这个私网设备上不上网，这个公网 IP 都被它霸占了，别人用不了。它支持双向通信，外网可以直接通过这个公网 IP 找到内网设备。
    
- **考场应用：** 现网极少用于普通上网，仅用于为内网**极少数特定的核心服务器**提供固定的外网访问地址。
    

### 二、动态 NAT (Dynamic NAT) —— “公共停车场”模式

- **标准描述：** 将内部网络的私有 IP 地址与公网 IP 地址池（Address Pool）中的地址进行**多对多（N:M）的动态映射**。转换时同样**不涉及端口号**。
    
- **深度理解：** 公司买了 5 个公网 IP 放进池子里。内网谁先发数据包，谁就拿走一个公网 IP，用完再还回来。如果 5 个 IP 都被占用了，第 6 个人就只能断网排队。
    
- **考场应用：** 依然浪费公网 IP，无法根本解决地址枯竭问题。现网中**基本已被淘汰**，考试时主要作为概念比对出现。
    

### 三、 NAPT (网络地址端口转换) —— “超级拼车”模式（🌟绝对核心）

- **标准描述：** 将多个内部私有 IP 地址映射到同一个（或少数几个）公网 IP 地址，并**通过不同的“传输层端口号（Port）”来区分不同的内部主机**。实现**多对一（N:1）**转换。
    
- **深度理解：** 拯救 IPv 4 的真神。成百上千台电脑共用一个公网 IP 出门，路由器给每个人分配一个临时的小纸条（端口号）。回来的时候，路由器看一眼纸条上的端口号，就知道该把数据包分给哪台具体的内网电脑。
    
- **考场应用：** **现网企业上网的绝对主流。** 只要题目要求“节约公网 IP”、“全公司共享上网”，必须选 NAPT。
    

### 四、 Easy IP —— NAPT 的“极简变体”（🌟华为配置必考）

- **标准描述：** 属于 NAPT 的一种特例。它不需要预先配置公网 IP 地址池，而是直接**利用路由器出接口（如拨号接口）当前动态获取的公网 IP 地址**作为转换后的地址。
    
- **深度理解：** 核心逻辑跟 NAPT 一模一样（也是靠端口区分）。但它极其聪明，专门为了应对那些“公网 IP 每天都在变”的场景（比如家庭宽带 PPPoE 拨号）。
    
- **考场应用：** 看到命令 `nat outbound 2000`（不带地址池参数），直接判定为 Easy IP。小型企业和家庭网络的最爱。
    

### 五、 NAT Server (内部服务器映射) —— “大堂接待员”模式

- **标准描述：** 将公网 IP 地址的**特定端口号**，静态映射到内网服务器的私网 IP 地址及**特定端口号**上。
    
- **深度理解：** 前面四种都是为了解决“里面的人怎么出去”，而 NAT Server 专为“外面的人怎么进来”而生。它不是把整台服务器暴露在公网上，而是只开一扇小窗（比如只映射 Web 服务的 80 端口），安全级别极高。
    
- **考场应用：** 题目出现“允许互联网用户访问企业内部的 Web / FTP 服务器”，毫不犹豫写下 `nat server` 的配置命令。

**🎯 考前一分钟极速回忆卡：**

- **只换 IP，一对一** = 静态 NAT
    
- **只换 IP，多对多** = 动态 NAT
    
- **连 IP 带端口一起换，多对一** = NAPT
    
- **连 IP 带端口一起换，且借用接口 IP** = Easy IP
    
- **公网端口映射私网端口（外网访问内网）** = NAT Server
