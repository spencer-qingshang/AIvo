在华为 VRP 设备的管理中，**Telnet** 和 **STelnet** 都是用来远程登录和管理路由器/交换机的协议（也就是让你不用抱着笔记本去机房插 Console 线，在工位上就能敲代码）。

这两者在考卷上的核心较量，只围绕两个字：**“安全”**。

### 一、核心对比速记表

|**对比维度**|**Telnet**|**STelnet (基于 SSH 协议)**|**🎯 考场得分点提炼**|
|---|---|---|---|
|**传输方式**|**明文传输 (Plaintext)**|**密文传输 (Ciphertext)**|Telnet 就像寄明信片，STelnet 就像把信锁进保险箱。|
|**默认端口**|TCP **23**|TCP **22**|选择题/填空题必考数字！|
|**安全性**|极差，容易被抓包软件截获密码|**极高**，数据被强加密，防篡改|现代网络环境强制要求使用 STelnet，淘汰 Telnet。|
|**配置复杂度**|简单|较复杂（必须生成加密密钥）|实操大题中，STelnet 会多出生成 RSA 密钥的步骤。|

_(注：STelnet 是华为设备上的专有名词，也就是大家常说的 SSH 远程登录。)_

---

### 二、考场实战：配置代码流的差异

这两者的配置在前期（如建立 AAA 本地用户）是完全一样的。**真正的分水岭在 VTY（虚拟终端）接口的放行，以及 STelnet 特有的加密服务。**

#### 1. 两者通用的“前置动作” (建立账号)

Plaintext

```
<Huawei> system-view
[Huawei] aaa
[Huawei-aaa] local-user admin password cipher Huawei@123
[Huawei-aaa] local-user admin privilege level 15             ! 给最高权限
[Huawei-aaa] local-user admin service-type telnet ssh        ! 宣告这个账号既能 Telnet 也能 SSH
[Huawei-aaa] quit
```

#### 2. 分水岭：如果你要配置 Telnet (传统且简单)

Plaintext

```
[Huawei] user-interface vty 0 4                              ! 进入虚拟终端接口，允许 5 个人同时登录
[Huawei-ui-vty0-4] authentication-mode aaa                   ! 认证模式为 AAA 账号密码
[Huawei-ui-vty0-4] protocol inbound telnet                   ! 核心：只允许 Telnet 协议进门
[Huawei-ui-vty0-4] quit
```

_至此，Telnet 就配置完了，直接可以用电脑连上来了。_

#### 3. 分水岭：如果你要配置 STelnet (大题最爱考，步骤更多)🌟

Plaintext

```
! 第一步：必须生成本地 RSA 密钥对（没有这把锁，加密就无从谈起。填空题必考！）
[Huawei] rsa local-key-pair create
  (系统提示按回车默认生成 1024 位或 2048 位密钥)

! 第二步：进入 VTY 接口放行协议
[Huawei] user-interface vty 0 4
[Huawei-ui-vty0-4] authentication-mode aaa
[Huawei-ui-vty0-4] protocol inbound ssh                      ! 核心：只允许 SSH (STelnet) 协议进门
[Huawei-ui-vty0-4] quit

! 第三步：全局开启 STelnet 服务
[Huawei] stelnet server enable
```

### 三、考场排错一眼看穿

如果在模拟器里配完了 STelnet 发现连不上，立刻排查这两个最容易忘的“暗雷”：

1. **忘记发钥匙：** 敲没敲 `rsa local-key-pair create`？没密钥，SSH 隧道根本建不起来。
    
2. **忘记开店门：** 敲没敲 `stelnet server enable`？华为部分设备的 STelnet 服务默认是关闭的，必须手动开启。
