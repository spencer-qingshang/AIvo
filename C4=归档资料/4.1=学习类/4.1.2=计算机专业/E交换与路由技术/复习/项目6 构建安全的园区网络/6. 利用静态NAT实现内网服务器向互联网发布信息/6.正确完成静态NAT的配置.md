在华为 VRP 体系中，静态 NAT（一对一绑定）的配置非常直接。考场上通常会遇到两种配置方法，阅卷老师都认，但你需要能看懂它们的区别：

### 🎯 考场实战场景

**背景：** 公司内网有一台极其重要的财务核心服务器（私网 IP：`192.168.1.100`）。为了让外部的审计部门能稳定访问它，公司向运营商花重金购买了一个专属的公网 IP：`200.1.1.10`。

**要求：** 将私网 IP `192.168.1.100` 与公网 IP `200.1.1.10` 进行一对一的静态绑定。路由器的出接口（连接公网）为 `GigabitEthernet 0/0/1`。

---

### 方法一：接口视图直接配置（最快、最推荐）

这是现网工程中最干脆利落的写法，一步到位，直接在连接外网的接口上敲定。

Plaintext

```
<Huawei> system-view
! 1. 直接进入连接公网的出接口
[Huawei] interface GigabitEthernet 0/0/1

! 2. 核心指令：将外部（global）的 200.1.1.10 与内部（inside）的 192.168.1.100 静态绑定
[Huawei-GigabitEthernet0/0/1] nat static global 200.1.1.10 inside 192.168.1.100

[Huawei-GigabitEthernet0/0/1] quit
```

### 方法二：全局视图配置 + 接口开启（极高频考点）

出题人非常喜欢在**排错题**里用这种方法挖坑。它的逻辑是：先在路由器的“全局大脑”里写好转换规则，然后再去具体的接口上把它“激活”。

Plaintext

```
<Huawei> system-view
! 1. 在全局视图下，写好一对一的映射规则
[Huawei] nat static global 200.1.1.10 inside 192.168.1.100

! 2. 进入连接公网的出接口
[Huawei] interface GigabitEthernet 0/0/1

! 3. 灵魂指令：必须在这个接口上激活静态 NAT 功能！（考试最容易漏掉这一句）
[Huawei-GigabitEthernet0/0/1] nat static enable

[Huawei-GigabitEthernet0/0/1] quit
```

---

### 💡 考场排错与验证必背

不管用哪种方法敲完，交卷前务必用下面这条命令检查一下你的劳动成果：

Plaintext

```
[Huawei] display nat static
```

- **得分眼：** 看屏幕输出的表格中，`Global IP`（公网 IP）和 `Inside IP`（私网 IP）是不是准确对应，并且确信已经应用到了正确的物理接口上。
