在《交换与路由技术》的考卷上，NAT 主要分为以下四大核心类型，它们的本质区别就在于**“公网 IP 的使用效率”**以及**“是否连带端口一起转换”**：

### 一、静态 NAT (Static NAT) —— 1 对 1 的“土豪模式”

- **工作原理：** 一个私网 IP 地址，固定死死地绑定一个公网 IP 地址。
    
- **核心特点：** 双向互通。内网电脑可以用这个公网 IP 上网，外网的人也可以通过这个公网 IP 直接主动访问这台内网电脑。
    
- **致命缺点：** 极其浪费公网 IP！公司有 100 个人，就得买 100 个公网 IP。
    
- **考试场景：** 现网中绝不会给普通员工电脑用，通常只用来给内部的**核心服务器**分配固定的公网“脸面”。
    

### 二、动态 NAT (Dynamic NAT) —— 多对多的“排队模式”

- **工作原理：** 路由器手里有一个“公网 IP 地址池”（比如包含 5 个公网 IP）。内网有 100 个人想上网，谁先发数据包，谁就先从池子里拿走一个公网 IP 用，用完再还不回去。
    
- **核心特点：** **不转换端口！只转换 IP！**
    
- **致命缺点：** 依然浪费！如果池子里只有 5 个 IP，那第 6 个人想上网时，就必须排队等前面的人下线。
    
- **考试场景：** 这种纯动态 NAT 在现网中**基本已经被淘汰**，考试时主要作为概念题出现，让你区分它和 NAPT 的不同。
    

### 三、 NAPT (网络地址端口转换) —— 多对一的“绝杀技术”🌟绝对核心

NAPT（Network Address Port Translation，在思科体系里叫 PAT）是拯救 IPv 4 地址枯竭的超级大英雄，也是现网最绝对的主流。

- **工作原理：** 允许成百上千个私网 IP，**同时共享 1 个（或少数几个）公网 IP** 上网。
    
- **它是怎么做到的？** 靠**端口号**区分！路由器不仅会把私网 IP 翻译成公网 IP，还会给每个内网电脑分配一个独一无二的“临时端口号”。
    
    - _举例：_ 电脑 A 和电脑 B 同时上网，路由器把它们都换成公网 IP `200.1.1.1`，但在发往公网时，电脑 A 带着端口 `10001`，电脑 B 带着端口 `10002`。等外网回包时，路由器看一眼端口号，就知道该把数据包还给内网的谁了。
        

### 四、 Easy IP —— NAPT 的“终极简化版”

这是华为设备里特有的叫法，也是大题里最常考的配置项。

- **工作原理：** 它本质上就是 NAPT。区别在于，普通 NAPT 还需要你手动建一个“公网 IP 池”；而 Easy IP 连池子都不建了，**直接借用路由器出接口（比如连接光猫那个拨号口）当前的公网 IP** 来做转换。
    
- **适用场景：** 小型企业、家庭宽带拨号上网。因为拨号上网的公网 IP 每天都在变，没法建固定地址池，用 Easy IP 自动绑定接口最完美。
    
- **配置口诀：** 只要看到代码里写着 `nat outbound 2000`（没有带 address-group 字眼），那就是 Easy IP。
    

### 五、 NAT Server (内部服务器映射) —— 专线迎客

前面三种都是为了“内网的人怎么出去”，而 NAT Server 是为了解决“外网的人怎么进来”。

- **工作原理：** 将公网 IP 的**特定端口**，精准映射到内网服务器的**特定端口**。
    
- **实战举例：** 把公网 IP `200.1.1.1` 的 `80` 端口，映射给内网服务器 `192.168.1.100` 的 `80` 端口。外网用户只需访问公网 IP 的 80 端口，路由器就会自动把他们领到内网的服务器上。
