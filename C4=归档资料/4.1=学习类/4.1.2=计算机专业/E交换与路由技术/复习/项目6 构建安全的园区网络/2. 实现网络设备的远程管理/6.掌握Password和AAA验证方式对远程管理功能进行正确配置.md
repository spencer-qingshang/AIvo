在华为 VRP 远程管理中，**Password（密码验证）** 和 **AAA（认证、授权、计费）** 是两种完全不同的安保逻辑。

---

### 一、 Password 验证方式（简单门栓）

- **逻辑：** “对事不对人”。不管是谁来登录，只要对上那个唯一的暗号（密码），就开门。
    
- **优缺点：** 配置极快，但安全性低，无法区分具体是谁在操作，审计困难。
    
- **实战配置（以 Telnet 为例）：**
    

Plaintext

```
<Huawei> system-view
[Huawei] user-interface vty 0 4
! 1. 核心指令：设置认证模式为单纯的密码验证
[Huawei-ui-vty0-4] authentication-mode password
! 2. 设置密文存储的验证密码（cipher 关键字是考点，保护配置文件不被偷看）
[Huawei-ui-vty0-4] set authentication password cipher Huawei@123
! 3. 极其重要：Password 模式必须手动指定权限等级，否则登录后什么都做不了
[Huawei-ui-vty0-4] user privilege level 15
[Huawei-ui-vty0-4] quit
```

---

### 二、 AAA 验证方式（严密门禁）🌟

- **逻辑：** “对人也对事”。每个人都有自己的专属工卡（账号+密码），保安（AAA 模块）会核对你的身份、能干什么、干了多久。
    
- **优缺点：** 安全性极高，支持多用户区分，是企业级实操大题的**默认要求**。
    
- **实战配置（以 STelnet/SSH 为例）：**
    

Plaintext

```
! 第一步：进入 AAA 视图，创建“员工档案”
[Huawei] aaa
! 1. 创建账号名为 admin，并设置密文密码
[Huawei-aaa] local-user admin password cipher Huawei@123
! 2. 给该账号分配权限等级（0-15，15 为最高）
[Huawei-aaa] local-user admin privilege level 15
! 3. 必须指定该账号可以使用的服务类型（Telnet 或 SSH）
[Huawei-aaa] local-user admin service-type telnet ssh
[Huawei-aaa] quit

! 第二步：进入 VTY 接口，调用“门禁系统”
[Huawei] user-interface vty 0 4
! 1. 核心指令：将认证模式改为 AAA
[Huawei-ui-vty0-4] authentication-mode aaa
[Huawei-ui-vty0-4] quit
```

### 三、考场对比与避坑指南

|**维度**|**Password 验证**|**AAA 验证**|**🎯 考场避坑指南**|
|---|---|---|---|
|**用户名**|**不需要**（直接弹密码框）|**必须有**|AAA 没建 `local-user` 绝对连不上。|
|**权限设置**|在 `vty` 接口下统一设置|在 `aaa` 的具体用户下设置|AAA 模式下在 VTY 配权限无效。|
|**灵活性**|所有人共用一个等级|不同用户不同权限|AAA 适合多人运维的大型网络。|
|**安全性**|较低|**高**|考试若提到“安全性”，首选 AAA。|

### 🌟 考前最后一遍“过电影”：

1. **Password 模式：** `authentication-mode password` ➔ `set authentication password cipher...` ➔ `user privilege level...`。
    
2. **AAA 模式：** 先进 `aaa` 建 `local-user`（名、密、级、类） ➔ 再进 `vty` 敲 `authentication-mode aaa`。
