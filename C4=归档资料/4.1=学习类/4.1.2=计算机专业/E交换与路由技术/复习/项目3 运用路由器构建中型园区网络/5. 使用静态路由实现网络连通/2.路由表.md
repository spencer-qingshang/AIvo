# 核心考点笔记：路由表 (Routing Table) 详解

## 一、什么是路由表？

**定义：** 路由表是路由器内部维护的一张“网络地图”或“导航系统”。

**核心作用：** 当路由器收到一个数据包时，它会提取数据包的**目的 IP 地址**，然后去查这张表，从而决定把数据包从哪个接口扔出去，以及交给下一个谁（下一跳）。

## 二、路由表的“三大来源” (填空/简答必考)

路由器里的“地图路线”是怎么来的？只有这三种方式：

1. **直连路由 (Direct)：** * **来源：** 只要给路由器的物理接口配了 IP 并且网线插好（接口双 Up），路由器就会天生知道自己身上连着哪些网段。
    
    - **特点：** 优先级最高，最可靠。
        
2. **静态路由 (Static)：**
    
    - **来源：** 网络管理员纯手工一条一条敲进去的（就是咱们上一节记的 `ip route-s`）。
        
    - **特点：** 稳定、不占资源，但死板，拓扑变了不会自动适应。
        
3. **动态路由 (Dynamic)：**
    
    - **来源：** 路由器之间运行相同的协议（如 **OSPF、RIP、BGP**），互相聊天交换情报，由算法全自动计算出来的路线。
        
    - **特点：** 聪明、自适应强，但会消耗路由器 CPU 和网络带宽。
        

## 三、剖析路由表的核心字段 (⚠️ 大题/选择题重灾区)

在华为设备上敲 `dis ip rou` (查看路由表)，出来的表格有几个致命考点字段，必须倒背如流：

|**关键字段**|**英文全称**|**核心含义与考点**|
|---|---|---|
|**目的/掩码**|Destination/Mask|目的地网段及子网掩码（例如 `192.168.2.0/24`）。|
|**协议**|Proto (Protocol)|这条路是怎么学来的？常见有 `Direct` (直连)、`Static` (静态)、`OSPF` 等。|
|**优先级**|Pre (Preference)|**⚠️ 极其重要：衡量不同协议的可靠度。数值越小，优先级越高！**<br><br>  <br><br>华为默认值必背：Direct= `0`，OSPF= `10`，Static= `60`，RIP= `100`。|
|**开销/度量值**|Cost|**⚠️ 极其重要：同一种协议内比距离。数值越小，路径越优！**|
|**下一跳**|NextHop|数据包离开本路由器后，到达的**下一个相邻路由器的入接口 IP**。|
|**出接口**|Interface|数据包从本路由器的哪个本地接口发出去。|

- **💡 黄金避坑口诀：** **“不同协议比优先级（Pre 小者胜），相同协议比开销（Cost 小者胜）。”**
    

## 四、路由器查表的第一铁律 (大题必考原则)

**【最长掩码匹配原则】 (Longest Prefix Match)**

- **规则：** 当一个数据包的目的 IP 在路由表里匹配到了**多条**可以走的路线时，路由器会选择**子网掩码最长（即范围最精确、最具体）**的那一条。
    
- **举例：** 目的 IP 是 `10.1.1.10`。
    
    - 路由表里有 `10.0.0.0/8` (能去) 和 `10.1.1.0/24` (也能去)。
        
    - **结果：** 路由器一定会选 `/24` 这条路，因为它描绘的目的地更精确！
