**DNS（Domain Name System，域名系统）地址**，通俗来讲就是互联网的**“114 查号台”或“通讯录”**。

在网络世界里，路由器和计算机只认识由数字组成的 IP 地址（比如 `110.242.68.3`），但人类根本记不住这么多数字，我们只能记住带有明确意义的域名（比如 `www.baidu.com`）。

当你用浏览器访问一个网站时，你的电脑会第一时间向 **DNS 地址**所在的那台服务器发问：“请问百度的 IP 地址是多少？”DNS 服务器查表后把数字 IP 告诉你，你的电脑才能真正发起连接。

对于咱们的日常上网和接下来的网络排错实战，你需要掌握以下关于 DNS 的核心知识：

### 1. 现实中最常用的公共 DNS 地址

如果你在现实中遇到电脑“能上 QQ，但是打不开网页”的经典故障，99% 是因为 DNS 地址配错了或者没获取到。手动将电脑网卡的 DNS 改为以下几个公用地址，就能瞬间修复：

- **114.114.114.114**：国内移动、电信和联通通用的公共 DNS，解析成功率极高，也是工程中最爱用的测试地址。
    
- **223.5.5.5 / 223.6.6.6**：阿里云提供的公共 DNS，速度很快，防劫持能力强。
    
- **8.8.8.8 / 8.8.4.4**：Google 提供的全球公共 DNS。
    
- **运营商本地 DNS**：比如广东电信通常会有自己专属的 DNS 地址，由光猫自动下发，解析本地网络速度最快。
    

### 2. VRP 实战：如何在局域网中下发 DNS（考试高频点）

在 4 月份的《交换与路由技术》考试中，通常不会让你给每台 PC 手动去填 DNS，而是要求你在路由器或核心交换机上配置 **DHCP 服务器**，让终端连上网线就能自动获取到 IP 和 DNS。

在 eNSP 中，这条命令是配置 DHCP 地址池时的**必考项**：

Plaintext

```
[Huawei] dhcp enable                                // 全局开启 DHCP 服务
[Huawei] ip pool OFFICE                             // 创建一个名为 OFFICE 的地址池
[Huawei-ip-pool-OFFICE] network 192.168.1.0 mask 24 // 宣告可分配的 IP 网段
[Huawei-ip-pool-OFFICE] gateway-list 192.168.1.254  // 下发网关地址
[Huawei-ip-pool-OFFICE] dns-list 114.114.114.114 8.8.8.8  // 【核心命令】下发主备 DNS 地址
[Huawei-ip-pool-OFFICE] quit
```

_（配置完地址池后，还要进到对应的接口下敲一行 `dhcp select global`，这套组合拳打完，底下的 PC 就能自动拿到能上网的完整网络参数了。）_

### 3. 路由器自身的 DNS 客户端功能

有时候，我们需要让路由器自己也能通过域名去 `ping` 外网（比如 `ping www.huawei.com`），用来测试网络连通性。这就需要给路由器本身配置 DNS 解析功能：

Plaintext

```
[Huawei] dns resolve                       // 开启路由器的动态域名解析功能
[Huawei] dns server 114.114.114.114        // 指定路由器去哪里查域名
```
