距离下个月的《交换与路由技术》考试越来越近了。在前一节我们聊到，手动配置静态 IP 极易引发令人崩溃的“IP 地址冲突”。为了彻底解决这个问题，并且极大地减轻网络工程师的负担，**DHCP（Dynamic Host Configuration Protocol，动态主机配置协议）**应运而生。

在真实的企业网中，成百上千台电脑、手机和打印机接入网络，全靠 DHCP 服务器在后台默默地为它们“发牌”（自动分配 IP 地址、子网掩码、网关和 DNS）。

在考试中，DHCP 的**报文交互过程（DORA）**和 **VRP 配置命令**是绝对的必考点。

### 1. 核心理论：DHCP 的“达尔文”四步交互（DORA）

当一台电脑刚插上网线（或连上 Wi-Fi）并设置为“自动获取 IP”时，它与 DHCP 服务器之间会进行一次经典的 4 步对话。**（请务必记住这四个报文的顺序和单/广播属性，选择题极爱考！）**

1. **第一步：Discover（发现）—— 电脑大喊：“谁能给我个 IP？”**
    
    - 电脑没有 IP，于是向全网发送一个 **DHCP Discover 广播报文**。寻找网络中隐藏的 DHCP 服务器。
        
2. **第二步：Offer（提供）—— 服务器回应：“我这有个 IP，你要吗？”**
    
    - DHCP 服务器收到广播后，从自己的地址池里挑出一个未被使用的 IP（比如 `192.168.1.10`），通过 **DHCP Offer 报文**（通常是单播，视客户端实现而定）发送给电脑。
        
3. **第三步：Request（请求）—— 电脑确认：“好的，我就用你给的这个 IP！”**
    
    - **⚠️ 考场陷阱**：电脑收到 Offer 后，会向全网发送一个 **DHCP Request 广播报文**。为什么要用**广播**？因为网络里可能有多个 DHCP 服务器都发了 Offer，电脑用广播是为了正式告诉“被选中的服务器”我用你的 IP，同时也是告诉“落选的服务器”你们可以把预留的 IP 收回去了。
        
4. **第四步：ACK（确认）—— 服务器盖章：“成交，租期是 8 天。”**
    
    - 服务器收到 Request 后，记录下这台电脑的 MAC 地址与 IP 的绑定关系，并回复一个 **DHCP ACK 报文**，正式确认电脑可以使用该 IP。
        

_（注：DHCP 是应用层协议，基于 UDP 传输，服务器端使用 67 端口，客户端使用 68 端口。）_

### 2. VRP 实战：在 eNSP 中配置全局 DHCP 服务器

在考卷的综合大题中，通常会要求你在路由器或三层交换机上配置 DHCP 服务。最经典、最常用的配置方式是**基于全局地址池（Global Pool）**。

假设我们要为内网的 VLAN 10（网段 `192.168.10.0/24`，网关 `192.168.10.254`）配置 DHCP 服务：

Plaintext

```
<Huawei> system-view
[Huawei] dhcp enable                                // 第1步：全局开启 DHCP 功能（如果不敲这句，后面的全白搭）

[Huawei] ip pool VLAN10_POOL                        // 第2步：创建一个名为 VLAN10_POOL 的地址池
[Huawei-ip-pool-VLAN10_POOL] network 192.168.10.0 mask 24 // 第3步：宣告可以分配的 IP 网段
[Huawei-ip-pool-VLAN10_POOL] gateway-list 192.168.10.254  // 第4步：下发网关地址给终端
[Huawei-ip-pool-VLAN10_POOL] dns-list 114.114.114.114     // 第5步：下发 DNS 服务器地址
[Huawei-ip-pool-VLAN10_POOL] excluded-ip-address 192.168.10.200 192.168.10.253 // (高阶考点) 排除一段 IP 不分配，留给网络打印机或服务器用固定 IP
[Huawei-ip-pool-VLAN10_POOL] quit

[Huawei] interface GigabitEthernet 0/0/0            // 第6步：进入连接该内网的物理接口（或者是三层交换机的 Vlanif 10 接口）
[Huawei-GigabitEthernet0/0/0] ip address 192.168.10.254 24 // 确保接口配了正确的网关 IP
[Huawei-GigabitEthernet0/0/0] dhcp select global    // 第7步：【极其关键】告诉这个接口，如果收到 DHCP Discover 报文，去全局地址池里找 IP 分配
[Huawei-GigabitEthernet0/0/0] quit
```

### 3. 终极验证与排错

配置完成后，在 eNSP 里把 PC 的 IPv 4 配置设为 **DHCP** 并点击应用。然后打开 PC 的命令行：

- 输入 `ipconfig`，如果你看到了 `192.168.10.X` 的地址，说明实验完美成功！
    
- 在路由器上输入 `display ip pool name VLAN10_POOL used`，可以查看当前地址池里有哪些 IP 已经被分配出去了，以及对应的电脑 MAC 地址。
    

---

如果 DHCP 服务器和需要获取 IP 的电脑**不在同一个网段**（中间隔着路由器，广播包过不去），DHCP Discover 报文就会被无情丢弃。这时候就需要用到一种极其巧妙的“代购”技术：**DHCP 中继（DHCP Relay）**。
