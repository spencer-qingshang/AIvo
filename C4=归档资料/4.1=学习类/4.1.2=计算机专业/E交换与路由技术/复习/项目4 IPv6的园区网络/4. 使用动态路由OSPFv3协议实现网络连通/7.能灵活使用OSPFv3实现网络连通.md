已经是 4 月 6 日了，距离《交换与路由技术》自考上考场只剩最后几天的时间！在这个节骨眼上，掌握 OSPFv 3 的“灵活使用”，就是要突破基础的单区域互通，拿下实操大题里最爱考的**多区域互联（ABR）**、**引入外部默认路由（ASBR）**以及**路由汇总**这三大高阶综合场景。

只要你把下面这个“三合一”的综合场景吃透，考场上无论题目怎么变形，你都能灵活应对。

假设考试给出了这样一个综合拓扑：

- **R 1 (ASBR 核心网关)：** 属于 Area 0，负责连接外网（ISP）。
    
- **R 2 (ABR 汇聚路由器)：** 左手连着 Area 0（对接 R 1），右手连着 Area 1（对接 R 3）。
    
- **R 3 (内部路由器)：** 属于 Area 1，下挂着多个内部办公网段（如 `2001:DB8:10::/64` 到 `2001:DB8:13::/64`）。
    

我们要实现：全网互通、内网能上网、且 Area 0 的路由表被精简。

---

### 灵活场景一：多区域互联与 ABR 部署 (R 2 的配置)

ABR（区域边界路由器）是多区域网络的核心。它的灵活之处在于：同一个 OSPFv 3 进程，不同的接口划入不同的区域。

Plaintext

```
<Huawei> system-view
[Huawei] sysname R2
[R2] ipv6
[R2] ospfv3 1
[R2-ospfv3-1] router-id 2.2.2.2
[R2-ospfv3-1] quit

! 左手接口，灵活宣告进 Area 0
[R2] interface GigabitEthernet 0/0/0
[R2-GigabitEthernet0/0/0] ipv6 enable
[R2-GigabitEthernet0/0/0] ipv6 address 2001:db8:12::2 64
[R2-GigabitEthernet0/0/0] ospfv3 1 area 0
[R2-GigabitEthernet0/0/0] quit

! 右手接口，灵活宣告进 Area 1
[R2] interface GigabitEthernet 0/0/1
[R2-GigabitEthernet0/0/1] ipv6 enable
[R2-GigabitEthernet0/0/1] ipv6 address 2001:db8:23::2 64
[R2-GigabitEthernet0/0/1] ospfv3 1 area 1
[R2-GigabitEthernet0/0/1] quit
```

### 灵活场景二：为全网引入缺省路由 (R 1 的配置)

内网的 R 2 和 R 3 都需要上网，但我们不可能在每台路由器上都写一条静态默认路由。最灵活的做法是：在出口网关 R 1 上写一条静态默认路由指向外网，然后通过 OSPFv 3 的 ASBR 功能，把这条缺省路由“自动下发”给全网。

Plaintext

```
<Huawei> system-view
[Huawei] sysname R1
[R1] ipv6
[R1] ospfv3 1
[R1-ospfv3-1] router-id 1.1.1.1
[R1-ospfv3-1] quit

! 1. 配置物理接口
[R1] interface GigabitEthernet 0/0/0
[R1-GigabitEthernet0/0/0] ipv6 enable
[R1-GigabitEthernet0/0/0] ipv6 address 2001:db8:12::1 64
[R1-GigabitEthernet0/0/0] ospfv3 1 area 0
[R1-GigabitEthernet0/0/0] quit

! 2. 写一条通往 ISP 外网的静态缺省路由（假设 ISP 接口 IP 为 2001:db8:8888::2）
[R1] ipv6 route-static :: 0 2001:db8:8888::2

! 3. 核心大招：在 OSPFv3 进程中强制下发默认路由
[R1] ospfv3 1
[R1-ospfv3-1] default-route-advertise always    ! always 参数表示无论本地路由表是否活跃，都强制下发缺省路由
[R1-ospfv3-1] quit
```

_(此时，R 2 和 R 3 的 IPv 6 路由表中，会自动多出一条 `::/0` 的外部 OSPFv 3 路由，全网打通上网路径！)_

### 灵活场景三：ABR 上的区域间路由汇总 (回到 R 2 配置)

假设 R 3 在 Area 1 下挂了 `2001:DB8:10::/64`、`11::/64`、`12::/64`、`13::/64` 四个连续网段。如果不做处理，R 2 会把这 4 条明细路由全部发给 Area 0 的 R 1。

最灵活的优化手段，是在 ABR（R 2）上执行汇总，只向 Area 0 发送一条 `2001:DB8:10::/62`。

Plaintext

```
[R2] ospfv3 1
[R2-ospfv3-1] area 1                                     ! 必须进入包含明细路由的源区域 (Area 1)
[R2-ospfv3-1-area-0.0.0.1] abr-summary 2001:db8:10:: 62  ! 执行区域间汇总
[R2-ospfv3-1-area-0.0.0.1] quit
```

### 考场终极验证口诀

一套复杂的配置敲完，监考老师/系统主要看你的检查结果。请熟练盲打这三条命令：

1. **查邻居稳不稳：** `display ospfv3 peer` (必须看到状态为 Full)
    
2. **查汇总成没成：** 在 R 1 上敲 `display ipv6 routing-table protocol ospfv3` (看是不是只有一条 /62 的汇总路由，而不是 4 条 /64 的明细)
    
3. **查缺省下发没：** 在 R 3 上敲 `display ipv6 routing-table protocol ospfv3` (看有没有 `::/0` 的路由条目)
