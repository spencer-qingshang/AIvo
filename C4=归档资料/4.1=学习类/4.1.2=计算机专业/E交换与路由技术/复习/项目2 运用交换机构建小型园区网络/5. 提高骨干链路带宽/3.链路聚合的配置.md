在华为交换机上配置链路聚合（Eth-Trunk），必须遵循一套严谨的逻辑：**先建立逻辑组，再加入物理成员，最后统一配置属性。**

以下是两种常见模式的配置手册：

### 1. 手工负载分担模式配置（最基础）

当你只需要快速叠加带宽，且两端物理链路非常稳定时使用。

**配置步骤：**
1.  **创建 Eth-Trunk 接口**：
    ```bash
    [SW1] interface Eth-Trunk 1
    ```
2.  **加入物理成员接口**（有两种方法）：
    - **方法 A（在接口下加）**：
      ```bash
      [SW1] interface GigabitEthernet 0/0/1
      [SW1-GigabitEthernet0/0/1] eth-trunk 1
      ```
    - **方法 B（在聚合组里拉）**：
      ```bash
      [SW1] interface Eth-Trunk 1
      [SW1-Eth-Trunk1] trunkport GigabitEthernet 0/0/1 to 0/0/3
      ```
3.  **配置逻辑属性**（如设为 Trunk）：
    ```bash
    [SW1-Eth-Trunk1] port link-type trunk
    [SW1-Eth-Trunk1] port trunk allow-pass vlan 10 20
    ```

### 2. LACP 模式配置（推荐/进阶）

当你需要更高的可靠性，或者需要“M 根活跃，N 根备份”的冗余设计时使用。

**配置步骤：**
1.  **创建接口并切换模式**：
    ```bash
    [SW1] interface Eth-Trunk 1
    [SW1-Eth-Trunk1] mode lacp  # 核心步骤：将默认的手工模式切换为 LACP
    ```
2.  **加入成员口**：
    ```bash
    [SW1-Eth-Trunk1] trunkport GigabitEthernet 0/0/1 to 0/0/2
    ```

### 3. 验证与检查命令（排错神器）

配置完成后，必须检查状态是否为 **Selected（选中/活跃）**。

- **查看聚合组摘要**：
  `display eth-trunk 1`
  - **关键指标**：`Status` 应为 `Up`；`MemberPort` 的 `Status` 应为 `Selected`。如果显示 `Unselect`，说明配置有冲突或物理链路有问题。

---

### 4. 考点挑战：配置纠错与实战

**场景描述**：
小张在 eNSP 里配置 SW1 和 SW2 的聚合。他先给 GE0/0/1 配了 `port link-type trunk`，然后尝试在接口下敲 `eth-trunk 1`，结果系统报错拒绝执行。

**任务**：
1. 请解释系统报错的原因。
2. 请给出正确的纠正步骤。

**标准答案：**
1.  **报错原因**：华为 VRP 平台要求加入 Eth-Trunk 的物理成员接口必须是**“白板状态”**（即没有特殊的二层/三层配置）。因为小张已经给 GE0/0/1 配了 Trunk，这与聚合组可能存在的默认配置不匹配，系统为了防止冲突会拒绝加入。
2.  **纠正步骤**：
    - 第一步：进入 GE0/0/1，清除之前的配置（`undo port link-type` 或恢复默认）。
    - 第二步：将“干净”的物理口加入 Eth-Trunk。
    - 第三步：**所有的业务配置（如 Trunk, VLAN 允许列表）统一在逻辑接口 Eth-Trunk 下完成**，配置会自动同步给成员口。

---

**实战逻辑总结**

在大型组网中，请记住：**物理口只是“搬运工”，Eth-Trunk 才是“指挥官”。**
一旦接口加入聚合组，你就不应该再单独进入物理接口敲任何业务命令。

掌握了链路聚合，你是否已经察觉到一个潜在的威胁：如果我们在两台交换机之间连了多根线，但在配置聚合之前不小心把线全插上了，会发生什么？没错，那就是恐怖的**网络环路**。为了彻底消灭环路，我们需要进入 **项目 2.6：避免网络环路（STP/RSTP）** 的学习。
