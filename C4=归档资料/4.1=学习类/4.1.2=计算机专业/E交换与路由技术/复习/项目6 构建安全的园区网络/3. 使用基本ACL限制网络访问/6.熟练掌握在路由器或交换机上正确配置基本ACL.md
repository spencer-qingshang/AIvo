在《交换与路由技术》的实操考试中，**基本 ACL（编号 2000-2999）**的配置是最基础也是必拿的分数。它的核心特点是**“只看源 IP 地址”**。

为了在考场上做到肌肉记忆般的熟练，我们通过一个最经典的考试场景，把配置代码流和易错点彻底砸实。

### 🎯 实战场景设定

**考试要求：**

公司网络中，研发部网段为 `192.168.1.0/24`，其中有一台违规主机 PC 1（IP：`192.168.1.10`）。财务部服务器网段为 `192.168.2.0/24`。

要求：**禁止 PC 1 访问财务部服务器，但允许研发部其他所有电脑访问财务部服务器。**

---

### 满分配置“三步曲”

在配置之前，牢记基本 ACL 的**部署灵魂法则：尽可能靠近目的端（Destination）！**

因此，我们必须来到**连接财务部服务器的那台路由器/交换机**上进行配置，而不是在研发部的网关上配置（否则 PC 1 连外网都上不了了）。

#### 第一步：创建基本 ACL 并编写规则

Plaintext

```
<Huawei> system-view
! 1. 创建编号为 2000 的基本 ACL（进入 ACL 视图）
[Huawei] acl 2000

! 2. 核心考点：先写范围小的，再写范围大的（自上而下匹配原则）
! 拒绝 PC1 这台特定主机（通配符掩码 4 个 0 代表精确匹配）
[Huawei-acl-basic-2000] rule 5 deny source 192.168.1.10 0.0.0.0

! 允许研发部网段的其他所有人（通配符掩码 0.0.0.255 代表匹配前三位）
[Huawei-acl-basic-2000] rule 10 permit source 192.168.1.0 0.0.0.255

! 3. 退出 ACL 视图
[Huawei-acl-basic-2000] quit
```

_💡 考场避坑：很多同学会忘记敲 `rule 10`。如果只敲了 `rule 5`，由于华为包过滤默认放行其他人，虽然也能通，但在严谨的考试中，明确写出 `permit` 规则能确保不被扣分。_

#### 第二步：进入正确的接口（找准大门）

我们需要把这套规则挂载到**靠近财务部服务器的那个接口**上。假设该接口是 `GigabitEthernet 0/0/2`。

Plaintext

```
[Huawei] interface GigabitEthernet 0/0/2
```

#### 第三步：调用 ACL 并区分进出方向（Outbound / Inbound）

这一步决定了规则的生死！

- 如果数据包是准备**离开**路由器，发往服务器，那就用 `outbound`。
    
- 如果数据包是刚刚**进入**路由器，那就用 `inbound`。
    

在这个场景中，数据包是从研发部发来，正准备从 `G0/0/2` 接口**出去**，发给下面连接的服务器。

Plaintext

```
! 核心指令：在出方向上调用包过滤 ACL 2000
[Huawei-GigabitEthernet0/0/2] traffic-filter outbound acl 2000
[Huawei-GigabitEthernet0/0/2] quit
```

---

### 🌟 考场排错与验证必背命令

配置全部敲完后，千万别急着交卷，敲下这两条命令验证你的成果：

**1. 检查 ACL 规则是否写错（查名单）：**

Plaintext

```
[Huawei] display acl 2000
```

_看屏幕输出，确认 `deny` 和 `permit` 的顺序对不对，通配符掩码有没有敲反（比如把 `0.0.0.255` 错敲成了 `255.255.255.0`）。_

**2. 检查接口是否调用成功（查保安是否上岗）：**

Plaintext

```
[Huawei] display traffic-filter applied-record
```

_看屏幕输出，确认 ACL 2000 是不是稳稳地挂在了 `G0/0/2` 接口的 `Outbound` 方向上。_

只要把这套代码流和“靠近目的端”的原则吃透，无论考卷上的拓扑图画得多复杂，找准那台离目标最近的设备，一套连招下去，分数就稳稳到手了！
