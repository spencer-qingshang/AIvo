# 核心考点笔记：路由原理与静态路由配置

## 一、路由器数据转发的“三步曲”

路由器在收到一个数据包时，只做三件事：

1. **拆解提取：** 拆开数据包的二层帧头，提取出第三层（网络层）的**目的 IP 地址**。
    
2. **查表匹配（核心）：** 在自己的路由表（Routing Table）中寻找与目的 IP 匹配的网段。
    
    - **💡 必考原则：最长掩码匹配原则 (Longest Match)。** 如果路由表中有多个网段都能包含这个目的 IP，路由器会选择子网掩码最长（即网络范围最精确）的那一条路径。
        
3. **转发或丢弃：** * 如果找到匹配的路由，则根据表项指示的**出接口**和**下一跳**进行转发。
    
    - 如果找遍路由表都没有匹配项，且没有配置默认路由，路由器会直接**丢弃**该数据包。
        

## 二、剖析路由表 (Routing Table)

- **查看命令：** `display ip routing-table`
    
- **路由表核心字段（选择/填空题重灾区）：**
    
    - **Destination/Mask (目的/掩码)：** 目标网络地址及其子网掩码。
        
    - **Proto (Protocol 协议)：** 这条路由是怎么学来的。常见有 `Direct` (直连)、`Static` (静态)、`OSPF`、`RIP`。
        
    - **Pre (Preference 优先级)：** 衡量不同路由协议的可靠程度。**数值越小，优先级越高！**
        
        - _⚠️ 华为默认优先级（必背）：_ Direct = `0`，OSPF = `10`，Static = `60`，RIP = `100`。
            
    - **Cost (开销/度量值)：** 当到达同一个目的地有多条**同种协议**的路由时，比较 Cost。**数值越小越优！**（代表距离越近或带宽越宽）。
        
    - **NextHop (下一跳)：** 数据包离开本路由器后，到达的**下一个相邻路由器**的接收接口 IP 地址。
        
    - **Interface (出接口)：** 数据包从本路由器的哪个物理或逻辑接口发送出去。
        

## 三、静态路由 (Static Route) 配置指南

静态路由是网络管理员手工一笔一画配置的地图，指引数据包如何走向非直连的网段。

**1. 静态路由优缺点**

- **优点：** 配置简单，不占用路由器的 CPU 算力和网络带宽（因为不需要像动态路由那样频繁发包交换信息），安全性极高。
    
- **缺点：** “死板”。当网络拓扑发生物理变化（比如网线断了），静态路由无法自动感知和切换，必须由管理员手工重新配置。适用于中小型、稳定的网络。
    

**2. 核心配置公式**

必须在**系统视图** `[Huawei]` 下配置：

> `ip route-static <目的网段IP> <目的网段子网掩码> <下一跳IP地址>`

- **举例：** 你的路由器要去 `192.168.2.0/24` 这个网段，隔壁连接你的路由器接口 IP 是 `10.1.1.2`。
    
- **命令：** `ip route-static 192.168.2.0 255.255.255.0 10.1.1.2`
    
    _(也可以简写为：`ip route-static 192.168.2.0 24 10.1.1.2`)_
    

**3. 经典变种：默认路由 (Default Route) / 缺省路由**

默认路由是静态路由的一种极其特殊的形态。它代表“所有未知的目的地”。当路由器查遍路由表找不到具体方向时，就会把数据包统统丢给默认路由（通常指向互联网的出口网关）。

- **配置公式：** 目的网段和掩码全是 0。
    
- **命令：** `ip route-static 0.0.0.0 0.0.0.0 <下一跳IP地址>`
