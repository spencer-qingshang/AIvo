# 核心考点笔记：默认路由与浮动路由综合配置 (纯简写版)

**实验场景：**

- **主链路 (ISP-1)：** 1000 M 光纤，下一跳 IP 为 `10.1.1.2`。
    
- **备份链路 (ISP-2)：** 100 M 宽带，下一跳 IP 为 `20.1.1.2`。
    
- **目标：** 电脑访问互联网时，优先走 ISP-1；当 ISP-1 断开时，自动切换到 ISP-2。
    

---

## 一、正确配置步骤

### 1. 基础接口配置

Plaintext

```
<Huawei> sys
[Huawei] sysn R1
[R1] int g0/0/0
[R1-GigabitEthernet0/0/0] ip add 10.1.1.1 24    // 主链路接口
[R1-GigabitEthernet0/0/0] int g0/0/1
[R1-GigabitEthernet0/0/1] ip add 20.1.1.1 24    // 备份链路接口
[R1-GigabitEthernet0/0/1] q
```

### 2. 部署默认路由 + 浮动备份

这里是整门课最核心的两个 `ip route-s` 命令：

Plaintext

```
// 1. 配置主用默认路由 (保持默认优先级 60)
[R1] ip route-s 0.0.0.0 0 10.1.1.2

// 2. 配置浮动备份默认路由 (手动设置优先级为 100)
[R1] ip route-s 0.0.0.0 0 20.1.1.2 pre 100      // pre 100 使其成为“备胎”
```

---

## 二、验证与故障模拟 (上机考试必做)

### 步骤 1：检查正常状态

Plaintext

```
[R1] dis ip rou
```

- **预期结果：** 你在路由表中只能看到去往 `10.1.1.2` 的那条 `0.0.0.0/0`。因为它的优先级（60）优于备份路径（100）。
    

### 步骤 2：模拟主链路故障

Plaintext

```
[R1] int g0/0/0
[R1-GigabitEthernet0/0/0] sh    // 强行关闭主接口 (shutdown)
```

### 步骤 3：观察路由“浮动”

Plaintext

```
[R1] dis ip rou
```

- **预期结果：** 原本 60 的路由消失，**下一跳为 20.1.1.2、Pre 为 100 的路由自动浮现**。此时网络依然保持连通。
    

### 步骤 4：恢复主链路

Plaintext

```
[R1-GigabitEthernet0/0/0] un sh // 重新开启主接口 (undo shutdown)
[R1-GigabitEthernet0/0/0] dis ip rou
```

- **预期结果：** 主路由（60）由于优先级更高，会立即**抢占**回路由表，备份路由再次进入隐藏（浮动）状态。
    

---

## 三、总结与避坑指南

1. **默认路由的精髓：** 目的和掩码全是 `0`，代表“所有流量”。
    
2. **浮动路由的灵魂：** 必须设置一个比主路由**更大**的 `pre` 值。
    
3. **考试重点：** 如果题目没要求具体数值，备份路径设为 `70` 或 `100` 均可，只要比 `60` 大。
